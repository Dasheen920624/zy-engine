# CHANGELOG

本文件只记录项目重启节点后的客户可见变更。项目尚未正式上线，重启前历史不进入当前工作树；需要追溯时使用 Git 历史。

版本号遵循 [Semantic Versioning 2.0](https://semver.org/)，维护约定见 [VERSIONING.md](VERSIONING.md)。

## [Unreleased]

### Added

- 新增中文文档与 AI 协作规范：当前文档主体必须使用简体中文，功能完成后通过 PR 合并到 GitHub 远程 `main`。
- 新增 MedKernel v1.0 GA 基础底座：统一 API 包络、错误处理、追踪上下文、组织范围、审计事件发布和 5 方言迁移基线。
- 新增组织单元 API、JWT 组织范围注入、声明式权限模型和数据范围切面，为后续控制器提供统一安全模板。
- 新增术语映射 API 能力：标准术语、本地术语、候选映射、冲突处理、人工确认、术语包发布与回滚。
- 新增审计事件持久化 OpenSpec 变更，用于把业务操作留痕落库并向合规审计 API 提供真实数据。
- 新增容器化开发平台 OpenSpec 与 Superpowers 设计：MedKernel PostgreSQL、后端、前端网关、Neo4j、监控和独立 Dify 运行边界。
- 新增前端产品体验重构设计：路由元数据单一来源、5 组主菜单、页面六态、7 步配置流和逐页体验治理。
- 新增登录页与前端基础体验优化，为后续业务页面逐页重构提供统一视觉与交互基线。

### Changed

- 项目运行方式收束为远程 `main` 单主干：所有功能、修复和文档治理均从 `origin/main` 开短分支，经 PR、远端检查后合并回远程 `main`。
- 当前权威文档不再引用旧归档作为事实源；历史设计中仍有效的业务范围已经吸收到产品宪法、实施落地方案、体验规范和业务场景详细规范。
- README、文档索引、OpenSpec 配置和 PR 模板统一为中文协作口径。
- 部署说明收束到当前 Docker 开发平台和运行手册；旧部署路径不再作为当前入口。

### Removed

- 移除当前工作树中的历史归档、旧任务锁、旧 AI 任务卡、旧开发分支脚本和旧前置检查脚本。
- 移除以旧分支模型为前提的协作口径；当前工作流只承认远程 `main` 单主干。

### Security

- PR 模板新增医疗安全、中文文档、远程 `main` 合并门禁。
- 保留前端历史命名禁用规则，防止旧品牌、旧路径和旧环境变量重新进入代码。

# 02 任务台账

## 0. 用途

本文是 **项目内唯一权威任务台账**。所有任务的当前状态、负责人/AI、独占文件范围、依赖、最低能力等级、归属泳道都以此为准。

任何 AI 领任务前必须做的事：

1. 在台账中找到任务编号（如未存在 → 先在"新增任务申报区"登记并 push）。
2. 把状态从 `TODO` 改成 `IN_PROGRESS`，写上你的代号 + 开始时间。
3. commit + push 台账更新（**这一次 push 是"占领"**）。
4. 完成后状态改 `DONE`，写完成时间 + commit hash + 一行核心改动。
5. commit + push。

> 状态值：`TODO / IN_PROGRESS / BLOCKED / DONE / DEPRECATED`。
> 任何同一编号同时存在多个 `IN_PROGRESS` 视为冲突，必须立即协调。

## 6. 维护规则

- 状态变更必须 commit + push，否则视为"未占领"。
- DEPRECATED 任务不删除，加 `~~ ~~` 删除线和 deprecation 理由。
- 同一编号执行过多批时，新增"第 N 批"段，不覆盖历史 DONE 记录。
- 本文是活文档，对应 DOC-xxx 任务。

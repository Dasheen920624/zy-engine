# P1研发任务拆解

## 1. P1目标

P1 目标是把当前 MVP 从“硬编码样例可运行”推进到“配置可导入、规则可模拟、路径可发布、过程可审计”的工程化闭环。

P1 完成后，AMI/STEMI 路径应可以通过配置文件导入，而不是写死在 Java 代码中。

## 2. 后端任务

### P1-BE-001 规则定义管理

范围：

- 新增规则导入接口。
- 保存规则定义到 `RE_RULE_DEF`。
- 支持规则状态：`DRAFT`、`PUBLISHED`、`DISABLED`。
- 支持规则版本号。

接口：

- `POST /api/rules`
- `POST /api/rules/{ruleCode}/publish`
- `GET /api/rules/{ruleCode}`
- `GET /api/rules`

验收：

- `sample_ami_rules.json` 可导入。
- 同一 `rule_code + version_no` 不重复。
- 发布后可用于模拟执行。

### P1-BE-002 规则 DSL 执行器

范围：

- 支持 `all`、`any` 条件组合。
- 支持 `equals`、`in`、`contains`、`exists`、`within_minutes_from`。
- 支持从患者上下文读取嵌套字段。
- 支持缺失数据返回明确原因。

验收：

- STEMI 候选规则命中。
- 心电图 10 分钟内完成返回合格。
- 心电图超时返回质控命中。
- 缺少 `arrival_time` 不抛系统异常。

### P1-BE-003 规则执行日志

范围：

- 写入 `RE_RULE_EXEC_LOG`。
- 记录 `trace_id`、`rule_code`、命中结果、输入快照、输出快照、耗时、异常。

验收：

- 每次模拟执行可查询到日志。
- 日志中不保存数据库密码、接口密钥等敏感信息。

### P1-BE-004 路径配置导入与发布

范围：

- `POST /api/pathways` 接收路径配置。
- 校验路径编码、节点编码、任务编码、流转目标。
- 保存 `PE_PATHWAY_DEF` 和 `PE_PATHWAY_VERSION`。
- 发布后可作为患者入径依据。

验收：

- `sample_ami_pathway.json` 可导入。
- 路径发布后状态为 `PUBLISHED`。
- 配置 JSON 原文可回查。

### P1-BE-005 患者路径实例与节点流转配置化

范围：

- 入径时读取已发布路径版本。
- 首节点由配置决定。
- 完成节点时根据 `transitions` 判断下一节点。
- 写入节点状态、任务状态和审计日志。

验收：

- AMI 入径首节点为 `AMI_CHEST_PAIN_IDENTIFY`。
- 完成首节点后流转到 `AMI_REPERFUSION_EVAL`。
- 重复入径不创建重复 ACTIVE 实例。

### P1-BE-006 图谱引擎 Neo4j 适配

范围：

- 增加 Neo4j 配置。
- 实现候选疾病召回 Cypher。
- 实现证据查询 Cypher。
- 支持图谱版本参数。

验收：

- 根据 `CHEST_PAIN` 和 `ST_ELEVATION_CONTIGUOUS_LEADS` 返回 `AMI_STEMI`。
- 返回匹配关系和证据 ID。
- Neo4j 不可用时返回可解释降级结果。

### P1-BE-007 Dify 适配器

范围：

- 增加 Dify API 地址和 Token 配置。
- 调用工作流运行接口。
- 支持超时、错误降级和调用日志。
- 返回解释文本和结构化补充信息。

验收：

- Dify 可用时返回推荐解释。
- Dify 不可用时路径推荐仍然可用。

## 3. 前端/原型任务

### P1-FE-001 医生确认入径页面

内容：

- 展示候选路径。
- 展示综合评分、命中证据、缺失资料。
- 支持确认入径、暂不入径、记录原因。

验收：

- 医生能在 30 秒内理解推荐原因。
- 关键建议不遮挡、不混乱。

### P1-FE-002 规则模拟页面

内容：

- 选择规则。
- 粘贴患者上下文。
- 查看命中结果、原因、缺失字段。

验收：

- 工程人员可用样例数据完成规则调试。

### P1-FE-003 路径配置页面

内容：

- 阶段、节点、任务、规则绑定、Dify 绑定。
- 支持 JSON 导入和结构化预览。
- 支持发布前校验。

验收：

- 产品/实施人员可理解路径结构。

## 4. 测试任务

### P1-QA-001 自动化接口测试

范围：

- 健康检查。
- 规则模拟。
- 路径导入发布。
- 候选推荐。
- 医生确认入径。
- 节点流转。

### P1-QA-002 数据库验证

范围：

- Oracle 表存在。
- 表备注和字段备注为中文。
- 推荐记录、路径实例、节点状态、规则日志可写入。

### P1-QA-003 编码验证

范围：

- 文档中文正常。
- JSON 样例可解析。
- API 返回中文正常。
- Oracle 中文备注正常。

## 5. P1完成定义

P1 完成必须同时满足：

- GitHub CI 通过。
- 本地 JDK 1.8 构建通过。
- AMI 样例端到端跑通。
- Oracle 模式可落库。
- 核心配置不再依赖硬编码。
- 关键接口有样例请求和验收结果。

# 00 总入口与 AI 接手导航

## 0. 一句话

本项目是面向集团化医院、多院区、卫生所/站点和科室的 **医疗智能引擎平台**（路径 + 规则 + 图谱 + 标准化 + 适配器 + Dify/AI 工作流 + 质控 + 配置发布 + 前端配置/演示校验）。生产库与开发库必须严格区分：Oracle 是当前生产权威验收库，达梦 / PostgreSQL / KingbaseES 是生产交付兼容库，`LOCAL_H2_FILE` 只作为无内网数据库时的开发本地文件库。Neo4j 是图谱投影，Dify 是 AI 工作流执行目标，DB-only 模式必须独立可验收。

后续会有 **多个 AI（不同能力）和多位工程师并行开发**。本文是任何 AI、任何能力等级接手时必须先读的 **唯一入口**。

### Step 6（硬门禁）— 生产库 / 开发库分层

开发前必须先确认当前任务运行在哪一类数据库上：

| 类型 | 允许用途 | 禁止事项 |
|---|---|---|
| 生产权威库：Oracle | 正式验收、真实落库、上线迁移、审计追责 | 不允许用 H2 结果冒充 Oracle smoke |
| 生产兼容库：达梦 / PostgreSQL / KingbaseES | 国产化和客户部署交付 DDL、迁移脚本、差异说明 | 不允许只维护 Oracle，不同步兼容库结构 |
| 开发本地库：`LOCAL_H2_FILE` | 无 Oracle/内网环境时的 AI 开发、DB-only 验证、本地文件持久化 | 不允许作为生产替代库，不允许提交本地数据文件 |
| `IN_MEMORY_DEMO` | 快速演示、纯单测兜底 | 不允许作为持久化、DDL、跨实例验收依据 |

任何新增表、索引、约束、持久化 SQL 或落库行为，都必须说明“生产库如何验收”和“开发库如何验证”。无 Oracle 时也必须跑 `LOCAL_H2_FILE`，并把生产库 smoke 标为待有环境补验。

## 2. 开发硬性 5 条（任何任务都适用）

| 编号 | 规则 | 解释 |
|---|---|---|
| H1 | **只改任务台账声明的独占文件** | 共享文件需先在台账登记申请，避免与其它 AI 冲突 |
| H2 | **不硬编码单医院 / 单院区 / 单系统厂商逻辑** | 差异沉淀为配置包、字典映射、适配器绑定、Provider |
| H3 | **Neo4j / Dify 不可用时核心路径/规则必须仍可用** | DB-only 模式必须可独立完成验收 |
| H4 | **关键医疗 / 医保 / 质控配置必须有来源追溯** | 缺来源、来源过期、未审核不得发布 |
| H5 | **临床建议必须由医生最终确认** | 系统给建议，医生做决定；阻断类规则必须院方授权 |
| H6 | **生产库和开发库分离** | Oracle/DM/PG/Kingbase 是生产交付口径；LOCAL_H2_FILE 只做开发等价验证 |

## 3. 完成前 Definition of Done（统一口径）

> 本节是 **唯一权威 DoD**，其它文档若有冲突以本节为准。

### 3.1 代码与文档

- [ ] 只修改了任务台账声明的独占文件；意外改到共享文件已在最终回复中说明。
- [ ] 新增接口有契约测试（`@SpringBootTest` + MockMvc）。
- [ ] 新增配置有样例 JSON 或 API 示例。
- [ ] 新增数据库表/字段同步更新 Oracle、达梦、PostgreSQL/Kingbase、LOCAL_H2_FILE DDL（`ai-dev-input/04_database/`、`zy-engine-mvp/db/{oracle,dm,postgres}/`、`src/main/resources/db/local/`）。
- [ ] 更新本任务影响到的 README 或 docs。
- [ ] 新增中文字段、备注、消息文本时按 [编码与中文备注规范.md](编码与中文备注规范.md) 保证 UTF-8 无 BOM。

### 3.2 验证命令（按需）

| 任务类型 | 必跑命令 |
|---|---|
| 任何任务 | `.\zy-engine-mvp\scripts\run-tests.ps1`、`.\zy-engine-mvp\scripts\build.ps1`、`git diff --check` |
| 涉及 DDL / 迁移 / 持久化 SQL / 落库行为 | 上面 3 条 + `.\zy-engine-mvp\scripts\detect-db-env.ps1 -BootstrapLocal` + 无 Oracle 时启动 `LOCAL_H2_FILE` 验证 + 有 Oracle 时 `run-oracle-ddl.ps1` / Oracle smoke；DM/PG/Kingbase DDL 必须同步提交 |
| 涉及前端 | 上面 3 条 + 必读 `07_前端开发规范.md` 和 `前端产品交互与视觉规范.md` + 浏览器手动验证 + （若有）Playwright E2E |
| 仅文档 | `verify-encoding.cmd` |

> 数据库验证：生产库真实验证读取本地凭据文件或环境变量，**禁止提交真实凭据**。若无法连接 Oracle/DM/PG/Kingbase，最终回复必须说明未验证原因、已完成的 `LOCAL_H2_FILE` 开发库验证和生产库补验计划。

### 3.3 提交与推送

- [ ] `git status -sb` 确认范围干净。
- [ ] `git add` **只暂存本任务相关文件**（不要 `git add -A`）。
- [ ] 提交信息：**中文短句、动词开头**，例：`增加配置包发布回滚`、`补充病历质控样例`。
- [ ] 立即 `git push` 到远端当前分支（**单任务单 push**）。
- [ ] 因权限/网络无法推送 → 最终回复必须说明原因、commit hash 和替代交接方式。

## 4. 红线（违反即必须停止）

| 红线 | 例子 |
|---|---|
| 不允许硬编码单医院逻辑 | `if (hospital.equals("ZYHOSPITAL"))` 形式禁止进入业务服务 |
| 不允许把 Neo4j / Dify 作为测试环境强依赖 | 测试不可在 Neo4j 关闭时失败 |
| 不允许外部系统传任意 SQL / Cypher 到核心引擎执行 | 必须走 Adapter / GraphProvider 受控接口 |
| 不允许配置静默覆盖已发布版本 | 同 `package_code + package_version` 内容不同必须返回错误 |
| 不允许关键医疗建议绕过医生确认 | 阻断类规则必须有 `action_mode + doctor_decision` 字段 |
| 不允许发布、回滚、同步绕过审计 | 必须写入 `ENGINE_AUDIT_LOG` |
| 不允许缺来源的医学/医保/质控配置发布 | review 必须返回 `MISSING_SOURCE` 阻断 |
| 不允许日志输出数据库密码 / Dify API Key / 患者完整隐私 | 日志必须脱敏 |
| 不允许回滚 / 覆盖其它 AI 未提交的工作 | 不确定的未提交改动先问用户 |

## 5. 多 AI 并行三句话

1. **领任务前**：先去 [02_任务台账.md](02_任务台账.md) 把编号状态改成 `IN_PROGRESS` 并写上你的代号；commit + push。
2. **开发中**：只动独占文件；需要碰共享文件 → 先在最终回复中告知用户和协调。
3. **完成后**：立刻 commit + push，把台账状态改成 `DONE`，记录核心改动一行 + commit hash。

## 9. 历史一致性约定

- 编号体系：ORG / PKG / TERM / RULE / PATH / GRAPH / DIFY / ADAPT / QC / PROV / FE / AUDIT / OPS / SEC / TEST / DOC。
- 状态机：`DRAFT → REVIEWED → PUBLISHED → SYNCED → ACTIVE → RETIRED`。
- 错误码：`SUCCESS / VALIDATION_ERROR / DATA_MISSING / CONFIG_NOT_FOUND / ENGINE_TIMEOUT / ADAPTER_TIMEOUT / DIFY_TIMEOUT / DB_ERROR / UNKNOWN_ERROR / NO_RULES_MATCHED / PENDING_MAPPING / MISSING_SOURCE`。
- 统一返回：`ApiResult { success, code, message, data, trace_id }`。
- 组织上下文 Header 全集：`X-Tenant-Id / X-Group-Code / X-Hospital-Code / X-Campus-Code / X-Site-Code / X-Department-Code / X-Org-Code`，Body 同名字段优先级最高。

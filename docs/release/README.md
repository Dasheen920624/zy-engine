# MedKernel · 发布证据

> 状态：骨架占位 · 待 E5 引擎全能力验收后正式填充
> 适用：v1.0 GA · 内部发布评审 + 客户验收

---

## 1. 文档定位

每一次正式版本发布的**证据归档**入口，对应 [产品宪法](../CONSTITUTION.md) "一个证据链" 与 [总览](../MEDKERNEL_FOUNDATION_AND_SERVICES.md) §6 实施顺序中的 E5 验收。

发布证据是后续灰度→全量→回滚的依据，必须满足：

- 可验证：测试覆盖、性能基线、安全扫描、迁移演练均可重复
- 可追溯：traceId / 审计链 / 录像归档完整
- 可签字：院方信息科、医务部、乙方 SRE 同时签字
- 可回滚：每一项发布都有对应的回滚步骤

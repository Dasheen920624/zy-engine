# 医疗智能引擎平台产品手册

适用版本：MedKernel 演示验证版  
适用对象：医院管理人员、医务/质控/医保/药学业务老师、实施工程师、信息科管理员

## 1. 产品定位

医疗智能引擎平台用于把医学指南、院内制度、医保规则、诊疗路径和院内数据适配器统一治理，并以可追溯、可发布、可审计的方式服务临床业务。

本演示版覆盖以下完整流程：

- 来源追溯：导入指南、规范、目录等来源文档，并绑定到规则、路径、配置包。
- 规则引擎：按患者上下文执行路径入径、病历质控、医保审核、医嘱安全等场景。
- 路径引擎：从候选识别、医生入径、节点任务、任务完成到节点流转形成闭环。
- 配置包治理：导入配置包、校验来源、审核、发布、导出。
- Oracle 主库验证：演示数据写入 Oracle 权威库，并通过 SQLPlus 校验关键表。

## 4. 客户演示流程

### 4.1 Provider 状态

进入“系统管理 / Provider 状态”，确认：

- Database 为 READY，Oracle 模式下 provider 显示 ORACLE。
- Graph 可处于数据库或启发式降级，不影响 DB-only 演示。
- Dify 可处于本地模板降级，不影响核心规则、路径和配置包流程。

### 4.2 演示与校验工作台

进入“演示与校验工作台”，依次运行：

- 路径入径：胸痛 + 心电图 ST 段抬高触发 AMI/STEMI 候选。
- 病历质控：检查入院记录时限、主诉、现病史等。
- 医保审核：检查限定支付药品与适应症匹配。
- 医嘱安全：检查重复抗菌药物等风险。

每次运行后重点查看：

- 命中规则数、执行耗时、traceId。
- 规则命中详情、建议动作、证据来源。
- 右侧最近验证记录，证明结果可回查。

### 4.3 配置包审核发布

进入“配置包中心”，选择 `PKG_AMI_RULE_CONFIG / 2026.05.01`：

1. 查看清单、差异和完整快照。
2. 打开 Review，确认来源审查为通过。
3. 填写审核人并提交审核。
4. 发布时输入包编码确认，填写发布原因。
5. 发布后状态变为 PUBLISHED，可导出配置包快照。

### 4.4 AMI 路径闭环

演示路径时按以下口径说明：

1. 规则引擎识别 `P_AMI_001` 具备 AMI/STEMI 候选条件。
2. 医生确认入径，生成患者路径实例。
3. 首节点为“胸痛识别”，系统生成“完成十二导联心电图”任务。
4. 完成心电图任务后，节点流转到“再灌注策略评估”。
5. 路径中每个关键节点都带 `reference_document_code`，发布时缺来源会被阻断。

## 5. Oracle 主库初始化

演示环境连接 Oracle 后，先执行 DDL：

```powershell
cd medkernel-mvp
.\scripts\run-oracle-ddl.cmd
```

启动 Oracle 模式：

```powershell
.\scripts\start-oracle.cmd
```

导入并验证演示数据：

```powershell
.\scripts\seed-oracle-demo.cmd
```

脚本行为：

- 只清理 `TENANT_DEMO / HOSPITAL_DEMO` 演示范围数据。
- 通过 API 写入组织、来源、引用、资产绑定、规则、路径、配置包和一次患者路径验证。
- 通过 SQLPlus 校验 `ORG_UNIT`、`SRC_DOCUMENT`、`SRC_CITATION`、`SRC_ASSET_BINDING`、`RE_RULE_DEF`、`CFG_CONFIG_PACKAGE`、`PE_PATHWAY_DEF`、`PE_PATIENT_INSTANCE`、`RE_RULE_EVAL_RESULT` 等关键表。

如需保留已有演示日志：

```powershell
.\scripts\seed-oracle-demo.cmd -SkipCleanup
```

## 7. 常见问题

登录失败：

- 确认使用 `zhao01 / demo123`。
- 若使用真实后端，确认安全种子数据已初始化。

Provider 显示非 Oracle：

- 确认 `MEDKERNEL_DB_URL`、`MEDKERNEL_DB_USERNAME`、`MEDKERNEL_DB_PASSWORD` 已配置。
- 确认使用 `start-oracle.cmd` 启动服务。

配置包发布被阻断：

- 检查 `source_review` 是否 blocked。
- 检查规则、路径节点是否缺 `reference_document_code`。
- 检查来源文档是否已审核且未过期。

Oracle 种子脚本失败：

- 先运行 `check-env.cmd` 检查 sqlplus、JDK 和 Oracle 变量。
- 确认后端 Oracle 模式已经启动在 `http://localhost:18081/medkernel/api`。
- 确认已经执行 `run-oracle-ddl.cmd`。

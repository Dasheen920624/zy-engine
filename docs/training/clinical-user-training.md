## 1. 培训概述

### 1.1 培训目标

通过 1 天（8 学时）的培训，使参训人员能够：

- 熟练登录 MedKernel 系统并进行界面导航
- 正确处理 CDSS 临床决策支持提醒（接受/覆盖/忽略）
- 完成患者临床路径入径操作
- 使用知识查询功能查找临床指南和药物信息
- 查看质控指标和 AI 辅助功能

### 1.2 培训安排

| 项目 | 内容 |
|------|------|
| 培训时长 | 1 天（8 学时） |
| 培训对象 | 临床医生、护士 |
| 培训方式 | 讲解演示（30%）+ 实操练习（70%） |
| 考核方式 | 实操考核（100%） |

### 1.3 前置要求

| 技能 | 要求 |
|------|------|

## 2. 上午：系统入门与 CDSS 使用（4 学时）

### 2.1 MedKernel 系统简介（10 分钟）

**MedKernel 是什么？**

MedKernel 是一个临床决策支持系统，它就像您的"智能临床助手"，在您诊疗过程中提供实时提醒和建议。

**核心功能**：

| 功能 | 说明 | 类比 |
|------|------|------|
| CDSS 临床决策支持 | 医嘱保存时自动检查，提醒潜在风险 | 像"用药安全卫士" |
| 临床路径管理 | 标准化诊疗流程，减少遗漏 | 像"诊疗导航仪" |
| 知识查询 | 查询临床指南、药物信息 | 像"口袋临床指南" |
| 质控指标 | 查看科室和个人质量指标 | 像"质量体检报告" |
| AI 辅助 | AI 推荐知识、诊断建议 | 像"智能会诊助手" |

**MedKernel 不是什么**：

- 不是替代医生决策的系统——所有提醒和建议仅供参考，最终决策权始终在医生
- 不是电子病历系统——它是嵌入在 HIS 中的辅助工具
- 不是行政监控工具——它的目的是帮助医生，不是考核医生

### 2.3 CDSS 临床决策支持（90 分钟）

#### 2.3.1 什么是 CDSS？

CDSS（Clinical Decision Support System，临床决策支持系统）是在您诊疗过程中自动检查医嘱、提醒潜在风险的智能助手。

**工作原理**：

```
医生在 HIS 中开医嘱 → 医嘱保存时自动触发 CDSS 检查 → 系统返回提醒 → 医生处理提醒
```

**CDSS 不会**：
- 自动修改或取消您的医嘱
- 阻止您做出临床判断
- 在您不知情的情况下做任何操作

#### 2.3.2 实时提醒：医嘱保存时的自动检查

当您在 HIS 中保存医嘱时，MedKernel 会自动进行以下检查：

| 检查类型 | 说明 | 示例 |
|----------|------|------|
| 药物相互作用 | 检查两种药物是否会产生不良相互作用 | 华法林 + 阿司匹林增加出血风险 |
| 过敏提醒 | 检查患者是否有相关药物过敏史 | 患者对青霉素过敏，开了阿莫西林 |
| 重复检查提醒 | 检查是否重复开具相同检查 | 3 天内已开过血常规 |
| 剂量提醒 | 检查药物剂量是否超出常规范围 | 地高辛剂量超过 0.25mg/天 |
| 禁忌症提醒 | 检查药物是否有相关禁忌症 | 孕妇开了禁用药物 |
| 医保合规提醒 | 检查医嘱是否符合医保报销规则 | 限定支付药品与适应症不匹配 |

#### 2.3.6 如何处理提醒（接受/覆盖/忽略）

**三种处理方式**：

| 操作 | 含义 | 适用场景 | 说明 |
|------|------|----------|------|
| **接受提醒** | 采纳系统建议，修改或撤回医嘱 | 提醒合理，确实存在问题 | 最安全的处理方式 |
| **覆盖提醒** | 不采纳建议，继续提交医嘱，但需填写覆盖原因 | 临床判断与系统建议不同 | 必须填写覆盖原因，系统会记录 |
| **忽略** | 暂不处理，稍后决定 | 需要进一步思考 | 提醒不会消失，下次保存时仍会触发 |

**覆盖提醒时需填写的原因**：

覆盖提醒时，系统会弹出原因填写框，您需要选择或填写覆盖原因：

| 覆盖原因类别 | 示例 |
|-------------|------|
| 临床必须 | "患者病情需要，已评估风险获益比" |
| 已知情同意 | "已与患者沟通，患者知情同意" |
| 替代方案不可行 | "替代药物不可及" |
| 其他 | 自由填写 |

> **重要提示**：所有覆盖操作都会被系统记录，用于质量追溯。这不是为了监控医生，而是为了在发生医疗纠纷时提供证据保护。

### 2.4 临床路径入径（60 分钟）

#### 2.4.1 什么是临床路径

临床路径是针对某一疾病建立的一套标准化治疗模式与治疗程序。它就像一个"诊疗导航"，告诉您在什么时间应该做什么检查、用什么药、达到什么标准可以进入下一阶段。

**临床路径的价值**：

- 减少诊疗遗漏——关键步骤不会遗漏
- 规范诊疗行为——遵循循证医学证据
- 提高医疗质量——标准化流程减少变异
- 便于质量监控——执行情况可量化评估

**MedKernel 中的临床路径**：

MedKernel 将纸质路径数字化，实现：
- 自动识别入径候选患者
- 在线完成入径操作
- 节点任务自动生成和提醒
- 变异在线记录
- 执行数据自动汇总到质控系统

#### 2.4.2 入径候选推荐

系统会根据患者的诊断和临床信息，自动推荐可能适合的临床路径。

**候选推荐逻辑**：

```
患者诊断 + 临床指标 → 规则引擎匹配 → 推荐候选路径
```

#### 2.4.3 入径操作

**入径步骤**：

1. 在路径候选推荐中，点击匹配度最高的路径旁的 **[入径]** 按钮
2. 系统显示入径条件确认页面
3. 确认患者满足入径标准，勾选确认
4. 填写入径备注（可选）
5. 点击 **[确认入径]**
6. 系统自动生成第一个阶段的待办任务

## 3. 下午：知识查询与质控（4 学时）

### 3.1 知识查询（60 分钟）

#### 3.1.1 查询临床指南

MedKernel 知识管理模块收录了各类临床指南，您可以在工作中随时查询。

**查询步骤**：

1. 在左侧菜单选择 **试点准备 → 知识源**
2. 在知识源列表中浏览已注册的指南来源
3. 或使用搜索框直接搜索关键词

**支持的指南类型**：

| 来源类型 | 说明 | 示例 |
|----------|------|------|
| 临床指南 | 国内外权威学会发布的诊疗指南 | ACC/AHA STEMI 指南、中国心衰指南 |
| 专家共识 | 专家组达成的共识意见 | 中国房颤抗凝共识 |
| 标准目录 | 国家/行业标准编码目录 | ICD-10、药品编码、手术编码 |
| 院内制度 | 医院内部制定的诊疗规范 | 院内抗菌药物使用规范 |
| 医保政策 | 医保目录和报销规则 | DRG/DIP 分组规则 |

**查询示例**：


#### 3.3.3 如何判断 AI 建议的可信度

**AI 建议可信度判断标准**：

| 判断维度 | 高可信度 | 低可信度 |
|----------|----------|----------|
| 证据来源 | 权威指南、多中心研究 | 单一研究、专家意见 |
| 置信度标注 | 高 | 低 |
| 逻辑一致性 | 与已知临床知识一致 | 与临床经验矛盾 |
| 患者匹配度 | 患者情况完全符合 | 患者情况部分符合 |

**使用 AI 建议的原则**：

1. **AI 是助手，不是决策者**——所有 AI 建议仅供参考
2. **看证据，不看结论**——关注 AI 建议背后的证据来源
3. **结合临床判断**——AI 不了解患者的全部情况
4. **有疑问就忽略**——不确定时不要依赖 AI 建议
5. **反馈帮助改进**——发现 AI 建议不准确时，通过反馈功能告知系统

**AI 建议标识说明**：

| 标识 | 含义 | 可信度 |
|------|------|--------|
| 🤖 AI 生成 | 完全由 AI 模型生成 | 需人工审核 |
| 📄 有来源引用 | AI 建议有明确的指南/文献引用 | 较高 |
| ⚠️ 低置信度 | AI 对该建议不确定 | 较低 |

### 3.4 常见问题解答（20 分钟）

#### Q1：系统提醒太多怎么办（疲劳管理）

**问题**：每天收到大量 CDSS 提醒，感觉影响工作效率。

**解决方案**：

1. **区分提醒类型**：重点关注"警告"和"阻断"级别的提醒，"提示"级别可快速处理
2. **接受合理提醒**：如果提醒确实合理，接受后就不会再重复提醒
3. **反馈不合理的提醒**：如果某个规则不合理，向质控科或信息科反馈，规则可以调整
4. **提醒分级说明**：

| 严重度 | 图标 | 含义 | 建议处理方式 |
|--------|------|------|-------------|
| 提示（INFO） | 💡 | 仅供参考 | 快速浏览后决定 |
| 警告（WARNING） | ⚠️ | 需要关注 | 认真评估后决定 |
| 阻断（BLOCK） | 🚫 | 高风险 | 必须处理才能继续 |

5. **联系质控科**：如果某类提醒频繁触发且不合理，可以请求调整规则阈值或关闭特定规则

#### Q2：提醒不正确怎么办（覆盖和反馈）

**问题**：系统提醒与临床判断不一致，提醒似乎不正确。

**解决方案**：

1. **覆盖提醒**：如果您经过临床判断认为提醒不适用，选择"覆盖"并填写原因
2. **填写详细的覆盖原因**：详细的原因有助于后续规则优化
3. **反馈给质控科**：如果同一规则多次被覆盖，向质控科反馈，规则可能需要调整
4. **不要因为嫌麻烦而接受提醒**：如果提醒确实不合理，覆盖比接受更正确

#### Q3：找不到需要的信息怎么办

**问题**：在知识查询中找不到需要的指南或药物信息。

**解决方案**：

1. **尝试不同的搜索词**：使用通用名而非商品名，或使用疾病英文名
2. **检查知识源状态**：在"试点准备 → 知识源"中查看相关来源是否已同步
3. **联系信息科**：如果确实缺少某类知识，可以请求添加新的知识来源
4. **使用 AI 推荐**：AI 知识推荐可能会发现您未想到的相关知识

#### Q4：系统响应慢怎么办

**问题**：系统操作响应缓慢，影响工作效率。

**解决方案**：

1. **检查网络连接**：确认医院内网网络正常
2. **刷新浏览器**：按 F5 刷新页面
3. **清除浏览器缓存**：在浏览器设置中清除缓存
4. **换一个浏览器**：推荐使用 Chrome 或 Edge
5. **联系信息科**：如果持续缓慢，可能是服务器问题，联系信息科排查

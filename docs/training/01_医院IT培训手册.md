# MedKernel 医院 IT 培训手册

> 版本：1.0 · 2026-05-24
> 任务：GA-DOC-03
> 受众：医院信息科 IT 人员
> 培训时长：4 小时

---

## 培训目标

完成本培训后，IT 人员能够：
1. 独立完成 MedKernel 系统的日常运维
2. 处理常见故障和用户问题
3. 执行数据备份和恢复操作
4. 管理用户账户和权限

---

## 第一模块：系统概览（30 分钟）

### 1.1 系统架构

MedKernel 采用前后端分离架构：
- **前端**：浏览器访问，SPA 单页应用
- **后端**：Spring Boot 微服务，REST API
- **数据库**：H2（开发）/ MySQL（生产）
- **可选组件**：Neo4j 图数据库、Dify/LLM AI 服务

### 1.2 网络拓扑

```
[浏览器] → HTTPS → [Nginx] → HTTP → [MedKernel:18080]
                                         ↓
                                   [数据库] [监控:18081]
```

### 1.3 关键端口

| 端口 | 服务 | 用途 |
|------|------|------|
| 443 | Nginx | HTTPS 入口 |
| 18080 | MedKernel | 业务 API |
| 18081 | Actuator | 健康检查/监控 |

## 第二模块：用户管理（45 分钟）

### 2.1 创建用户

1. 以管理员登录
2. 进入「系统管理 → 用户管理」
3. 点击「新建用户」
4. 填写：用户名、姓名、科室、角色
5. 设置初始密码（需满足复杂度要求：8位+大小写+数字+特殊字符）
6. 点击「保存」

### 2.2 角色权限

| 角色 | 权限范围 |
|------|----------|
| 系统管理员 | 全部权限 |
| 质控管理员 | 质控模块 + 规则管理 |
| 临床医生 | 路径执行 + 规则查询 |
| 审计管理员 | 审计日志 + 安全管理 |

### 2.3 密码策略

- 最少 8 位，必须包含大写字母、小写字母、数字、特殊字符
- 登录失败 5 次后账户锁定 30 分钟
- 支持管理员解锁和密码重置

### 2.4 实操练习

- 创建 3 个不同角色的用户
- 测试不同角色的权限差异
- 练习账户解锁和密码重置

## 第三模块：日常运维（60 分钟）

### 3.1 服务管理

```bash
# 查看服务状态
systemctl status medkernel

# 启动/停止/重启
systemctl start medkernel
systemctl stop medkernel
systemctl restart medkernel

# 查看日志
tail -f /opt/medkernel/logs/medkernel.log
```

### 3.2 健康检查

```bash
# 快速检查
curl http://localhost:18080/api/health

# 详细检查（含 Provider 状态）
curl http://localhost:18081/medkernel/actuator/health
```

### 3.3 数据备份

```bash
# 每日备份脚本
mysqldump -u root -p medkernel > /opt/medkernel/backup/daily_$(date +%Y%m%d).sql
```

### 3.4 监控查看

- 访问 Grafana：http://服务器IP:3000
- 默认账号：admin/admin
- 查看 MedKernel MVP 看板

### 3.5 实操练习

- 执行服务启停操作
- 查看健康检查结果
- 执行数据备份
- 查看 Grafana 监控看板

## 第四模块：故障处理（60 分钟）

### 4.1 常见故障速查

| 故障现象 | 可能原因 | 处理方法 |
|----------|----------|----------|
| 页面无法打开 | Nginx 未运行 | `systemctl restart nginx` |
| 登录失败 | 账户锁定 | 管理员解锁 |
| API 超时 | 数据库连接满 | 重启 MedKernel |
| 规则不生效 | 规则未发布 | 检查规则状态 |
| AI 功能不可用 | LLM 服务异常 | 检查 Provider 状态 |

### 4.2 故障升级流程

1. **L1 处理**（IT 人员）：按速查表处理
2. **L2 处理**（系统管理员）：查看日志、重启服务
3. **L3 处理**（厂商支持）：提供日志和故障描述

### 4.3 实操练习

- 模拟服务停止并恢复
- 模拟账户锁定并解锁
- 查看错误日志并定位问题

## 第五模块：安全管理（45 分钟）

### 5.1 密钥轮换

- 定期轮换加密密钥（建议每 90 天）
- 通过 API 触发轮换操作
- 轮换期间新旧密钥共存（24 小时宽限期）

### 5.2 审计日志

- 审计日志保留 180 天（等保三级要求）
- 审计链 SHA-256 防篡改
- 支持导出和查询

### 5.3 安全基线检查

- 定期检查密钥状态
- 验证审计链完整性
- 检查安全配置

# MedKernel 实施工程师培训手册

> 版本：1.0 · 2026-05-24
> 任务：GA-DOC-03
> 受众：实施工程师
> 培训时长：8 小时

---

## 培训目标

完成本培训后，实施工程师能够：
1. 独立完成 MedKernel 的部署和配置
2. 执行数据迁移和初始化
3. 配置规则引擎和路径引擎
4. 处理常见部署和配置问题

---

## 第一模块：部署安装（2 小时）

### 1.1 环境准备

| 组件 | 要求 |
|------|------|
| 操作系统 | CentOS 7+ / Ubuntu 18.04+ |
| JDK | 17（推荐）/ 8u151+（最低） |
| 内存 | 8 GB+ |
| 磁盘 | 50 GB+ SSD |
| 数据库 | MySQL 8.0+ / H2（开发） |

### 1.2 安装步骤

```bash
# 1. 安装 JDK
sudo apt install openjdk-17-jdk -y

# 2. 创建应用目录
sudo mkdir -p /opt/medkernel/{app,frontend,logs,backup,data}

# 3. 部署应用
cp medkernel-mvp.jar /opt/medkernel/app/
cp application.yml /opt/medkernel/app/

# 4. 配置 systemd
cp medkernel.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable medkernel

# 5. 配置 Nginx
cp medkernel-tls.conf /etc/nginx/conf.d/
nginx -t && systemctl reload nginx

# 6. 启动服务
systemctl start medkernel
```

### 1.3 配置说明

关键配置项：

```yaml
# 数据库模式
medkernel:
  database:
    mode: mysql    # local/h2/mysql
  graph:
    enabled: false  # 图数据库开关
  dify:
    enabled: false  # LLM 服务开关

# 服务端口
server:
  port: 18080

# 管理端口
management:
  server:
    port: 18081
```

### 1.4 实操练习

- 在测试环境完成完整部署
- 验证各端口可访问
- 验证健康检查端点

## 第二模块：数据迁移（1.5 小时）

### 2.1 数据库初始化

MedKernel 使用 Flyway 管理数据库版本，首次启动自动执行迁移：

```bash
# 启动时自动创建表结构
systemctl start medkernel

# 查看迁移状态
# 通过 Actuator 查看
curl http://localhost:18081/medkernel/actuator/health
```

### 2.2 初始数据导入

```bash
# 导入规则包
curl -X POST http://localhost:18080/api/rule-engine/packages/import \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@rules_package.json"

# 导入路径模板
curl -X POST http://localhost:18080/api/pathway/templates/import \
  -H "Authorization: Bearer $TOKEN" \
  -F "file=@pathways_package.json"
```

### 2.3 实操练习

- 执行数据库初始化
- 导入示例规则包
- 验证数据完整性

## 第三模块：规则引擎配置（2 小时）

### 3.1 规则创建流程

1. 确定规则类型（用药安全/医嘱合理性/医保合规）
2. 定义触发条件（WHEN）
3. 定义执行动作（THEN）
4. 设置优先级和分类
5. 保存草稿 → 验证 → 发布

### 3.2 规则验证

- 每条规则发布前必须通过验证
- 使用真实患者数据测试
- 验证命中率和误报率

### 3.3 规则包管理

- 规则可打包导入导出
- 支持跨环境迁移（测试→生产）
- 版本化管理

### 3.4 实操练习

- 创建 3 条不同类型的规则
- 执行规则验证
- 打包导出并导入

## 第四模块：路径引擎配置（1.5 小时）

### 4.1 路径模板创建

1. 定义路径基本信息（名称、编码、科室、病种）
2. 添加阶段（入院评估、诊疗执行、出院评估）
3. 在阶段中添加诊疗项目
4. 设置阶段条件（入组条件、出组条件）
5. 保存草稿 → 验证 → 发布

### 4.2 路径实例化

- 患者入组：自动或手动匹配路径
- 节点执行：按阶段推进
- 变异记录：记录偏离原因

### 4.3 实操练习

- 创建 1 个完整路径模板
- 模拟患者入组
- 记录变异

---

## 第五模块：安全配置（1 小时）

### 5.1 用户与权限

- 创建管理员账户
- 配置角色权限
- 设置组织范围

### 5.2 SSO 集成

支持 4 种 SSO 协议：
- CAS
- OIDC
- SAML
- LDAP-AD

配置步骤：
1. 在「系统管理 → SSO 配置」中添加身份源
2. 填写协议参数
3. 测试连接
4. 启用 SSO

### 5.3 密钥管理

- 初始密钥自动生成
- 建议每 90 天轮换一次
- 轮换操作通过 API 触发

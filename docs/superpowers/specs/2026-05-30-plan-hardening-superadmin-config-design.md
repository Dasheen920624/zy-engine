## 1. 决策摘要

| 取舍 | 结论 |
|---|---|
| 超级管理员定位 | **不旁路**（走 RBAC 引擎）+ 系统强制内置、启动自动授满五维 + 系统配置中心、**不可降权/删除/移出超管组** + 全程审计 + 强制 MFA |
| 配置前台化边界 | 除启动必需（DB 连接/端口/profile/迁移位置/启动密钥）外全搬前台配置中心 + 高危护栏（审计持久化等不可从 UI 关闭）|
| 统一管理界面落点 | 合规运维 D5「系统配置中心」（不突破 27 二级菜单上限）|

## 3. B 超管 + 配置前台化

### B1 · 超级管理员 — 新增 `SUPERADMIN-01`（落 D0）

- 启动强制内置种子超管（承接 BASE-11 平台首发种子身份）；系统**自动授满**五维权限（菜单/动作/数据/资产/环境）+ 系统配置中心访问。
- 账号**不可降权 / 不可删除 / 不可移出超管组**（系统级约束，非配置可改）。
- **不旁路**：仍走 RBAC 引擎鉴权，只是权限被系统配满——修正"维护全部功能依赖手配、漏配即达不到"的缺陷，且无旁路黑洞、全程可审计。
- 超管所有操作**独立高亮审计** + 强制 MFA。
- 牵动：BASE-02（RBAC 引擎须支持不可撤销满权组）、BASE-11（种子身份 → 种子超管）、INFRA-05（权限目录含超管自动授满）。

### B2 · 配置中心引擎 — 新增 `CONFIG-01`（落 D0 平台脊柱）

- DB 配置存储：key/value + 元数据（display-name/risk/owner/description，沿用 yml 现有 feature-flags 结构）+ **热生效** + 变更审计 + 可回滚 + 高危项二次确认。
- 启动引导只从 yml/env 读 **DB 连接 / 端口 / profile / 迁移位置 / 启动密钥**；其余 `medkernel.*`（功能开关、JWT TTL、Cookie 策略、备份 RPO-RTO、国产化、Provider、安全基线、日志级别）从配置存储读。
- **高危护栏**：审计持久化（audit-persistence）等关键项**不可从 UI 关闭/删除**（沿用 yml 注释"生产必须开启"口径）；高危项变更走影响提示 + 二次确认。
- 牵动：BASE-07（运行底座 feature flag 改为消费配置存储；yml 仅作启动引导/兜底）。
- 落 D0 理由：功能开关、认证策略在启动与全平台生效，属脊柱级；D5 仅提供管理界面。

### B3 · 系统配置中心界面 — 新增页面 `D5-PAGE-系统配置中心`（落 D5 合规运维）

- 前台统一管理：功能开关 / 认证策略 / 备份 / 国产化 / Provider / 安全基线 / 日志级别。
- 超管可达；高危项护栏 UI（置灰 + 说明，不可关）；改配置走影响提示 + 二次确认 + 审计 + 回滚点。
- **IA 约束**：不突破宪法 §2.2 的 27 二级菜单上限。**默认**：扩展/重命名 D5 既有"安全基线"为"安全基线与系统配置"承载统一配置，"Provider 状态"保留为只读状态展示（配置写入归系统配置中心）——二级菜单不净增。宪法 §2.2 同步登记；最终菜单标签可在实施计划微调，但"不净增二级菜单"是硬约束。

## 4. C 验收标准强化

- **新增验收铁律 #11 配置外置**：除启动必需（DB/端口/profile/迁移/密钥）外，业务与运营配置不得写死 yml；必须经配置中心管理、可审计、高危项有护栏。（backlog §0.2 + 质量基线 §1 的"10 条铁律" → 11 条）
- **新增宪法硬约束**：§1 增 #19 配置外置 + #20 内置超级管理员（强制内置、授满不可降权、不旁路、全审计、强制 MFA）。
- **超管验收**：不可降权/删除可证；超管操作全审计；超管无法从 UI 关闭审计持久化；强制 MFA 生效。
- **D5 域级验收补**：在系统配置中心改一项配置 → 热生效 + 审计留痕 + 高危项被护栏拦截 + 可回滚。
- **场景实现卡（详规 §10.1）**：页面/域任务开发前须有实现卡（关联 S 编号），列入质量基线 pre-dev gate。

## 6. 非目标 / YAGNI

- 不在第一波做模型/AI 配置（仍属第二波）。
- 不把 DB 连接/启动密钥搬前台（启动悖论 + 安全）。
- 不为系统配置中心新增客户一级菜单（IA 锁定）。
- 不实现配置的多版本工作流（配置变更走审计 + 回滚点即可，不套 7 步流灰度——配置非临床资产）。

---

## 7. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 超管满权 = 高危账号 | 不旁路（走 RBAC 可审计）+ 独立高亮审计 + 强制 MFA + 不可删除（防被顶替） |
| 配置前台化误关键能力 | 高危护栏（审计持久化/国密不可从 UI 关）+ 二次确认 + 影响提示 + 回滚点 |
| 配置存储不可用时启动 | yml 作启动引导/兜底；配置存储读失败回退安全默认 + 诚实告警 |
| 铁律从 10 改 11 牵动多文档 | 四份文档同 PR 改透；一致性 grep 核查 |

## 1. 背景与问题

旧节奏（宪法 §0 + backlog v7.0 §0.4 + 落地规划 §17/§18）是**引擎优先、自底向上**：

```
E0 文档 → E1 底座 → E2 引擎接口 → E3 引擎执行 → E4 嵌入/模型 → E5 引擎验收 → E6 业务包装（最后）
```

问题：自底向上先堆了上百天的后端引擎，长期没有"能点开、能跑、能验证"的用户界面。导致 done 无法证伪——连续 6 轮逐页核查仍暴露 done 注水，已无任何 done 可信。**这正是"之前的开发方式无法准确把握和验证实现"的根因。**

既有代码现状：前端约 47 个页面（按 advanced/clinical/compliance/quality/tenant 分域）+ 独立 `Login.tsx`，后端 30 个 Controller。即"已有但不可信"，需逐域坐实。

## 3. 推进结构

### 3.1 两波

- **第一波 · B0 真实纵向**：沿用户旅程逐域做到"无模型也真实可用、可验证"。每个域跑通域级验收后才走下一个。这是"从登录一点点落实"的主线。
- **第二波 · AI 加深**：把 AI 工厂、模型网关、首发知识资产等**跨域共享深引擎**建起来，再回灌相关域做模型增强。必须第二波，因为①铁律 #4 要求 B0 路径先于模型存在；②AI 工厂/知识资产是多域共用基础设施，不属于任何单一页面。

### 3.2 第一波 7 个域（旅程顺序＝依赖顺序）

| 序 | 域 | 含二级菜单页 | 该域 B0 主要坐实 |
|---|---|---|---|
| **D0** | 登录域（平台脊柱）| 登录 + 登入后可导航空壳 | 组织/租户上下文、五维 RBAC、API 契约、审计骨干、设计 token、菜单壳、会话/退出、真实性门禁 |
| **D1** | 工作台 | 工作台 · 演示与校验 | 角色首屏、租户生命周期面板、多维切片、健康/演示校验 |
| **D2** | 试点准备 | 实施向导·租户开通·配置包中心·路径配置·规则库·字典映射·适配器中心 | 确定性规则/路径/知识/字典引擎 + 7 步流 + 4 状态机 + 灰度/回滚（D3 的料）|
| **D3** | 临床运行 | 患者主索引·患者路径·临床提醒治理·规则校验·待办中心·通知中心·智能随访 | MPI、患者路径推进、确定性 CDSS（消费 D2 已发布规则/路径）、待办/通知、随访 |
| **D4** | 质控改进 | 驾驶舱·质控预警·医保审核·评估指标库·评估结果·AI 知识审核 | 指标/评估/整改闭环、DRG/DIP 确定性、驾驶舱下钻（AI 知识审核的"审"先做，"AI 生成"留第二波）|
| **D5** | 合规运维 | 用户管理·身份绑定·审计日志·安全基线·Provider 状态·通知设置 | 五维权限管理界面、审计查询、安全基线、Provider 状态（无连接诚实显示）|
| **D6** | 高级工具 | 来源追溯·图谱查询·AI 工作流·国产化自检·开发者控制台 | 来源追溯、图谱投影查询、国产化自检、开发者控制台（AI 工作流编排留第二波）|

### 3.3 第二波回灌

待 D0–D6 的 B0 全跑通后，按序加深到对应域页面：
X-AIK（AI 工厂）→ X-LLM（模型真实接入）→ X-KNOWGEN（首发知识资产）→ X-DOMAIN（15 领域门面）→ X-OPT（世界级+国情剩余）。

### 3.4 依赖即顺序

D2 把规则/路径/知识引擎做成 B0 真实，D3 临床才有真东西可消费；D2→D3→D4 的依赖天然顺位，正好就是菜单顺序，不用人为倒排。

## 4. 单域切片的标准模板

每个域 = 它名下所有二级菜单页都做到「每页必交」，再加一道「域级验收」。**不另立标准，复用质量基线的 10 铁律 + 核查证据模板。**

### 4.1 每页必交

- **后端**：真实 API（Record DTO + Bean Validation + ApiResult/ProblemDetail + traceId + 幂等）+ service 真实逻辑（无 mock/无写死）+ 该页所需引擎能力的 **B0 确定性实现** + 审计留痕 + 5 方言迁移。
- **前端**：配置类走 7 步流 / 资产走 4 状态机 Badge / 1 主按钮 ≤3 默认筛选 / **六态完整** / 设计 token / 中文优先 / 服务端分页。
- **权限**：该页五维 RBAC（菜单/动作/数据/资产/环境）到 13 角色矩阵粒度真实生效。
- **诚实降级**：无模型/无连接器/无图时返回诚实状态（B0/NOT_CONNECTED/NOT_SYNCED/MODEL_DISABLED），B0 主链路仍真实可用。
- **证据**：质量基线核查证据清单（代码 permalink + 测试 + T-GATE 全绿 + 文档锚点 + 审计员签字，owner ≠ reviewer）。

### 4.2 域级验收

该域所有页之上再加一道端到端：登录该域涉及角色 → 逐页打开无 Hook 错误/null/404 → 跑通该域 B0 主链路 E2E。

> 例 D3：建患者 → 入径 → 节点推进 → 确定性提醒 → 采纳/拒绝带原因 → 待办闭环。

**域级验收过，才算这个域落实，才走下一域。**

## 5. 149 项映射原则

块级搬法（写 backlog 时逐条落位）：

| 旧块 | 搬到 |
|---|---|
| E1 基础底座(BASE-01~11) + X-SYS 基础 + X-INFRA 门禁/权限/会话(INFRA-01/02/03/04/05/08) | **D0 平台脊柱** |
| E2/E3 字典·规则·路径·知识·包引擎的 B0 确定性部分 | **D2**（第一个用到）|
| E3 CDSS-01 · 随访 FOLLOW-01 | **D3**；评估 EVAL-01 → **D4** |
| E6 业务包装 SVC-PILOT/CLINICAL/QUALITY/COMPLIANCE/INTEGRATION | **溶解进 D2/D3/D4/D5**（E6 不再是最后阶段）|
| E5 验收 QA-01~08 + INFRA-06 全 36 页真实化 + INFRA-07 可打开 E2E | **拆成每域验收子集 + 最后一道 GA 总验收** |
| 第二波：X-AIK + X-LLM + X-KNOWGEN + X-DOMAIN + 各引擎模型增强 + X-OPT 剩余 | **第二波回灌 D2/D3/D4/D6** |
| E0 文档清场 | **当前这次重排动作本身** |

一句话：**基础进 D0；引擎按"谁第一个用"进 D2/D3/D4 的 B0；业务包装溶进各域；验收拆成域级+GA；模型/AI/知识资产/领域门面整体后移第二波。**

> 注：X-MED（规则算子 MED-C2、旧版隔离 MED-C3）→ D2/D3；X-OPT 中 OPT-04 临床安全红线 → D3（B0 必备），OPT-01 FHIR / OPT-02 CDS Hooks → D2/D3 对接，其余随域或第二波。

## 7. 执行顺序

1. **先改四文档**（宪法 §0 / 落地规划 / backlog / 质量基线）到口径完全一致，无遗留矛盾。
2. **再从 D0 登录域开工**：先坐实平台脊柱，登录界面是脊柱上第一个可验证界面（按角色登入、菜单按 RBAC 正确呈现、各页路由可打开到六态空态、退出/会话过期可用）。D0 承接既有 [user-auth-dual-mode 设计](2026-05-29-user-auth-dual-mode-design.md)，核查整改而非重做。
3. D0 域级验收过 → D1 → D2 → … 逐域推进。

## 8. 非目标 / YAGNI

- 不在第一波建任何模型/AI 工厂/知识自动生成能力（铁律 #4：B0 先行）。
- 不重写既有可用代码——逐域核查整改为主，缺口才补建，假闭环必拆。
- 不保留旧 E0~E6 节奏的任何并行口径（宪法/落地规划/backlog/质量基线四份改透后旧节奏即废止）。
- 不在本设计内逐条列 149 项落位——那是写 backlog 的动作；本设计只锁定映射原则。

## 9. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 某页拉入当下建不起来的深引擎 | B0 优先：先做确定性/人工路径，深引擎与模型增强后移第二波 |
| 域级验收流于形式（又一轮 done 注水）| 验收必须是"真登录真点真跑"的 E2E + T-GATE 全绿 + owner≠reviewer 签字 |
| 四文档改不同步留矛盾 | 一次性改透 + 文档与代码同 PR（宪法 §1.#12）|
| 第二波（AI/知识资产）被无限推迟 | 第二波在 backlog 中保留为明确任务块与 GA 门禁，不是"可选" |

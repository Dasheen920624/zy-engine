# 文档体系重构：自包含施工卡 · 设计规格

> 日期：2026-05-30 · 状态：待用户复核
> 触发：用户要求"文档多而杂导致每个 AI 开发都得读很多文档、输入 token 太大且分散，功能不按规划落实、上版各种缺失事实的实现；要求严格执行产品具体需求、一丝一毫不偏差；以产品架构师/产品体验师/系统架构师等多方位视角全梳理重构，让 AI 开发者读最少资料、获取最全实现要求，保证产品真正可用"。
> 性质：设计/计划证据，不作为并行事实源（宪法 §9.1）；正式口径以重构后的权威文档为准。
> 关联：[超管+配置前台化设计](2026-05-30-plan-hardening-superadmin-config-design.md) · [业务域纵向推进设计](2026-05-30-domain-vertical-delivery-design.md) · [宪法](../../CONSTITUTION.md) · [详规](../../MEDKERNEL_BUSINESS_SCENARIO_DETAIL_SPEC.md) · [落地规划](../../MEDKERNEL_IMPLEMENTATION_LANDING_PLAN.md) · [backlog](../../backlog.md) · [质量基线](../../audit/质量基线.md) · [AGENTS.md](../../../AGENTS.md) · [全系统核查总报告](../../audit/2026-05-29-全系统一次性核查与改造总报告.md)

## 1. 决策摘要

| 取舍 | 结论 |
|---|---|
| 重构力度 | **① 自包含施工卡**：把巨物拆成一卡一交付物，卡拥有事实、巨物退役 |
| 卡粒度 | **(A) 卡 = backlog 可交付物**（~175 张）+ **7 份域简报** + **1 个极简核心** |
| 权威模型 | **① 卡为唯一源**：每个事实只有一个家；详规仅留「场景→卡」索引；保真靠"旧锚点→新卡覆盖矩阵"而非留巨物兜底 |
| 不偏差强制 | **① 卡 = 带验收的合同**：卡内含原子验收 + 完工门规则（举不全不得 done）；门禁代码实现为下游被引用任务，不在本 spec 写 |
| 完整交付视角盘 | **11 个视角**焊进结构；每张卡逐条"给要求 or 显式 N/A+理由"，**任一留白即不得 done** |

本 spec 是**信息架构（IA）重构**的设计：把"AI 为一个开发单元要读的东西"从"7 文档阅读链 ≈ 10 万 token"压成"核心 + 1 域简报 + 1 卡 ≈ 几 K token"，且该卡被 11 视角逐一拷问过、自带可举证验收。

## 2. 问题与根因

- **阅读链过载**：[AGENTS.md](../../../AGENTS.md) 规定的权威阅读顺序为 7 份文档（README→docs/README→宪法→体验规范→落地规划→详规→backlog），合计 ≈ 36 万字 / 10 万+ token；其中两个巨物——[详规 174K/2214 行](../../MEDKERNEL_BUSINESS_SCENARIO_DETAIL_SPEC.md)、[落地规划 91K/1241 行](../../MEDKERNEL_IMPLEMENTATION_LANDING_PLAN.md)。
- **漂移机制**：需求分散在 6 个大文档，AI 一次端不住 → 要么只读一部分（漏要求），要么猜/造（缺失事实的实现）。文档散 = 漂移有缝可钻。
- **崩点实锤**（[全系统核查总报告](../../audit/2026-05-29-全系统一次性核查与改造总报告.md)）：~25 页系统性绕过 `no-page-mock`（eslint-disable + 函数包装规避 AST）、`SHA-256-MOCK-HASH`、Mock KPI、假审计抽屉、字典 LCS 误配红线、AI 工厂从未建。这些崩点恰好落在原 3 视角覆盖不到的维度上。
- **解药种子已存在**：详规 §10.1 已有「场景实现卡模板」、宪法 #15 已要求每领域有「实现卡」——但实现卡当前不是 AI 实际消费的单元，也未被强制。今天 AI 读的是巨物，不是卡。本重构把"卡"扶正为唯一消费单元与合同。

## 3. 四层模型与阅读流

一恒读 + 两单读 + 一索引，每个事实只有一个家：

```
① 极简核心 CORE（恒读 · 目标 ≤约2.5万字）
   按 11 视角分章的全局不变量：硬约束/菜单IA锁/状态机/token/临床红线/
   知识治理/安全合规/集团继承/集成边界/AI治理/运维国产化/铁律门禁/语言
        ▲ 卡只"点核心编号 + 写本卡落点"，绝不抄约束原文
② 域简报 D{0..6}/_brief（读 1 份 · ~100 行）
   本域角色/共享数据模型/依赖引擎/涉及租户层级+外部系统/最烫不变量/域级验收
        ▲ 同域 N 卡共享上下文上提至此，不在卡间复制
③ 施工卡 D{n}/<TASK-ID>（读 1 张 · ~120–250 行）= 合同
   身份/目标/功能要求/接口或页面契约/数据迁移/11视角清单/适用不变量/验收/证据

场景→卡 索引 cards/_index：S0–S40 → 哪张卡拥有其要求（找卡用，不通读）
```

**AI 建任意一个任务只读：核心 + 1 域简报 + 1 卡 ≈ 800–1200 行 / 几 K token**（页面卡另读共享体验契约）。省的不是核心瘦了多少，而是**永远不再通读那两个巨物**。

三视角各归其位（且被扩为 11，见 §4）：产品架构=四层+单一源（功能不再散落，领的卡就是合同）；产品体验=全局体验不变量进核心、每页页面契约进卡（治"页面乱样式乱"——每页样式只能从核心 token + 卡的页面契约来，无处可乱编）；系统架构=接口契约/状态机/验收绑定都在卡内。

## 4. 11 视角完整交付盘

视角集合 ≈ 卡必须覆盖的需求维度集合；少一个视角 = 一整类需求被静默漏掉。对集团化医疗中枢，确定 11 个：

| # | 视角 | 范围要点 |
|---|---|---|
| 1 | 产品架构 | IA / 单一源 / 功能边界 / 菜单锁 |
| 2 | 产品体验 | 页面 / 六态 / 低打扰 / 可信解释 / 无障碍（老年模式） |
| 3 | 系统与数据架构 | 引擎边界 / API 契约 / 状态机 / 数据流 / 性能与规模（10万级·并发幂等·P95） / 投影 |
| 4 | **临床医疗安全** ★ | 医师确认 / AI 标识 / 红线禁忌 / DDI / 危急值 / 剂量 / 高危双签 / 旧版隔离 / 临床知情同意 |
| 5 | **知识与数据治理** ★ | 唯一权威知识 / 版本去重 / 字典**语义**映射（反 LCS 红线） / 来源锚点+hash / 证据分级 / AI 工厂"只产候选不产事实" |
| 6 | **安全合规与监管** ★ | 等保 2.0 三级 / 国密 SM2-3-4 / 个保法 / 数据出境 / 脱敏+字段加密 / 审计链 / 互联互通与病历评级 / 伦理与（数据）知情同意 |
| 7 | **集团化与多租户治理** ★ | 平台→集团→医院→院区→社区→科室→专病 继承覆盖 / 租户隔离 / 6 阶段生命周期 / 灰度范围 |
| 8 | **集成与互操作** | HIS/EMR/LIS/PACS/医保/区域 适配 / FHIR / CDS Hooks / 字段映射 / 断连降级 |
| 9 | **运维 / SRE / 国产化交付** | 内外网双形态 / 国产化栈（麒麟·达梦·人大金仓·Ollama） / 5 方言迁移 / 监控 / 备份 RPO-RTO / 降级链 / 离线许可 / SLA / 灾备 |
| 10 | **质量与真实性审计** | T-GATE / no-page-mock / 11 铁律 / A1–A9 剧本 / 反假闭环（即"卡=合同"的验收维） |
| 11 | **AI / 模型治理与可降级** | B0 先于模型 / 模型能力网关 / 只产候选 / 路由与脱敏外调 / 提示词与版本治理 / 幻觉拦截 / 诚实降级 |

用户三例的显式归属（不另立视角）：**科研伦理** → #6「伦理与知情同意」（对齐 RWD-01 伦理授权）；**患者知情同意** → 临床进 #4、数据隐私进 #6（个保法）；**财务/计费** → 医保 DRG/DIP/价值 ROI 已在 #5 与质控域；平台级 SaaS 计费当前不设独立维度（YAGNI，需要时再补）。

划出范围外（不进卡维度，各有归处）：i18n（中文优先硬约束）、法务合同/SLA 文本（`legal/`）、培训/变更管理（`handbook/training/`）、灾备（并入 #9）。

**机制化保证**：11 视角全局不变量按视角归入核心（§5 ①）；每张卡带【视角清单】逐条 fill-or-N/A（§5 ③）；域级共享维度进域简报。读一张卡 = 被 11 视角逐一拷问过，漏一类需求在门口被挡。

## 5. 三个模板

### ① 核心 CORE（恒读，按视角分章，目标 ≤约2.5万字）

§0 定位与命名 · §1 硬约束 #1–#20（含 #19 配置外置 / #20 内置超管） · §2 菜单 IA 锁（5+1 一级 / 27+5 二级全树） · §3 4 状态机 · §4 7 步流 · §5 设计 token · §6 临床安全红线 · §7 知识与数据治理不变量 · §8 安全合规基线 · §9 集团化继承模型 · §10 集成互操作边界 · §11 AI/模型治理 · §12 运维国产化 · §13 11 验收铁律 + 门禁 + 真实性 · §14 中文优先与语言 · §15 术语表 / 13 角色矩阵指针。

= 宪法瘦身 + 体验规范全局部分 + 11 铁律 + 详规/落地规划的横切原则，按 11 视角各归其位、去重去散文。

### ② 域简报 `D{n}/_brief`（读 1 份，~100 行）

域目标（1 句） · 登入/使用角色（13 角色子集） · 共享数据模型/实体 · 依赖的引擎/API/上游卡 · 本域涉及的租户层级 + 外部系统 · 本域最烫的不变量（点核心编号） · 域级验收（D-验收）。

### ③ 施工卡 `D{n}/<TASK-ID>`（读 1 张，~120–250 行）= 合同

```
【身份】卡ID(=backlog ID) · 域 · 关联 S-x · 依赖卡 · 工作量 · owner
【目标】一句话交付物
【功能要求】本单元独有规格（从详规对应 S-x 搬入，单一归属）—— 原子可测条目
【接口契约】(引擎/API 卡) endpoints/DTO/校验/状态机选型/幂等/错误码
  或【页面契约】(页面卡) 路由元数据/PageShell/六态/1主按钮·≤3筛选/默认视图/
                        五维 RBAC 点位/样式 token 引用（页面卡另读 EXPERIENCE_CONTRACT）
【数据与迁移】表族/索引/5 方言迁移/审计字段
【视角清单 ✓填 / N·A+理由】← 完整性闸门，11 视角逐条不留白
  1产品架构 2体验 3系统数据架构 4临床安全 5知识数据治理 6安全合规
  7集团多租户 8集成互操作 9运维国产化 10质量真实性 11AI模型治理
【适用不变量】勾核心编号 + 本卡落点（点编号，不抄原文）
【验收+验证】逐条 done 标准（可举证） + 关联 A1–A9 + T-GATE + B0 验收
【完工证据】代码 permalink / 测试 / 迁移 / 截图 / traceId / 审计员签字（owner≠reviewer）
```

**卡内"细"的三条硬规则**（堵"卡内仍可漂移"）：

1. **原子可测条目**：【功能要求】【验收】每条 = 一个可验证断言（输入→期望输出/行为），能直接写成一条测试；条目与验收一一对应；**禁散文式要求**。
2. **大卡内有序工序**：>~5d 或"后端+多层前端"的卡在卡内声明有序 PR 拆分（如 PR1 迁移+门禁 → PR2 后端契约 → PR3 前端页面），每序自带验收；**仍是一张卡一份合同，不拆成多文件**。
3. **执行最细步骤由 writing-plans 在建该卡时产出**（TDD 原子步），不在本结构里预列。

**完工门规则**（不偏差的强制点）：任务 `pending → done` 必须逐条举证卡上的【验收】、【视角清单】无留白、卡↔代码↔测试三向绑定；**举不全即不得 done**。门禁的代码实现（CI / T-GATE 扩展 / 覆盖检查脚本 / card↔code 追溯）是**被本 spec 引用的下游任务**，不在本 spec 写代码。

## 7. 迁移与排序（保真不靠留巨物）

### P0 · 骨架与核心（一次性先行）

- 写**核心 CONSTITUTION**（按 11 视角重组，吸收体验全局 + 铁律 + 横切；**超管/配置 spec 的 #19 配置外置 / #20 内置超管 / 铁律 #11 在此原生写入**）。
- 写 `EXPERIENCE_CONTRACT`、卡模板 `_template`、域简报模板 `_brief-template`、`glossary` 起底。
- 建 `cards/_index` 骨架 + **覆盖矩阵**（旧文档锚点全清单：详规 S0–S40 + §章节、落地规划章节、体验规范、质量基线条目）。
- 改 `AGENTS.md` 阅读链。
- 巨物暂不删；核心即刻切为权威，新旧并存。

### P1–P7 · 逐域搬迁（D0→D6，与纵向推进同节奏，每域一个 PR）

1. 写 `D{n}/_brief`。
2. 该域 backlog 行逐个写成卡：从详规（对应 S-x）/落地规划/体验/质量基线把该单元要求**搬入卡**（单一归属），填 11 视角清单 + 原子验收。
3. 更新覆盖矩阵：本域涉及旧锚点 **100% 标"已迁入哪张卡"**。
4. **域级评审门**：owner≠reviewer 对照矩阵核"零要求丢失" + 卡内 11 视角零留白。
5. backlog 行链到卡。
6. **D0 先行**（含平台脊柱 + 超管 + 配置 + 最常读任务，先证明模型再铺开）。

### P8 · 巨物退役

## 9. 完整性与不偏差保证（如何根治"缺失事实"）

四道闸叠加，每道都在"门口"而非"事后审计"拦截：

1. **单一归属**：每个事实只有一个家（核心/域简报/卡），消除多源漂移与"读哪份算数"的歧义。
2. **11 视角清单**：每卡逐条 fill-or-N/A，留白即不得 done——漏一类需求在门口被挡。
3. **原子可测验收**：每条要求 = 一条可写成测试的断言，卡↔代码↔测试三向绑定。
4. **完工门**：举不全卡上验收即不得 `done`；门禁代码实现为下游任务。

读最少（核心+1简报+1卡）与拿最全（被 11 视角拷问 + 原子验收）在同一张卡上同时成立。

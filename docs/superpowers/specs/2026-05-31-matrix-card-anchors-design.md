## 1. 决策摘要

| 取舍 | 结论 |
|---|---|
| 口径 | **A 矩阵内·场景行拆到卡级**（而非 B 每卡回填来源 / C 两者都做）|
| 靶子 | 仅 **§2 全业务场景目录 + §3 场景详细设计的 Sx 行**——它们当前"一个场景塞一袋卡"；§1/§4/§5/§7/§8/§10 等行已是 §x.y 卡级精度，不动 |
| 产物位置 | 只改 `docs/cards/_coverage-matrix.md` 一个文件；**不动 113 张卡文件**（那是更重的 B/C，留后续可选）|
| 范围 | 已迁的 **18 个场景**：S0–S16 + S23（S17–S40 是 wave2 待迁，不碰）|

**一句话**：把场景行从"场景 → 一袋卡"细化为"场景子责任 → 单卡/小卡组"，让退役门 P8（零丢失）可在**卡级**而非场景级核验。

## 2. 现状与靶子

矩阵每行 = 一个旧锚点 → 承接它的卡。§3 场景行当前形如：

```
L554 S8 临床嵌入运行 → D3 API-02/API-07/API-11/CDSS-01/EMBED-01/OPT-02/OPT-04 + SVC-CLINICAL-01/02/03 + 6 页
```

问题：一格混装 ~10 张卡，**无法回答"哪张卡承接 S8 的哪段责任"**，P8 退役门只能在场景粒度核验、留有缝隙。

详规每个 `### Sx` 是一张紧凑表（目标/主流程/接口归类/验收 等行），其下无更深 §-子锚点——故"细化"＝按这些**责任行**把卡拆开归属。

## 3. 改法

每个已迁场景行 → 拆成多行，逐卡标注其承接的场景子责任。锚点列写 `Sx · <子责任>`，迁入列写承接的单卡/小卡组。以 S8 为例：

| 旧锚点（拆后） | 迁入 |
|---|---|
| `S8 · 嵌入方式（iframe/SDK/WebComponent/纯API）+ launch token` | `EMBED-01` |
| `S8 · 触发/推荐卡片接口（triggers/cards）` | `API-07` + `CDSS-01` |
| `S8 · 反馈回传（采纳/不采纳/关闭/稍后）` | `API-07` + `REMIND-01` |
| `S8 · 引擎超时降级（不阻断开医嘱 + 降级日志）` | `OPT-02` / `OPT-04` |
| `S8 · 临床运行服务包编排` | `SVC-CLINICAL-01/02/03` |

**执行铁律**：逐卡归属**核对每张卡的 FR/AC** 再落（铁律"核现状别信单次 read"），不照抄设计稿此刻的示例猜测。§2 目录行对齐指向细化后的 §3。

## 4. 范围与非目标

- **范围**：S0–S16 + S23 共 18 场景，§3 详细行为主拆、§2 目录行对齐。
- **非目标**：不动 113 张卡文件；不碰 S17–S40 等 wave2 待迁锚点；不重排已精确的 §1/§4/§5/§7/§8/§10 行；不改进度统计口径以外的矩阵正文。

## 5. 节奏与验收

1. 先拆 **S0 + S8 两个样板**贴用户过目定格式。
2. 用户点头后一气拆完 18 个场景（§3 + §2 对齐）。
3. 更新矩阵 `## 进度` 段（场景锚点已迁数随拆分刷新口径说明）。
4. **整文件一个 PR**（基于含 #165 的最新 main）。

**验收**：① 18 场景每个的卡袋已逐卡拆为子责任行；② 每条迁入卡 ID 真实存在且其 FR/AC 确实承接该子责任；③ 不引入死链、不动非靶子行；④ `_HANDOFF` 线1 #1 标记完成。

## 2. 重大规范违规（增量 [NEW]）

### 2.1 [NEW] Java 包目录命名违规 — 必修

```
medkernel-mvp/src/main/java/com/medkernel/data-governance/
   ├── controller/DataGovernanceController.java   ← package com.medkernel.datagovernance.controller;
   ├── entity/*.java                              ← package com.medkernel.datagovernance.entity;
   └── repository/*.java                          ← package com.medkernel.datagovernance.repository;
```

**目录名 `data-governance` 含连字符**——违反 Java 语言规范（包名只能是合法 Java 标识符，禁止连字符）。23 个文件均把目录写成 `data-governance/` 但 package 声明为 `datagovernance`，路径与包名不一致，javac 在严格模式下会按照"file in wrong directory"拒绝。

同时违反 06_后端开发规范 §3：

> 包名：全小写（`com.medkernel.ruleengine`）

**修复：** 把 `data-governance/` 整体重命名为 `datagovernance/`（git mv 23 文件）。归 DATA-001 owner（或 REFIT-002 接手）。

## 3. 冲突识别

### 3.1 文档权威性声明不一致

**问题：**
- `01_产品事实源.md` 声明自己是"产品权威"
- `05_架构总图.md` 声明"架构边界冲突 → 本文为准"
- `06_后端开发规范.md` 声明"审查代码也以本文为准"

**冲突：**
当多个文档都声称自己是权威时，AI 不知道该听谁的。

**建议：**
在 `docs/README.md` 中明确权威链：
```
产品决策 → 01_产品事实源.md
架构决策 → 05_架构总图.md
后端规范 → 06_后端开发规范.md
前端规范 → 07_前端开发规范.md
任务状态 → 02_任务台账.md
```

---

### 3.2 任务编号体系混乱

**问题：**
- V2 体系使用 `PR-V2-XX` 编号
- 历史体系使用 `FE-XXX`、`RULE-XXX`、`SEC-XXX` 等编号
- `02_任务台账.md` 同时存在两种编号

**冲突：**
AI 可能混淆任务编号，导致认领错误任务。

**建议：**
在 `02_任务台账.md` 开头明确：
> V2 任务使用 `PR-V2-XX` 编号，历史任务使用 `FE-XXX` 等编号。两者互不兼容。

---

### 3.3 数据库生产/开发库定义不一致

**问题：**
- `数据库Provider与离线AI开发约定.md` 明确区分生产库（Oracle）和开发库（LOCAL_H2_FILE）
- `06_后端开发规范.md` §8 提到 DDL 但未明确区分生产/开发
- `08_国产化兼容性规约.md` 列出多种数据库但未说明优先级

**冲突：**
AI 可能不清楚何时用 Oracle，何时用 H2。

**建议：**
在 `06_后端开发规范.md` 开头引用：
> 数据库使用规则详见 [数据库Provider与离线AI开发约定.md](数据库Provider与离线AI开发约定.md)

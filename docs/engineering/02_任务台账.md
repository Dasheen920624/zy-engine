## 0. 用途

本文是 **项目内唯一权威任务台账**。所有任务的当前状态、负责人/AI、独占文件范围、依赖、最低能力等级、归属泳道都以此为准。

> **v1.0 GA 覆盖说明（2026-05-23）**：`v0.3-final` 已发布；旧 `PR-V2-*`、`PR-V3-*`、`PR-FINAL-*` 任务只保留历史记录。下一版本唯一目标是 **v1.0 GA / tag `v1.0.0`**，新任务从 [`../AI_TEAM_PR_BACKLOG_V1.0_GA.md`](../AI_TEAM_PR_BACKLOG_V1.0_GA.md) 领取并登记到本台账。产品和页面必须遵守 [`../PRODUCT_SIMPLIFICATION_V1_GA.md`](../PRODUCT_SIMPLIFICATION_V1_GA.md)：左侧 SideMenu 保持主菜单形态，所有功能按“最精简化，功能最全，选择最易用”收口。

> ⚠️ **全局路径变更说明（2026-05-18）：** 文档体系已重组，所有 `medkernel-mvp/docs/xxx.md` 路径自动等价于：
> - `medkernel-mvp/docs/00_*.md / 02_*.md / 05-09_*.md / 数据库*.md / 编码*.md / 用户体系*.md / AI医疗知识*.md / 产品功能*.md / smoke-plan-*.md / api-examples.http` → `docs/engineering/` 同名文件
> - `medkernel-mvp/docs/产品设计V2/*.md` → `docs/` 同名文件（金本位 5 份）
> - `medkernel-mvp/docs/{其它历史长文}` → 已物理删除，禁止作为实施依据；按 `docs/README.md` 的替代表使用当前 V2 金本位。
>
> 已 DONE 任务的独占文件列表中的旧路径不再回写，AI 接手新任务时按本说明自行换算。

任何 AI 领任务前必须做的事：

1. 在台账中找到任务编号（如未存在 → 先在"新增任务申报区"登记并 push）。
2. 把状态从 `TODO` 改成 `IN_PROGRESS`，写上你的代号 + 开始时间。
3. commit + push 台账更新（**这一次 push 是"占领"**）。
4. 完成后状态改 `DONE`，写完成时间 + commit hash + 一行核心改动。
5. commit + push。

> 状态值：`TODO / IN_PROGRESS / BLOCKED / DONE / DEPRECATED`。
> 任何同一编号同时存在多个 `IN_PROGRESS` 视为冲突，必须立即协调。

# 编码与中文备注规范

为避免文档、样例、数据库备注在 Windows、Oracle、达梦、Maven、JDK 1.8 环境之间出现乱码，项目统一采用以下规则。

## 1. 文件编码

所有可读文件统一保存为 **UTF-8**：

- Markdown：`.md`
- JSON：`.json`
- YAML：`.yml`、`.yaml`
- SQL：`.sql`
- Java：`.java`
- HTML：`.html`
- HTTP 示例：`.http`

仓库根目录已增加 `.editorconfig`，支持的编辑器会自动按 UTF-8 保存。

## 2. Windows PowerShell 读取中文文件

Windows PowerShell 5.1 对无 BOM 的 UTF-8 文件不够友好，读取中文文件时必须显式指定编码：

```powershell
Get-Content .\README.md -Encoding UTF8
Get-Content .\ai-dev-input\06_samples\sample_patient_context_ami.json -Encoding UTF8
```

建议研发环境优先使用 PowerShell 7 或支持 UTF-8 的 IDE。

## 3. Maven 与 JDK 1.8

`pom.xml` 已设置：

```xml
<project.build.sourceEncoding>UTF-8</project.build.sourceEncoding>
<maven.compiler.source>1.8</maven.compiler.source>
<maven.compiler.target>1.8</maven.compiler.target>
```

运行服务时建议增加：

```powershell
java -Dfile.encoding=UTF-8 -jar target/zy-engine-mvp-0.1.0-SNAPSHOT.jar
```

## 4. Oracle 中文备注

Oracle 表备注和字段备注有两种脚本：

- `zyengine_core_ddl_with_comments.sql`：中文可读的完整建表脚本，方便评审和维护。
- `zyengine_comments_unistr.sql`：使用 `UNISTR` 的中文备注脚本，适合实际执行，避免客户端字符集导致备注乱码。

推荐执行 `scripts/run-oracle-ddl.ps1`。该脚本会设置 SQLPlus 字符集环境，并在建表后执行 `zyengine_comments_unistr.sql` 覆盖写入中文备注。

## 5. 数据库连接客户端

使用 SQLPlus 时建议设置：

```powershell
$env:NLS_LANG='AMERICAN_AMERICA.AL32UTF8'
```

如果查询结果在当前终端仍显示异常，优先检查终端字体和代码页；数据库内备注可通过 Java/JDBC 或支持 UTF-8 的数据库工具查看。

## 6. 校验方式

执行以下脚本检查常见乱码特征，并校验 JSON 样例是否可解析。推荐使用 `.cmd` 入口，它会自动优先调用 PowerShell 7：

```powershell
.\scripts\verify-encoding.cmd
```

校验重点：

- 是否出现替换字符或典型 UTF-8/GBK 错读文本。
- `ai-dev-input` 下 JSON 是否能按 UTF-8 正常解析。
- 关键文档是否存在中文内容。

## 7. 编写建议

- 面向院方评审、产品设计、实施配置的文档使用中文。
- 代码字段名、接口字段名、枚举值保持英文，保证系统集成稳定。
- JSON 样例中的 `name`、`description`、`message_template`、`note` 等可读字段使用中文。
- 数据库表名、字段名使用英文，表备注和字段备注使用中文。

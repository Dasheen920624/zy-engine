# 07 前端开发规范

## 0. 用途

本文是 `frontend/` 前端工程的 **强制规范**。任何 AI 写前端代码前只需先读本文；需要更深的产品设计背景时，再按需查 [前端产品交互与视觉规范.md](前端产品交互与视觉规范.md)。

与 [前端配置平台规划与开发验证.md](前端配置平台规划与开发验证.md)（业务规划）配套：
- 前端规划文档 = 要做什么、为什么、什么页面
- 本文规范 = 怎么写、怎么命名、怎么测、不允许什么

## 0.1 前端体验硬规则摘要

- 首屏必须是可操作工作台、列表、配置器、校验台或看板，不做营销落地页。
- 视觉风格贴近医院业务系统：白色工作区、清晰浅色菜单、克制信息密度；不做装饰性大渐变、大 hero、漂浮卡片。
- 所有核心操作必须接真实 API 或明确 MSW 契约，dry-run 也必须保留 traceId、组织上下文、错误码和审计口径。
- 前端只做体验控制，权限、发布、来源阻断、审计必须由后端校验。
- 每个页面必须有加载、空态、错误、无权限、处理中、成功反馈；表单必须有字段级错误。
- 涉及医学、医保、质控、规则命中、AI 解释时，必须显示来源、版本、审核状态和医生/人工确认入口。
- 变更后至少运行 `npm test` / `npm run build`；关键流程补 Playwright 或手动浏览器截图验证。

**强约束**：

- `pages/` 下的页面 **不允许** 互相 import（页面间通信通过 URL / store / queryClient）。
- `pages/` 下的组件 **不允许** import 兄弟页面的 `components/`，需要复用 → 移到 `src/components/`。
- `utils/` 内的函数 **不允许** import React / antd（保持纯净，便于复用与测试）。
- `api/` 内的函数 **不允许** 直接渲染 UI 或读 store；只关心数据请求。

## 3. 命名约定

| 类型 | 命名 | 示例 |
|---|---|---|
| React 组件 | PascalCase | `SourceCard.tsx`, `OrgContextBadge.tsx` |
| Hook | `useXxx` camelCase | `useOrgContext.ts`, `useRuleEvaluation.ts` |
| 工具函数 | camelCase 文件 + 同名 default/具名 | `format.ts`, `traceId.ts` |
| 类型定义 | PascalCase | `OrgContext`, `RuleHit`, `ApiError` |
| 枚举 / 常量 | UPPER_SNAKE | `SEVERITY_LEVELS`, `ACTION_MODES` |
| 路由路径 | kebab-case | `/demo-validation`, `/config-packages` |
| 文件后缀 | `.tsx` 含 JSX；`.ts` 无 JSX；`.test.tsx` 测试 | — |

## 4. TypeScript 约束

### 4.1 strict 全开

`tsconfig.app.json` 已开 `strict: true`、`noUnusedLocals` 等。**禁止**：

- `any`（除非显式注释 `// eslint-disable-next-line ... 原因：xxx`）
- 隐式 `any`（参数不写类型）
- `as` 强转跳过类型检查（除非 unknown → narrow）
- `// @ts-ignore`（极端情况用 `@ts-expect-error` 并注释原因）

### 4.2 后端契约类型

所有后端接口 DTO 必须在 `src/api/*` 同步类型定义，与后端 JSON snake_case 一致：

```ts
export interface RuleHit {
  rule_code: string;
  rule_name: string;
  severity: "INFO" | "LOW" | "MEDIUM" | "HIGH" | "CRITICAL";
  action_mode: "REMIND" | "STRONG_REMIND" | "BLOCK" | "REQUIRE_CONFIRM" | "RECORD_ONLY";
  message: string;
  actions: Array<{ type: string; [k: string]: unknown }>;
  evidence: Record<string, unknown>;
  reference_title?: string;
  reference_section?: string;
  reviewed_by?: string;
  trace_id: string;
}
```

类型字段命名跟随后端（snake_case），不做 camelCase 转换 —— 减少心智负担。

## 5. 状态管理

### 5.1 三层状态

| 层 | 选型 | 用法 |
|---|---|---|
| 服务端缓存 | TanStack Query | 所有 GET / POST 列表 / 详情 |
| 全局轻状态 | module-level store + `useSyncExternalStore` | 组织上下文、用户身份（FE-002 已示范） |
| 组件本地 | `useState` / `useReducer` | 表单、tab 切换、抽屉开关 |

**禁止**引入 Redux / MobX，除非有强需求评审通过。

### 5.2 TanStack Query 约定

```ts
useQuery({
  queryKey: ["rule-engine", "results", { scenario, hospital }],
  queryFn: () => listRuleResults({ scenario, hospital }),
  staleTime: 30_000,
});
```

- `queryKey` 第一段 = 模块名；第二段 = 资源；后段 = 参数对象（让缓存按参数维度生效）。
- `staleTime` 默认 30s；高频看板（如 Provider 状态）可缩短。
- `mutationFn` 必须在成功后 `queryClient.invalidateQueries(...)` 失效相关缓存。

### 5.3 全局 store 上限

- 只放真正全局的（org / user / theme）。
- 不放页面间临时数据（用 URL queryParams 或 props）。
- 不放服务端数据（用 TanStack Query）。

## 6. API 调用

### 6.1 client.ts 拦截器

[frontend/src/api/client.ts](../../frontend/src/api/client.ts) 已实现：

- 请求自动加 `X-Trace-Id`
- 请求自动加组织上下文 Header（`X-Tenant-Id / X-Hospital-Code` 等）
- 响应 `success === false` 自动抛 `ApiError`
- HTTP 非 2xx 自动抛 `ApiError`，保留 traceId

业务侧 **永远不要**：

- 直接 `import axios from "axios"` 调接口（必须用 `import { get, post } from "@/api/client"`）
- 自己再判 `success === false`（拦截器已抛错）
- 在 try/catch 里吞 `ApiError`（让上层 React Query 或 Error Boundary 处理）

### 6.2 错误展示

参考 [frontend/src/pages/ProvidersStatus.tsx](../../frontend/src/pages/ProvidersStatus.tsx) 模式：

```tsx
{isError && (
  <Alert
    type="error"
    showIcon
    message={`后端不可达：${error?.code || "UNKNOWN_ERROR"}`}
    description={
      <>
        <div>{error?.message}</div>
        {error?.traceId && <div>trace_id: <code>{error.traceId}</code></div>}
        <Button size="small" onClick={() => refetch()}>重试</Button>
      </>
    }
  />
)}
```

**强制**显示 traceId，便于用户截图给运维。

### 6.3 表单提交

```tsx
const mutation = useMutation({
  mutationFn: publishConfigPackage,
  onSuccess: () => {
    message.success("发布成功");
    queryClient.invalidateQueries({ queryKey: ["config-packages"] });
  },
  onError: (err: ApiError) => {
    Modal.error({
      title: `发布失败 · ${err.code}`,
      content: (
        <>
          <p>{err.message}</p>
          <p>trace_id: <code>{err.traceId}</code></p>
        </>
      ),
    });
  },
});
```

## 7. 路由

### 7.1 注册

所有路由在 `src/router/routes.tsx` 集中注册。新页面：

1. `pages/Foo.tsx`
2. `routes.tsx` 加路由对象
3. `layouts/AppLayout.tsx` 菜单加项 + 面包屑 `describePath` 加 case

### 7.2 URL 设计

- 列表 `/<module>`
- 详情 `/<module>/:id`
- 编辑 `/<module>/:id/edit`
- 操作 `/<module>/:id/review` / `/<module>/:id/diff`
- queryParams 用于筛选：`?hospital=HOSPITAL_DEMO&status=DRAFT`

### 7.3 路由守卫

权限按角色由后端校验。前端只做菜单可见性 + 路由进入前的 redirect。**绝不**用前端权限假装阻断。

## 8. 样式

### 8.1 token-first

颜色 / 间距 / 字号优先用 `global.css` 中 CSS 变量 + AntD ConfigProvider theme token；不直接写颜色字面值。

```css
/* 推荐 */
.my-card { background: var(--color-surface); padding: var(--space-4); }

/* 禁止 */
.my-card { background: #ffffff; padding: 16px; }
```

### 8.2 写在哪里

| 样式类型 | 写在哪里 |
|---|---|
| 全局 token | `src/styles/global.css` |
| 组件局部 | 同目录 `Component.module.css` 或 inline style（小量）|
| AntD 主题 | `main.tsx` `ConfigProvider theme` |
| 状态颜色 | 用 AntD `Tag color="success/warning/error/processing"` |

**禁止**：

- 全局 `.foo { ... }` 选择器（破坏其它组件）
- 用 `!important` 覆盖 AntD（应该改 ConfigProvider 或换 className）
- 内联 200 行 `style={{...}}`（应该抽 CSS 文件）

### 8.3 颜色非唯一状态

[01_多角色诉求矩阵](01_多角色诉求矩阵.md) §2.2 强调：状态必须 **颜色 + 图标 + 文字** 三重编码。

```tsx
{/* 推荐 */}
<Badge status="error" text="不可用" />
<Tag color="error" icon={<CloseCircleOutlined />}>失败</Tag>

{/* 禁止 */}
<div style={{ color: "red" }}>失败</div>   // 弱视用户不可读
```

## 12. 安全

| 项 | 规则 |
|---|---|
| 凭据 | **绝不**进代码 / localStorage。Cookie / Session 由后端管理 |
| API Key | 同上 |
| 患者隐私 | 不在 localStorage / IndexedDB 缓存完整患者信息 |
| URL 参数 | 不放敏感字段（身份证、密码）|
| postMessage | 限制 origin |
| `dangerouslySetInnerHTML` | 禁用；如必须用，先 DOMPurify |
| 外部链接 | `rel="noopener noreferrer"` |
| HTTPS | 内网部署也建议；自签证书走 Java 信任库 |

## 13. 国际化（i18n）

第一版只做 zh-CN，但代码必须为未来 i18n 留口：

- 所有可见文案 **避免**写在组件深处。可以接受短期写中文字面值，但 [medkernel 后期] 会引入 i18n 时不破坏现有结构。
- 时间用 `dayjs`，统一时区。
- 数字 / 金额用 `Intl.NumberFormat` 或 `dayjs` 格式化函数封装在 `utils/format.ts`。

```nginx
server {
  listen 80;
  server_name medkernel.hospital.local;
  root /zoesoft/medkernel/frontend/dist;
  index index.html;

  location / {
    try_files $uri $uri/ /index.html;   # SPA fallback

  location /medkernel/ {
    proxy_pass http://localhost:18080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
```

## 17. 反模式（出现即驳回）

- `any` 类型未注释原因
- 直接 `import axios`，未走 `api/client`
- 用 `setState` 持续轮询 / 不取消订阅
- 不使用 `key` 渲染列表 / 用 index 当 key（动态列表）
- `useEffect` 缺依赖
- 在 `useMemo` 内调 setState
- 用 `localStorage` 缓存敏感凭据 / 患者完整数据
- 全局 `.foo` 选择器破坏其它组件
- 一个页面文件超 600 行还不拆
- 在前端"假装权限"（前端隐藏 + 后端未校验）
- AntD `notification.error("失败")` 不带 traceId / 不带错误码

## 18. 模板：新增一个页面

新增一个 `/foo-bar` 路由，必须同步：

- [ ] `src/pages/FooBar/index.tsx` 或 `src/pages/FooBar.tsx`
- [ ] 复杂页面：`pages/FooBar/components/*`、`hooks/*`、`types.ts`
- [ ] `src/router/routes.tsx` 注册路由
- [ ] `src/layouts/AppLayout.tsx` 菜单 + 面包屑
- [ ] `src/api/fooBar.ts`（如需新接口）
- [ ] `src/api/types.ts`（新 DTO）
- [ ] `src/mocks/handlers.ts`（mock 数据）
- [ ] 4 态：loading（Skeleton）/ error（Alert）/ empty（Empty）/ 数据
- [ ] 至少 1 个组件测试 + 1 个集成测试
- [ ] `frontend/README.md` 更新页面清单
- [ ] [02_任务台账](02_任务台账.md) 更新状态

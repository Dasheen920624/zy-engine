### FIX-DEV-004 · SEC-006 · 30 min · UserSyncController 类型 + 缺 method

**症状**
```
security/usersync/UserSyncController.java:[43,80] 不兼容的类型: java.lang.String 无法转换为 java.lang.Long
（共 8 处同类错误）
security/usersync/UserSyncController.java:[55,29] 找不到符号
  方法 notFound(java.lang.String)
  位置: 类 com.medkernel.common.ApiResult
```

**修复**：
1. `OrganizationContext.getTenantId()` 返回 `String`，但 `UserSyncService.listSources(Long)` 等接收 `Long`。8 处调用都要在调用方做转换：
   ```java
   Long tenantId = Long.parseLong(orgCtx.getTenantId()); // 或更稳的 NumberFormatException 处理
   ```
   或把 service 层签名改为接收 `String tenantId`（推荐，与项目其他 service 一致）
2. `ApiResult.notFound(String)` 不存在。两种选择：
   - **推荐**：抛 `throw new ConfigNotFoundException("Sync source not found");`，由 GlobalExceptionHandler 转 404
   - 备选：在 `common/ApiResult.java` 增加 `public static <T> ApiResult<T> notFound(String message) { return failure(ErrorCode.CONFIG_NOT_FOUND, message); }`

**验证**：`mvn -q compile`。

> **额外**：SEC-006 同时存在 `security/UserSyncController` 和 `security/usersync/UserSyncController` 两份实现。本 FIX-DEV-004 只修编译错；架构上要保留哪份请走 ADR（见 §3 后续动作）。

### FIX-DEV-003 · MPI-001 · 1 h · String.repeat + 3 处 SQL 反模式

**症状**
```
patientindex/util/MpiHashUtil.java:[41,36] 找不到符号
  方法 repeat(int)
  位置: 类 java.lang.String
```
（`String.repeat(int)` 是 JDK 11 引入，项目 pinned JDK 1.8）

**+**
```
patient/MpiPersistenceService.java:91, 261, 394
  ON DUPLICATE KEY UPDATE  ← MySQL 专有，Oracle/DM/PG/Kingbase/H2 均不支持
```

**修复**：
1. `MpiHashUtil.repeat(n)` 替换为 JDK 1.8 兼容写法：
   ```java
   // 替代 "0".repeat(n)
   char[] arr = new char[n];
   Arrays.fill(arr, '0');
   new String(arr);
   // 或者
   StringBuilder sb = new StringBuilder(n);
   for (int i = 0; i < n; i++) sb.append('0');
   sb.toString();
   ```
   或 Apache Commons Lang `StringUtils.repeat("0", n)`（项目已用 commons-lang3）
2. 3 处 `ON DUPLICATE KEY UPDATE` 套用 AUDIT-FULL §2.8 修复 commit `51f17e2` 的模式：同 Connection 内先 SELECT id by unique-key → 存在则 UPDATE，不存在则 INSERT；依赖 Service 层 `@Transactional` + 表唯一键兜底并发。

**验证**：
```powershell

# 然后跑（如有）MPI smoke
.\medkernel-mvp\scripts\run-mpi-smoke.ps1
```

# 08 国产化兼容性规约

## 0. 用途

医院内网部署常常涉及 **国产化要求**：信创 OS（统信 UOS / 麒麟 Kylin / OpenEuler）、国产 CPU（鲲鹏 / 飞腾 / 龙芯 / 申威）、国产数据库（达梦 DM / 人大金仓 Kingbase）、国产 JDK（毕昇 / Loongnix OpenJDK）。本文是所有 AI 在写代码、写 DDL、写部署脚本时的 **兼容性强制约束**。

冲突仲裁：架构边界 → [05_架构总图](05_架构总图与服务边界.md)；具体兼容点 → 本文为准。

## 2. CPU 架构差异

### 2.1 关键差异点

| 维度 | x86_64 | aarch64 | loongarch64 | sw_64 |
|---|---|---|---|---|
| 字节序 | LE | LE | LE | LE |
| 指针宽度 | 64 | 64 | 64 | 64 |
| 浮点 | x87 / SSE / AVX | NEON | LSX/LASX | — |
| JIT 支持 | C1/C2 | C1/C2 | C2 部分 | C1（毕昇优化中）|
| JDK 8 可用 | ✅ 全发行 | ✅ Temurin / 毕昇 | 仅 Loongnix | 仅毕昇 |
| Neo4j 5.x 包 | ✅ 官方 | ✅ 官方 | ⚠️ 需源码编译 | 🚫 不可用 |
| Oracle 客户端 | ✅ | ✅ instant client for Linux ARM | 🚫 | 🚫 |
| 达梦客户端 | ✅ | ✅ | ✅（DM V8 支持）| ✅ |
| PG 客户端 | ✅ | ✅ | ✅ | ✅（社区包）|

### 2.2 Java 项目对 CPU 的依赖

Spring Boot + JDBC 项目 **是 pure JVM**，CPU 架构差异基本被 JDK 屏蔽。但需注意：

- **Native 依赖**：本项目目前没有 JNI / Native 库。如未来引入（如 jblas / sqlite-jdbc 等），必须先评估 ARM / 龙芯 / 申威可用性。
- **加密库**：BouncyCastle / Conscrypt 在 ARM 上需用对应包；推荐用 JDK 内置 JCE（够用）。
- **fork 子进程**：`-Dfile.encoding=UTF-8` 在 ARM Linux 默认 OK；龙芯需确认 glibc locale 已生成 `zh_CN.UTF-8`。

### 2.3 部署架构判定

启动脚本（批次 3）必须包含：

```bash
ARCH=$(uname -m)
case "$ARCH" in
  x86_64)       JDK_ARCH=x64 ;;
  aarch64)      JDK_ARCH=aarch64 ;;
  loongarch64)  JDK_ARCH=loongarch64 ;;
  sw_64)        JDK_ARCH=sw_64 ;;
  *) echo "不支持的 CPU 架构: $ARCH"; exit 1 ;;
esac
echo "检测到 CPU: $ARCH ; 期望 JDK 子目录: jdk-1.8-$JDK_ARCH"
```

## 3. OS 差异要点

### 3.1 系统库

| OS | glibc | 注意 |
|---|---|---|
| CentOS 7 | 2.17 | EOL 2024-06，建议迁移 |
| OpenEuler 22.03 | 2.34 | 长期支持 |
| 统信 UOS V20 | 2.28 | 系统调用兼容 |
| 麒麟 Kylin V10 SP3 | 2.28 / 2.34 | 视小版本而定 |

JDK 1.8 在以上所有 glibc 版本均可运行。不要静态链接非 JVM 库。

### 3.2 用户与权限

部署用户必须**非 root**：

```bash
useradd -m -s /bin/bash medkernel
chown -R medkernel:medkernel /zoesoft/medkernel
```

服务用 `systemd` 启动（批次 3 提供 unit 文件）；不允许 `nohup ... &` 长期挂着。

### 3.3 防火墙 / SELinux

- CentOS / Kylin 默认 `firewalld` + SELinux enforcing。
- 部署脚本必须：

```bash
firewall-cmd --permanent --add-port=18080/tcp
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --reload

# SELinux：日志目录 / 端口
setsebool -P httpd_can_network_connect 1
semanage port -a -t http_port_t -p tcp 18080 || true
```

- 不允许 `setenforce 0` 关掉 SELinux（医院安全合规会拒）。

### 3.4 时间 / NTP / 时区

```bash
timedatectl set-timezone Asia/Shanghai
systemctl enable --now chronyd     # 或 ntpd
```

JVM 必须 `-Duser.timezone=Asia/Shanghai`。

### 3.5 locale

```bash
localedef -i zh_CN -f UTF-8 zh_CN.UTF-8   # 如未预装
export LANG=zh_CN.UTF-8
export LC_ALL=zh_CN.UTF-8
```

JVM 必须 `-Dfile.encoding=UTF-8 -Dsun.jnu.encoding=UTF-8`，否则日志中文乱码。

### 3.6 文件描述符

```bash
# /etc/security/limits.d/medkernel.conf
medkernel soft nofile 65535
medkernel hard nofile 65535
```

### 3.7 systemd 单元

批次 3 提供 `deploy/systemd/medkernel.service` 模板：

```ini
[Unit]
Description=ZY Engine backend
After=network.target

[Service]
Type=simple
User=medkernel
WorkingDirectory=/zoesoft/medkernel
Environment=JAVA_HOME=/zoesoft/medkernel/jdk
EnvironmentFile=/zoesoft/medkernel/conf/medkernel.env
ExecStart=/zoesoft/medkernel/jdk/bin/java $JAVA_OPTS -jar /zoesoft/medkernel/lib/medkernel.jar
Restart=on-failure
RestartSec=10
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
```

## 5. 数据库方言差异（开发必读）

### 5.1 类型映射

| 抽象 | Oracle | 达梦 8 | PG 16 | Kingbase V8 |
|---|---|---|---|---|
| 主键 ID | `NUMBER(20)` | `BIGINT` | `BIGINT` | `BIGINT` |
| 短字符串 | `VARCHAR2(64)` | `VARCHAR(64)` | `VARCHAR(64)` | `VARCHAR(64)` |
| 长描述 | `VARCHAR2(1000)` | `VARCHAR(1000)` | `VARCHAR(1000)` 或 `TEXT` | 同 PG |
| 大 JSON | `CLOB` | `CLOB` | `TEXT` 或 `JSONB` | `CLOB` |
| 时间戳 | `TIMESTAMP` | `TIMESTAMP` | `TIMESTAMPTZ` 推荐 | `TIMESTAMP` |
| 布尔 | 用 `NUMBER(1) 0/1` | 用 `TINYINT 0/1` 或 `BIT` | 原生 `BOOLEAN` | 同 PG |
| 自增 | 序列 + 触发器 / IDENTITY (12c+) | 序列 / IDENTITY | `BIGSERIAL` / `IDENTITY` | 同 PG |
| 中文备注 | `COMMENT ON ... IS '...'`（NLS）或 `UNISTR` | `COMMENT ON ... IS '...'` | `COMMENT ON ... IS E'...'` | 同 PG |

**项目固定约定**：

- **ID 应用层生成**，避免依赖序列 / IDENTITY。
- **JSON 大字段统一 CLOB**（Oracle/DM/Kingbase）/ `TEXT`（PG）。**查询关键字段必须结构化为独立列**。
- **布尔统一用 `NUMBER(1)` / `TINYINT`** 0/1，避免方言差异（PG 端 mapper 翻译为 boolean）。
- **时间戳统一 `TIMESTAMP`** 不带时区，应用层用 UTC 或 Asia/Shanghai 一致约定。

### 5.2 分页

| 方言 | 语法 |
|---|---|
| Oracle 11g | `SELECT * FROM ( SELECT a.*, ROWNUM rn FROM (...) a WHERE ROWNUM <= ? ) WHERE rn > ?` |
| Oracle 12c+ | `... OFFSET ? ROWS FETCH NEXT ? ROWS ONLY` |
| 达梦 8 | `... LIMIT ? OFFSET ?`（也支持 `TOP n`） |
| PG / Kingbase | `... LIMIT ? OFFSET ?` |

封装位置：`com.medkernel.persistence.dialect.PaginationDialect`（按方言注入）。**业务层禁止拼 ROWNUM**。

### 5.3 字符串拼接

| 方言 | 写法 |
|---|---|
| Oracle / DM | `'a' \|\| 'b'` 或 `CONCAT(a, b)` |
| PG | `'a' \|\| 'b'` 或 `CONCAT(a, b)` |

**推荐统一 `CONCAT`**，跨方言一致。

### 5.4 NULL 排序

| 方言 | 默认 | 显式 |
|---|---|---|
| Oracle | NULLS LAST（升序）/ FIRST（降序） | `NULLS FIRST / LAST` 显式指定 |
| DM | 同 Oracle | 同 Oracle |
| PG / Kingbase | NULLS LAST 升序，FIRST 降序 | 显式 |

**约定：所有排序显式 `NULLS LAST` 或 `NULLS FIRST`**，不依赖默认。

### 5.5 DUAL

| 方言 | 写法 |
|---|---|
| Oracle / DM / Kingbase | `SELECT 1 FROM DUAL` |
| PG | `SELECT 1` 即可，**无 DUAL** |

**约定：`SELECT` 不依赖 DUAL** 或 mapper 层翻译。

### 5.6 序列与时间函数

| 概念 | Oracle | DM | PG | Kingbase |
|---|---|---|---|---|
| 当前时间 | `SYSDATE` / `CURRENT_TIMESTAMP` | 同 Oracle | `NOW()` / `CURRENT_TIMESTAMP` | 同 PG |
| UUID | `SYS_GUID()` | `SYS_GUID()` / `UUID()` | `gen_random_uuid()`（需 pgcrypto）| `SYS_GUID()` |

**约定：用 `CURRENT_TIMESTAMP`**，跨方言可用；UUID 由应用层生成。

### 5.7 中文备注

| 方言 | 推荐脚本 |
|---|---|
| Oracle | `db/oracle/medkernel_comments_unistr.sql` 使用 `UNISTR('\6CE8\91CA')` |
| DM | 直接 `COMMENT ON COLUMN xx IS '中文注释'`（库 UTF8 即可）|
| PG | `COMMENT ON COLUMN xx IS '中文注释'`，确保 `client_encoding=UTF8` |
| Kingbase | 同 PG |

详见 [编码与中文备注规范.md](编码与中文备注规范.md)。

### 5.8 大小写

| 方言 | 标识符默认 |
|---|---|
| Oracle / DM | 转大写存储 | 必须 `"abc"` 才区分大小写 |
| PG / Kingbase | 转小写存储 | 必须 `"abc"` 才区分大小写 |

**约定：所有表名、列名 `snake_case` 全小写**，DDL 中既不加引号也不刻意大写。Oracle/DM 内部存为大写，PG/Kingbase 存为小写；JDBC mapper 用 case-insensitive。

## 8. 已知坑（实战记录）

| # | 现象 | OS / DB | 原因 | 解决 |
|---|---|---|---|---|
| 1 | Oracle 11g `getString(CLOB)` 返回 null | Oracle 11g | 11g 行为差异 | 用 `rs.getClob().getSubString(1, len)` |
| 2 | 启动后 controller 中文乱码 | 任意 + Tomcat 默认编码 | URI 编码非 UTF-8 | `server.tomcat.uri-encoding=UTF-8`（已配） |
| 3 | 日志 `?????` 替代中文 | CentOS 7 | locale 未设 zh_CN.UTF-8 | `localedef + LANG` 设好 |
| 4 | 达梦时间字段返回偏差 8h | DM 8 + JDBC | session 时区与 server 不一致 | URL 加 `?timeZone=Asia/Shanghai` |
| 5 | PG `BIT` 字段被映射成 `byte[]` | PG | JDBC 默认类型 | DDL 用 `BOOLEAN` 或 `SMALLINT`，避免 `BIT` |
| 6 | Kylin V10 ARM 启动慢 | aarch64 + 通用 OpenJDK | 未用毕昇优化 | 换毕昇 JDK |
| 7 | Oracle DDL 在 11g 报错 `IDENTITY` 不支持 | Oracle 11g | 11g 无 IDENTITY | 用序列 + 触发器或应用层 ID |
| 8 | `firewalld` 阻断 18080 | CentOS 7 / Kylin | 默认拒外 | `firewall-cmd --add-port` |
| 9 | SELinux 拒写 logs | CentOS / Kylin | 路径标签错 | `semanage fcontext + restorecon` |
| 10 | 麒麟自带 OpenSSL 1.0.2 + JDK 8 SSL 握手失败 | Kylin V10 部分小版本 | TLS 1.2 cipher 不匹配 | JDK 加 `-Djdk.tls.client.protocols=TLSv1.2` |

随实战补充，每条记录"现象 / OS+DB / 原因 / 解决"四段。

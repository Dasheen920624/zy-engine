# MedKernel v1.0 GA 产品极简化原则

> 版本：1.0 · 2026-05-24
> 任务：GA-PROD-01 全功能极简/完整/易用收口

## 1. 核心原则

### 1.1 三级递进

| 级别 | 定义 | 客户感知 |
|---|---|---|
| **极简** | 1 个主目标、1 个主按钮、最多 3 个默认筛选 | "打开就会用" |
| **完整** | 高级能力折叠，不删除功能 | "需要时找得到" |
| **易用** | 默认路径清晰，错误提示可操作 | "用起来不犯错" |

### 1.2 页面结构规范

每个客户可见页面必须满足：

1. **主目标**：页面标题即主目标，用户不需要思考"我来这里做什么"
2. **主按钮**：页面有且仅有 1 个视觉最强的主按钮（primary），其他操作为次级按钮或文字链接
3. **默认筛选**：列表页最多 3 个默认可见筛选条件，更多筛选折叠在"高级筛选"中
4. **高级折叠**：高级参数、技术名词、管理员选项默认折叠，点击"高级"或"更多"展开
5. **空状态引导**：列表为空时显示引导文案和主按钮，而非空白页

## 2. 模块级规范

### 2.1 配置包中心（ConfigPackages）

| 元素 | 规范 |
|---|---|
| 主目标 | "导入配置包，让系统跑起来" |
| 主按钮 | "导入配置包"（绿色 primary） |
| 默认筛选 | 配置包名称、状态、来源（3 个） |
| 高级折叠 | 版本号、创建时间范围、导入时间范围 |
| 列表列 | 名称、版本、状态、来源、更新时间、操作（6 列） |
| 空状态 | "还没有配置包，点击导入让系统跑起来" + 导入按钮 |

### 2.2 路径模板（Pathway）

| 元素 | 规范 |
|---|---|
| 主目标 | "查看和管理临床路径模板" |
| 主按钮 | "新建路径"（仅在编辑模式下显示） |
| 默认筛选 | 路径名称、疾病分类、状态（3 个） |
| 高级折叠 | 版本号、创建时间、适用科室 |
| 列表列 | 名称、疾病分类、版本、状态、更新时间、操作（6 列） |
| 空状态 | "暂无路径模板，请先导入配置包" + 跳转链接 |

### 2.3 规则库（Rule）

| 元素 | 规范 |
|---|---|
| 主目标 | "查看和管理规则定义" |
| 主按钮 | "新建规则"（仅在编辑模式下显示） |
| 默认筛选 | 规则名称、类型、状态（3 个） |
| 高级折叠 | 优先级、触发条件、创建时间 |
| 列表列 | 名称、类型、优先级、状态、更新时间、操作（6 列） |
| 空状态 | "暂无规则定义，请先导入配置包" + 跳转链接 |

### 2.4 图谱查询（Graph）

| 元素 | 规范 |
|---|---|
| 主目标 | "探索知识图谱关系" |
| 主按钮 | "搜索"（primary） |
| 默认筛选 | 节点名称、关系类型（2 个） |
| 高级折叠 | 深度限制、时间范围、数据源 |
| 空状态 | "输入关键词开始探索知识图谱" |

## 3. 交互规范

### 3.1 筛选交互

- 默认筛选条件直接显示在列表上方
- 超过 3 个筛选条件时，第 4 个起折叠到"高级筛选"面板
- 高级筛选面板点击"高级筛选"按钮展开/收起
- 筛选条件变更后自动刷新列表（无需手动点击"查询"）

### 3.2 列表交互

- 默认按更新时间倒序排列
- 每页 20 条，底部分页器
- 行操作：查看（默认）、编辑（次级）、删除（危险确认）
- 批量操作折叠在"更多操作"下拉中

### 3.3 详情页交互

- 顶部面包屑导航
- 主信息区：标题 + 状态标签 + 主操作按钮
- Tab 切换：基本信息（默认）、关联数据、变更历史
- 高级信息折叠在"详细信息"区域

### 3.4 表单交互

- 必填字段标记红色星号
- 保存按钮始终可见（固定在底部或顶部）
- 取消操作弹出确认（如有未保存变更）
- 校验错误定位到第一个错误字段

## 4. 禁止模式

| 模式 | 原因 |
|---|---|
| 空白页 | 客户不知道下一步做什么 |
| 超过 5 个可见筛选 | 信息过载 |
| 技术名词作为默认标签 | 客户不需要理解"Flyway migration" |
| 隐藏主操作 | 主按钮必须一眼看到 |
| 多个 primary 按钮 | 视觉混乱，不知道点哪个 |

## 5. 验收检查表

- [ ] 每个页面有明确的主目标（标题即目标）
- [ ] 每个页面有且仅有 1 个 primary 按钮
- [ ] 列表页默认筛选不超过 3 个
- [ ] 高级参数已折叠
- [ ] 空状态有引导文案和主按钮
- [ ] 技术名词已替换为客户可理解的标签
- [ ] 主菜单保持左侧 SideMenu

## 0. 一句话结论

> **后端确定性引擎内核（规则/路径/评估/随访/知识版本/字典/上下文/审计/国密）大体真实、质量不低；但用户视为"整个引擎生产核心"的「AI 医疗知识工厂」——把可信来源自主生成为说明书事实、规则、路径、指标、随访等可治理资产的全流程生成机制——实际落地仅约 10–15%（只有"来源/版本/身份"脚手架 + 一个诚实的 B0 模型网关空壳），生成、分流、探索、质量门禁、专业审核台、静默验证、灰度同步、反馈闭环这 8 个核心环节基本是空白。这是本系统最大、最该优先重构的缺口。**

配套次级问题（与既有审计一致并已部分整改）：前端约 11 页系统性假闭环（C2/C3/C6）、防造假门禁可绕过、第三方对接未真连外部、E5/E6 验收先标 done 不可信、字典映射用字符相似度有临床误配风险、规则 DSL 临床表达力有天花板。

## 3. 核心议题：AI 医疗知识工厂（生产核心）— 规范 vs 现状

这是用户本次最关注、也是本系统价值的命门。规范在 **§7.12（模型能力网关）+ §8（AI 医疗知识工厂标准化生成规范，含 GA-AIK-STD-01~12、GA-LLM-01~08）** 描述得相当完整、专业。下面对照现状。

### 3.1 规范设计的生成全流水线（§8.6）

```
探索策略 → 来源发现/离线导入 → 来源核验与原文存证 → 解析与切片(锚点)
 → 术语标准化 → 知识身份解析与版本比对 → 资产候选生成与分流
 → 自动质量门禁 → 专业审核工作台 → 静默验证与影响分析
 → 灰度与正式发布 → 院内同步与运行反馈 → (回到探索)
```

|---|---|---|---|---|---|

### 3.3 模型路由现状（§7.12.3 / GA-LLM-01~08）

| 规范要求 | 现状 | 证据 |
|---|---|---|
| provider 无关网关 + 能力码 + 统一 DTO | ✅ 框架在 | `ModelGatewayService`、8 个稳定能力码 `:45-49` |
| 路由模式 DISABLED/LOCAL_ONLY/APPROVED_EXTERNAL/LOCAL_THEN_EXTERNAL | ⚠️ 仅记录策略字符串，**无任何真实 provider** | `:129-141` 注释自承"未接入真实 provider，一律降级 B0" |
| 数据最小化（字段白名单/脱敏模板/审批编号） | ⚠️ 仅正则脱敏（手机/身份证/卡号/邮箱 + MASK_ALL 姓名/病历号），**无按能力声明的字段白名单与外发审批** | `desensitize:325-348` |
| 结构化输出 Schema 校验 | ✅ 真 JSON 解析 + required 字段校验（已从旧 contains 伪校验修复） | `validateSchema:372-387` |
| 提示词/工具/模型版本治理（可重放/回滚/审计） | ❌ 缺失（无 prompt_version 真实治理，当前恒为 "baseline"） | `:136 promptVersion="baseline"` |
| 降级记录与审计 | ✅ 诚实降级 + IsolatedAuditPublisher 失败留痕 | `:313-316` |

**正面评价**：当前网关的"诚实降级"是对的——无 provider 时返回 `mode=B0、citations=[]、confidence=null`，**绝不伪造模型名/置信度/引文/患者数据**（这是上一轮"诚实化"整改的成果，符合宪法 #9/#13）。问题不是"造假"，而是**核心生成能力根本没建**。

### 3.4 结论：生产核心 ≈ 10–15% 建成

- **已建（真实、可作地基）**：来源/版本/片段登记 + SHA-256 锚点去重；知识身份 + 版本状态机（draft→active→withdraw，悲观锁原子切换）+ supersession 替代链；通用知识包发布脚手架；诚实 B0 网关空壳 + 真 Schema 校验。
- **未建（生成核心）**：探索发现、文档解析切片、**候选生成**、**知识身份分流（8 态）**、**质量门禁（11 项）**、**专业审核台**、冲突仲裁、**静默验证**、AI 候选→包→同步→反馈闭环、真实模型 provider、提示词版本治理、字段白名单外调安全。

> 一句话：**地基（来源/版本/身份）打了，工厂的生产线（探索→解析→生成→分流→门禁→审核→仿真→发布→反馈）还没建，发动机（真实模型）还没装。**

## 4. 医疗业务严谨性专项（临床从业者视角）

### 4.1 字典映射：字符相似度 = 临床误配红线（High）

- **现状**：`autoRecommendCandidates` 用 LCS 字符相似度（`2*LCS/(m+n)`），`sim≥0.2` 即生成候选，`≥0.9` 判 LOW 风险、`≥0.5` MEDIUM，否则 HIGH（[TerminologyService.java:412-419](../../medkernel-backend/src/main/java/com/medkernel/engine/terminology/TerminologyService.java#L412)）。
- **临床风险**：字符相似 ≠ 医学等价。危险近似对会被判为高相似/低风险：
  - **肌钙蛋白T vs 肌钙蛋白I**（Troponin T/I，不同检测物）→ 差 1 字、LCS≈0.9 → 判 LOW 风险，可能被批量通过 → 心梗标志物错配。
  - **氯化钾 vs 氯化钠**（KCl/NaCl）→ 差 1 字 → 高相似 → 致命给药错配。
  - **左/右、阿托伐他汀/瑞舒伐他汀、0.25mg/0.025mg** 同理。
- **应改为**：医学语义匹配（同义词典 + 标准编码交叉表 + 模型嵌入相似度）+ **高危近似负样本判别器**（钾/钠、T/I、左/右、剂量量级差异强制 HIGH 且禁批量、禁自动确认）。这正是 AI 工厂术语标准化（STD-03）+ 安全门禁（STD-05）该承担的；当前确定性兜底在"自动确认/批量"场景下临床不安全。

### 4.2 规则引擎临床表达力天花板（Medium-High）

- **现状（真实可用）**：`RuleDslEvaluator` 是真确定性引擎——all/any/leaf 条件树、10 算子（exists/equals/not_equals/contains/gt/gte/lt/lte/in/not_in）、BigDecimal 数值比较、风险升级、HIGH/CRITICAL/BLOCK 强制医师确认（[RuleDslEvaluator.java](../../medkernel-backend/src/main/java/com/medkernel/engine/rule/RuleDslEvaluator.java)）。这是系统里质量较高的一块。
- **临床天花板**：缺以下算子，导致真实临床规则无法表达或须把复杂度外移：
  - **单位感知比较**：`0.5 g` vs `gt 250(mg)` 不会换算 → 误判（医嘱单位不统一时高危）。
  - **区间/between**：参考值/危急值区间需拆两条 gte+lte。
  - **时序/趋势**：「48h 内连续两次 K⁺>6」「eGFR 7 天内」无法表达。
  - **算术派生**：CrCl/eGFR/BSA/累计剂量须上游算好，DSL 不能算。
- **建议**：补 `between`、`unit_compare`（带单位换算表）、`temporal`（窗口/连续次数）、`derived`（受控算术）算子；高危算子配测试病例门禁。

### 4.3 CDSS / 推荐红线（既有审计已整改，保持）

- 既有审计 A7 发现并已修复：前端伪造药理卡、反馈署名硬编码 PHYS-1002、谎称"医嘱流转成功"。当前应保持"采纳/不采纳两主按钮 + 真实登录医师署名 + 不写医嘱"契约。AI 工厂产出的 CDSS 推荐模板必须经此红线。

### 4.4 版本替换 / 安全撤回 / 旧版隔离（部分）

- **已有**：knowledge 版本状态机 + supersession 链 + 旧版隔离（A3 红线判真）。
- **缺口**：§8.13 的**安全撤回（召回/禁忌升级紧急停用旧版）+ 受影响患者/路径复核任务自动生成**未端到端落地；STD-09/11"待审新版与现行版共存、未审不参与临床"需在审核台 + 包发布联动实现。

## 5. 业务流程流通性（端到端主链路）

规范主链：**源 → 标准化 → 规则/路径 → CDSS/推荐 → 评估质控 → 随访 → 知识包 → 嵌入 → 证据**。

- **确定性主链（无模型）**：各引擎单元真实，理论可串通；但既有审计 D1 指出 E5"全链路 E2E"是 1 天速通、疑 happy-path，**真实跨引擎 E2E（R-09）未可信验证**。
- **生成侧入口断裂**：主链的"源→标准化→候选"这一段没有生成工厂支撑，等于**临床资产只能纯人工录入/配置**，规模化与"最新知识自动跟进"无从谈起——这正是 AI 工厂要补的价值。
- **第三方对接**（A14）：已诚实化（不伪造 HEALTHY/SUCCESS），但**真实连接器（ping 真探活、retry 真重投递）未接（R-06）**，HIS/EMR/LIS/PACS/医保等仍是"未连接"诚实态。

## 6. 产品体验与前端（产品体验交互 / 产品架构视角）

- **假闭环重灾区 C2/C3/C6**：catch 吞错伪造成功、写死数组当真实数据展示、mock fallback。需按既有 R-04 逐页清理，并以 C5（最干净）/C4 为诚实样板。
- **防造假门禁失效**：no-page-mock 曾被 camelCase 命名绕过（根因已修），但存量 13 页 eslint-disable 与"检测 catch 伪造/写死数组"增强（R-03）待完成，否则边改边回潮。
- **7 角色硬指标对标**（宪法 §5）：院长首屏≤10s/0 技术名词、医务处 30 分钟跑 6 剧本不碰 JSON/DSL、临床 0 培训用 CDSS、信息科 1 小时接新医院。这些需在真实数据贯通后专项走查（当前部分页因假闭环无法真实验收）。
- **AI 工厂前端**：审核台/探索队列/冲突仲裁/仿真报告/新知队列 等 §8.12 指定页面绝大多数未建，AiReview 仅 18 行占位。

## 8. 重构路线图（按"一件件派单"组织）

设计原则（来自规范，强烈认同）：**工厂必须先做到"无模型可运行"（确定性 + 人工），再把大模型作为增强层叠加**（§7.12.1 / §8.9 末项 / §8.2.2）。这样既符合医疗安全（模型只产候选），又让交付可分批验收。

### 阶段 A — AI 工厂骨架（无模型，确定性 + 人工）【最高优先，生产核心的脊柱】

| 任务 | 内容 | 关键交付 | 预估 |
|---|---|---|---|
| **AIK-A1** 候选与生成任务数据模型 | `generation_job` + `asset_candidate`（含 §8.4 统一元数据）+ `candidate_disposition` + 五方言迁移 | 资产候选可落库、可分流、可追溯 | 3–4d |
| **AIK-A2** 知识身份解析与分流引擎 | 确定性优先：批准文号/指南名+版本/生效日期/canonical code/source hash/适用域 → 判定 8 态分流；hash 幂等去重 | 重复/旧版进普通审核数=0 | 4–5d |
| **AIK-A3** 文档解析与引用锚点流水线 | 解析→切片→anchor_path→SHA-256（无模型时规则/人工切片），产出 fragment 供候选引用 | 任一候选可定位原文 | 3–4d |
| **AIK-A4** 自动质量门禁（§8.9 11 项） | 来源真实性/结构完整/医学冲突/安全红线/测试病例/语言一致/去重分流/无模型可运行等发布前阻断 | 阻断项=0 才可进审核 | 4–5d |

## 9. 建议的执行方式（与既有工作流一致）

1. **派单粒度**：每个 AIK-*/MED-*/R-* 任务出一份 `docs/superpowers/specs/` 设计 + `plans/` 计划，按"PR-1 门禁/迁移 → PR-2 后端 → PR-3 前端"拆分，分支 `codex/<task-id>-<scope>`。
2. **验收门禁**：AI 工厂每阶段必须先过"无模型可运行"验收（§7.12.4 无模型组），再接 provider；高风险资产强制双审 + 测试病例齐全才可发布。
3. **backlog 据实回退**：GA-ENG-LLM-01/02、GA-AIK-STD-* 不应保持 done，按真实进度置 in_progress（与既有审计建议一致）。
4. **先后顺序建议**：**阶段 A（骨架）> 阶段 C（医疗严谨性，A2/A4 内联做掉一部分）> 阶段 B（接模型）> 阶段 D（前端红线，可与 A 并行另起一人）**。理由：先有可治理、可审核、可发布的"无模型工厂"，模型才有安全的落点；字典/规则的临床安全是任何生成的前置防线。

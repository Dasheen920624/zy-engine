## 1. 10 条验收铁律

每项任务通过验收必须不违反任一条：

### #1 真实性
**禁止**：
- 写死 `switch` / 常量当结果（如 LLM B0 写死"高血压"候选）
- `Math.random()` 生成业务关键数据（如 Ping RTT、健康分、采纳率）
- `catch` 块吞错并返回成功（伪造算法）
- `UUID` / 时间字符串 hash 充当数据完整性凭证（如证据 hash）
- 前端写死业务数据当真展示（如 CDSS 提醒卡硬编码"权威度90/Class I"）
- 假证据 / 假同步（如包同步返回时间戳哈希当同步证据）
- 测试钩子混入生产（如 `if (env.test) return ...`）
- 硬编码身份署名 / 写死医师姓名

### #2 诚实降级
无模型 / 无连接器 / 无图投影 / 外部系统断开时：
- 返回**诚实状态**：`B0` / `NOT_CONNECTED` / `NOT_SYNCED` / `MODEL_DISABLED`
- 主链路仍真实可用（确定性规则 + 已发布知识 + 人工流程）
- **绝不伪造**：不假装连接成功、不假装模型输出、不假装同步完成

### #3 医疗安全
- AI 内容必须明显标识（"AI 候选 / AI 生成 / AI 建议"）
- 医师确认才进病历（无自动开医嘱）
- 高风险强制审核 + 双签（DDI / 危急值 / 剂量上限 / 抗菌限制 / 特殊人群禁忌）
- 旧版本知识在新版生效后隔离（不参与新临床判断）
- 高危近似映射（钾/钠、肌钙蛋白T/I、左/右、剂量量级）强制 HIGH + 禁批量 + 禁自动确认

### #4 无模型可运行
- 每个含 AI 增强的能力必须先交付"无模型确定性 + 人工"路径并通过验收，再叠加模型
- B0 验收：删除/关闭全部模型配置后，来源导入/规则执行/路径推进/质控整改/包发布/同步通过

### #5 关系库权威
- 业务事实唯一权威源为院内关系数据库
- 图数据库（Neo4j）、Dify、模型、缓存只能是**投影或执行器**
- 图数据库不得成为唯一存储；Dify 不得成为业务权威源
- 同步到图/Dify 的内容必须可由关系库重建

### #6 唯一权威知识
- 同一适用域（`knowledge_identity + organization_scope + applicable_population + effective_time`）同时只能有一个 `ACTIVE_AUTHORITATIVE` 版本
- 待审新版只能审核不能执行
- 新版激活时原子失效旧版

### #7 五维权限（详规 §7.3）
| 维度 | 示例 |
|---|---|
| 菜单权限 | 27 二级菜单 + 5 高级工具粒度可见性 |
| 动作权限 | 新建/编辑/审核/发布/灰度/回滚/导出 |
| 数据权限 | 本科室/全院/集团跨院/脱敏 |
| 资产权限 | 病种/字典/路径/知识包级别 |
| 环境权限 | 测试/试运行/正式/应急 |

13 角色矩阵真实运行（不是只到 6 个一级 sectionKey）。

### #8 六态完整
每个页面齐全：加载 / 空 / 错误 / 无权限 / 部分成功 / 正常。
- 错误状态必须含中文原因 + 重试 + traceId 复制按钮
- 无权限不暴露敏感数据 + 说明权限范围 + 申请入口
- 部分成功展示成功数 + 失败数 + 失败明细 + 可重试

### #9 中文优先
- 客户可见默认中文
- 英文缩写仅保留 ICD/DRG/DIP/MPI/FHIR/OIDC 等行业标准
- 技术对象（JSON / DSL / trace / 图谱原始 / 模型提示词）默认隐藏到专家模式
- 任何"Login / Username / Payload / DSL / STRONG_REMINDER" 等技术词暴露给业务用户视为违反

### #10 文档同步
- 代码与文档同 PR
- 任何 API/状态机/字段变更必须同步详规对应章节
- commit message 必须引用规划锚点（如"对齐 详规 §8.13"）
- 同 PR 必须合并 main；功能完成后远程 main 单线机制保护

## 2. 任务通过标准模板

### 2.1 PR 中必须提供的核查证据

每项 `pending → done` 的 PR 描述必须包含：

```markdown
## 核查证据清单（任务 ID：xxx-xx）

### 代码
- 后端实现：https://github.com/.../blob/<sha>/path/to/Service.java#L10-L50
- 前端实现：https://github.com/.../blob/<sha>/path/to/Component.tsx#L10-L50
- 数据库迁移：https://github.com/.../blob/<sha>/path/to/V_n__xxx.sql

### 测试
- 单测覆盖：xxx.test.java / xxx.test.tsx（关键用例 N 个）
- 契约测试：xxx-contract.test.java
- E2E：xxx.spec.ts（剧本 N 个）
- 真实性门禁：T-GATE 红绿截图

### 文档同步
- 宪法对应章节：§x.x
- 落地规划对应章节：§x.x
- 详规对应章节：§x.x
- 体验规范对应章节：§x.x
- backlog.md 任务行 + 状态 pending → done

### 验收剧本
- A1-A9 对应剧本（若涉及）：完成 / 截图 / traceId

### 审计员签字
- @<reviewer1>（owner ≠ reviewer）
- @<reviewer2>（若高风险任务双签）
```

### 2.2 任务工作流

```
[pending]
   ↓ owner 领单
   ↓ 实现（含真实性铁律自查）
   ↓ 提交 PR 含核查证据
   ↓ CI 真实性门禁 + 测试 + lint 全绿
   ↓ 审计员（≠ owner）核对证据清单 + 抽查代码 + 关键测试本地复跑
   ↓ 审计员签字
[done]
```

任何"无证据回 done"视为注水，立即回退到 `pending`。

## 3. 通用拒绝项（PR 出现任一项直接 reject）

1. 新增一级菜单（IA 锁定 5+1）
2. 页面出现多个同权主按钮
3. 默认筛选超过 3 个
4. 大列表前端全量加载
5. AI 建议没有标识、来源或人工确认
6. 高风险发布没有影响分析、审核、回滚和证据
7. 错误、无权限、空状态缺失
8. 客户可见页面暴露 JSON / DSL / trace / 模型提示词
9. 嵌入式临床提醒阻断医生主流程且无降级策略
10. 驾驶舱指标不能下钻到行动对象
11. `eslint-disable medkernel/no-page-mock` 或同等绕门禁
12. `.module.css` 含 hex / rgb / hsl 字面量（除 `theme.ts` 一处）
13. 写死医学常量（病种/药品/编码/剂量）
14. 业务模块直接绑定模型厂商或 Dify
15. 任何"Login / Username / Payload / DSL" 等技术词暴露给业务用户

## 4. 各资产类型专项验收标准

### 4.1 配置类资产（知识包/规则/路径/字典/图谱/评估指标/随访/适配器/AI 工作流）

**必须遵守 7 步流**（宪法 §1.#4）：
```
选模板/导入 → 自动校验 → 看影响 → 提交审核 → 灰度发布 → 全量 → 留证据/可回滚
```

**通过标准**：
- [ ] 7 步流组件统一复用（不可自创"草稿→提交"两步流）
- [ ] 每步都有真实状态机推进（详规 §1.3）
- [ ] 灰度范围可选科室/病区/病种/床位/医生团队
- [ ] 回滚不丢审计、不破坏历史推荐解释
- [ ] 留证可导出（来源/审核/发布/同步/反馈/回滚证据包）

### 4.2 临床决策类资产（CDSS 提醒卡 / 治疗推荐 / 用药建议）

**通过标准**：
- [ ] AI 候选标识"AI 候选"/"AI 生成"
- [ ] 显示风险级别 / 建议动作 / 来源摘要 / 反馈入口
- [ ] 高风险建议必须展示禁忌 / 适用条件 / 替代方案 / 人工确认
- [ ] 至少可追溯到来源类型 / 来源标题 / 版本 / 引用位置 / hash
- [ ] 默认嵌入卡片或抽屉，非红线风险不阻断医生主流程
- [ ] 采纳/不采纳/关闭/忽略/稍后处理全部回传并写审计
- [ ] 静默试运行通过后才允许打扰医生

### 4.3 数据资产（标准/院内字典 / 文档解析 / 编码映射）

**通过标准**：
- [ ] 标准字典覆盖 ICD-10 国临版 / ICD-9-CM-3 / 药品本位码 / LOINC 兼容映射
- [ ] 映射候选基于医学语义（同义词典 + 编码交叉表 + 模型嵌入），不是字符 LCS
- [ ] 高危近似负样本判别器：钾/钠、肌钙蛋白T/I、左/右、剂量量级强制 HIGH + 禁批量 + 禁自动确认
- [ ] 100% 映射项有来源
- [ ] 高风险映射 100% 人工确认
- [ ] 未映射队列可追踪
- [ ] 映射发布可回滚
- [ ] 跨系统一致性报告每次发布自动生成

### 4.4 AI 工厂候选

**通过标准（详规 §8.9 11 项门禁）**：
- [ ] 来源真实性：原文 + 版本 + 锚点 + hash 100% 可回溯
- [ ] 结构完整性：schema 必填 + 编码 + 单位 + 范围，阻断项 0
- [ ] 医学冲突：所有冲突已标识并仲裁/保留待审
- [ ] 安全红线：禁忌/过敏/特殊人群/危急值/自动决策，高风险未审核不得发布
- [ ] 测试病例：规则和路径的阳性/阴性/边界/缺失/冲突用例齐全
- [ ] 语言一致性：中文术语 / 缩写 / 文案 / 单位 无不受控混写
- [ ] 运行表现：静默运行准确性/干扰度/覆盖率达医院批准阈值
- [ ] 数据安全：脱敏 + 外部调用审批 + 数据范围，无越权或未审外传
- [ ] 时效真实性：发现任务/发布日期/版本/核验时间/差异 不出现"无检索依据的最新"声明
- [ ] 生成去重分流：身份匹配 + 版本先后 + hash 去重 + 冲突/撤回分类 + 待办幂等
- [ ] 无模型可运行：关闭模型后来源导入 + 人工配置 + 审核 + 包发布 + 同步 核心流程全通过

### 4.5 第三方对接

**通过标准**（落地规划 §11.4 + 详规 §1.5）：
- [ ] 接口文档（接入概览 + OpenAPI / 事件 schema + 字段映射 + 鉴权签名 + 幂等重试 + 回调 + 降级 + 审计证据 + 医疗边界）
- [ ] 字段映射率 / 缺失率 / 重复率 / 时间戳异常可统计
- [ ] 同步超时不阻断医生主流程
- [ ] 异步任务可重试 + 死信 + 回放 + 人工补偿
- [ ] 不自动执行医嘱 / 上报 / 支付 / 签署 / 输血 / 手术 / 设备控制 / 病历归档
- [ ] 断连/重试/降级/审计/证据 E2E 通过

### 4.6 业务页面

**通过标准（每页强制）**：
- [ ] 路由元数据齐全（sectionKey + menuKey + menuLabel + requiredPermissions + requiredRoles）
- [ ] PageShell + 六态完整
- [ ] 一页一目标（≤1 主按钮 / ≤3 默认筛选）
- [ ] 默认角色视图（普通用户不见 JSON/DSL/trace/模型提示词）
- [ ] 大列表服务端分页 + 详情抽屉 + 异步导出
- [ ] 真实数据（去 mock + 去 eslint-disable + 去写死医学常量）
- [ ] 设计 token（颜色/字号/圆角必须走 Antd token CSS 变量）
- [ ] 5 主题模式响应（default/elder/dark/eye/system）
- [ ] 中文文案 + 客户面不暴露技术词
- [ ] 错误处理（onError + Form.Item validateStatus + traceId 复制按钮）
- [ ] 嵌入式提醒按打扰等级（信息提示/弱打扰/红线强打扰）

## 6. 真实性门禁规则（INFRA-01 / INFRA-02）

### 6.1 前端 ESLint medkernel/no-page-mock

**阻断模式**：
- `MockAdapter` / `mock` / `fixture` 引入 production 代码路径
- 模块顶部 `/* eslint-disable medkernel/* */`（绕门禁）
- 函数包装绕 AST（如 `const x = () => { /* 假数据 */ }` 然后多层调用）
- camelCase 绕过（如 `Mock` → `mok` / `Demo` → `dem`）
- 写死医学常量字面量：`"高血压" / "糖尿病" / "DRUG-001" / "I10" / "E11" / "J18" / "肺炎" / "心梗" / "脑卒中"`
- `font-mono` 类名（暴露技术字段）
- `<pre>{JSON.stringify(...)}</pre>` 默认渲染
- `style={{ color: '#xxx' }}` 内联颜色

**放行**：
- 静态 UI 文案常量（如菜单标题、按钮文字）
- 测试文件（`*.test.*` / `*.spec.*`）
- Storybook fixture（`*.stories.*`）

### 6.2 前端 stylelint

**阻断**：
- `.module.css` / `.css` 含 `#xxx` / `#xxxxxx` hex 字面量
- `.module.css` 含 `rgb()` / `rgba()` / `hsl()` 字面量
- `.module.css` 含 `border-radius: <px>` / `font-size: <px>` 字面量

**放行**：
- `frontend/src/shared/config/theme.ts` 一处 token 定义
- CSS 变量引用（`var(--ant-color-text)` / `var(--mk-x)`）

### 6.3 后端 CI 脚本

**阻断**（`medkernel-backend/src/main/java/` 范围）：
- `Math.random()` 出现在生产代码（非测试目录）
- 写死医学常量字符串：`"高血压" / "糖尿病" / "I10" / "E11" / "DRUG-001" / "肺炎" / "心梗"`
- `catch (...) { return success(...); }` 模式（catch 吞错伪造成功）
- `UUID.randomUUID().toString()` 用于数据完整性 hash（应用 SHA-256/SM3）
- Javadoc 中含"模拟 / 仿真 / 演示 / 占位 / placeholder" 在 production 路径

**放行**：
- 测试目录（`src/test/`）
- Migration SQL（V*.sql）
- `dev` profile bean

## 7. 状态机定义（4 套，宪法 §1.#5）

任何资产/动作必须使用以下之一，不可自创：

### 配置类
```
草稿 → 待审核 → 已发布 → 生效中 → 已下线 →（已归档）
```
适用：知识包 / 规则 / 路径 / 图谱 / 字典 / 适配器 / 评估指标 / 随访计划 / AI 工作流

### 变更类
```
待发布 → 灰度（默认 10% 床位）→ 全量 → 回滚
```
默认走灰度；仅院级管理员可"直接全量"。

### 待办类
```
未读 → 处理中 → 已完成 →（已升级）
```
按 SLA 倒序。

### 告警类
```
新建 → 已派单 → 整改中 → 已闭环 →（已豁免）
```
按责任科室分组。

任务 PR 涉及新页面/资产/告警，缺以上之一直接 reject。

### 0.3 验收铁律（所有任务通用，缺一不可）

1. **真实性**：禁止 `写死 switch/常量当结果`、`Math.random 造数`、`catch 吞错伪造成功`、`UUID 充哈希`、`前端写死业务数据当真展示`、`假证据/假同步`、`测试钩子混生产`、`硬编码身份署名`。
2. **诚实降级**：无模型/无连接器/无图投影时返回**诚实状态**（B0/NOT_CONNECTED/NOT_SYNCED）+ 真实可用主链路，**绝不伪造**。
3. **医疗安全**：AI 内容明显标识；医师确认才进病历；高风险强制审核/双签；禁自动开医嘱/诊断；旧版隔离；高危近似/剂量/禁忌不可批量自动通过。
4. **无模型可运行**：每个含 AI 增强的能力，必须先交付"无模型确定性 + 人工"路径并通过验收，再叠加模型。
5. **门禁先行**：真实性门禁（§2 P0）必须先于业务重做生效，禁止 `eslint-disable` 绕过。
6. **测试有效性**：覆盖 正常/参数失败/跨租户/权限/并发幂等/降级；**禁止 mock 掉真实现把假算法固化为绿**。
7. **关系库权威**：任何业务事实必须以院内关系库为唯一权威源；图数据库、Dify、模型、缓存只能是投影或执行器。
8. **唯一权威知识**：同一适用域同时只能有一个 `ACTIVE_AUTHORITATIVE` 版本；待审新版只能审核不能执行。

## 3bis. 重做 A 组追加 · 前端专业维护界面缺口（用户 2026-05-30 反馈坐实）

> **核查证实**：后端引擎核查为真（保留 done），但前端两份配置页面 `RuleDefinitions.tsx` + `PathwayTemplates.tsx` **系统性违反多条规划硬约束**，是产品体验灾难——医务处/专科专家/科主任**无法在此界面使用规则/路径配置功能**。

### 3bis.1 核查发现明细

| 违反 | 规划本身硬约束 | 证据（两份页面同病同模式）|
|---|---|---|
| 绕真实性门禁 | 宪法 §1.#18 | 两份第一行 `/* eslint-disable medkernel/no-page-mock */` |
| 不走 7 步流 | 宪法 §1.#4 | 创建只一步 Modal、发布只一按钮，无"自动校验/看影响/灰度/留证"步骤 |
| 客户可见暴露 JSON/DSL | 宪法 §11 禁区 + 详规 §10.2 | 抽屉默认 `<pre>{JSON.stringify(DSL)}</pre>`；TextArea 手编 nodes/edges JSON / payload JSON |

| 多同权主按钮 | 宪法 §1.#6 | 创建/发布/仿真/新增用例 多个同权主按钮 |

| 违反角色硬指标 | 宪法 §5 | "医务处不打开任何 JSON / DSL"被实质性违反；"专科专家画 X6 节点"在路径页是 TextArea 编辑 JSON |

### 3bis.2 GA-ENG-RULE-02 规则引擎专业维护界面（前端）⟳R2

**前端要求**：
1. **L1 模板模式**（业务专家：医务处/质控办）：模板库 + 参数表单 + 智能默认值，零 JSON
2. **L2 可视化条件树编辑器**（专科/质控专家）：点击式 AND/OR/NOT + 条件原子 + 时间窗 + 阈值控件，可视化条件树
3. **L3 DSL 模式**（高级实施/AI 团队）：折叠到专家模式 Tab，默认隐藏
4. **真实 7 步流**：导入/选择 → 自动校验 → 看影响 → 提交审核 → 灰度 → 全量 → 留证/回滚

### 13.2 关键设计点

1. **不是从零生成**：AI 工厂 + 真实模型 + 公开权威来源连接器自动生成候选，专家只做审核。100d 是"AI 跑批 + 专家审核工作量"的总和。
2. **首发不等于全量**：追求"试点医院能用的最小覆盖集"，不追求"国家所有指南都进来"。后者是 GA-AIK-STD-08 的持续探索职责。
3. **门面 vs 内容分工**：GA-XXX-01 领域门面提供"该领域入径/触发/嵌入/反馈"机制，GA-KNOWGEN-* 提供该领域跑起来需要的具体医学事实。**两者必须配套**——只有门面没有资产 = 空架子；只有资产没有门面 = 资产没出口。
4. **OPT-04 红线规则库**与 GA-KNOWGEN-04 临床规则首发的关系：红线规则是 GA-KNOWGEN-04 的**高危子集**，必须先于普通规则进入审核台，且**禁止 AI 自动通过——必须临床医师双签**。
5. **走 §8.13 知识身份比对**：即使是首发也走"新主题/新版本"通道，不走绕过。首发包之后任何 GA-AIK-STD-08 探索更新都按"新版替换旧版"流程执行。

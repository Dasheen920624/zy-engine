## 0. 结论速览

1. **后端确定性引擎主体为真**（规则/路径/评估/随访/上下文/知识版本/字典/审计/国密），服务体量充实、无 `Math.random` 造数，质量不低。
2. **生产核心（AI 医疗知识工厂）从未真正建设**：模型网关只有诚实的 B0 桩（无真实 provider，且 B0 写死"高血压"示例当候选）；探索/解析/候选生成/身份分流/质量门禁/审核台/静默验证/灰度同步/反馈闭环 8 大环节空白；**backlog 里连 GA-AIK-STD-01~12、GA-LLM-03~08 这些核心任务都不存在**，却已宣告"引擎全能力 done"。
3. **前端"真调用 + 假覆盖"**：有真实 API 层（axios + 105 React Query hooks 对接 /api/v1/*），但 ~25 页**系统性绕过 no-page-mock 门禁**（eslint-disable + 用函数包装规避 AST 检测）并叠加确凿假数据（`SHA-256-MOCK-HASH`、"Mock KPI 以体现高设计感"、CDSS 假审计抽屉、硬编码循证强度）。
4. **医疗严谨性两处算法红线**：字典映射用 LCS 字符相似度（"肌钙蛋白T/I""氯化钾/钠"会误配且判低风险）；规则 DSL 缺单位换算/区间/时序/算术算子。
5. **后端坐实假闭环**：包同步用 `LenientPackageSyncAdapter` 把"租户;计划;时间"算个 SHA-256 当**同步证据**返回（实际没同步）；包"看影响科室"注释自承"模拟"。
6. **验收不可信**：69 任务全标 done，含 E2E/医疗安全/代码净化/第三方等验收门，都是在上述假闭环之上"通过"的。

> 一句话：**引擎内核大体是真的；但生产核心（AI 工厂）从未建、前端系统性假覆盖、防造假门禁被系统性绕过、验收在假闭环上盖了章。需要据实重置并按生产核心优先重建。**

## 1. 核查方法与信任声明

- **独立取证**：本轮不采信 `docs/audit/` 既有结论，全部从真实代码重新判定（关键处带 `file:line`）。
- **广度优先**：按用户要求"不过度深入、先出报告"，采用判伪模式全库扫描 + 关键服务速读 + 前端数据源核查 + 两个 P0 单元（LLM/CDSS）独立 agent 深审，综合成此报告。
- **可信度标注**：每条结论标 `[实锤]`（亲验 file:line）/ `[广度]`（扫描信号推断，未逐行）/ `[存疑]`（需进一步核）。

## 3. 三条系统性主线（改造的重点）

### 主线一：生产核心 AI 工厂从未建（最高优先）
规范 §7.12+§8 设计完整（12 个 GA-AIK-STD + 8 个 GA-LLM 任务），但 backlog 从未登记，代码仅脚手架 + B0 桩。详见配套 [AI 工厂深度报告](2026-05-29-全系统深度核查与AI工厂重构总报告.md)。**铁律：先做到"无模型可运行"（确定性+人工）的工厂，再叠加大模型；模型只产候选不产事实。**

### 主线二：前端"真调用+假覆盖"+ 门禁失效
前端有真实 API 层，但被系统性绕过门禁 + 叠加假数据。需：①增强 no-page-mock（识别 catch 伪造、写死业务数组、函数包装规避、camelCase 绕过），禁止 eslint-disable 滥用；②逐页 triage 分清"合法初始/静态 UI 常量"与"假业务数据"，后者清零（catch 改报错+空态、数据仅来自 query）。诚实样板：disabled 占位页。

### 主线三：医疗严谨性算法红线
①字典映射：LCS→医学语义匹配（同义词典+编码交叉表+模型嵌入）+ 高危近似负样本判别（钾/钠、T/I、左/右、剂量量级强制 HIGH、禁批量/禁自动确认）。②规则 DSL：补 between/unit_compare/temporal/derived 算子 + 高危测试病例门禁。③安全撤回与旧版下游隔离端到端（§8.13）。

## 5. 改造总计划（分阶段、可一件件派单）

> 排序原则：先据实归零与防回潮 → 建生产核心骨架 → 清前端假覆盖 → 接真实模型 → 医疗严谨性横切 → 真连接器与重新验收。

### 阶段 0 · 据实归零与门禁加固（1–2 周）
- T0-1 backlog 据实重置（本报告 §4）+ 修订记录。
- T0-2 增强 no-page-mock 门禁（识别 catch 伪造/写死业务数组/函数包装绕过/camelCase），禁 eslint-disable 绕过；存量 25 页爆出待清。

### 阶段 1 · AI 工厂骨架（无模型可运行，生产核心脊柱，4–6 周）
- AIK-A1 候选与生成任务数据模型（generation_job/asset_candidate/candidate_disposition + §8.4 元数据 + 五方言迁移）
- AIK-A2 知识身份解析与 8 态分流引擎（确定性优先 + hash 幂等去重）
- AIK-A3 文档解析与引用锚点流水线
- AIK-A4 自动质量门禁（§8.9 共 11 项）
- AIK-A5 专业审核工作台（替换 AiReview 18 行占位；左右对照/高风险优先/批量限低风险/差异留痕/一键追溯）
- AIK-A6 静默验证与影响分析（复用 rule/pathway 引擎跑历史脱敏病例）
- AIK-A7 探索策略与新知队列（连接器框架 + 离线导入 + 最新性治理）
- AIK-A8 候选→包→发布→同步→反馈闭环（接既有 package/sync）

### 阶段 2 · 假闭环清零（前后端，3–4 周，可与阶段 1 并行另起一人）
- T2-1 后端坐实假闭环修复：LLM B0 候选去写死、PKG 同步证据诚实化（未同步标 NOT_SYNCED）、EVID 大导出接真实文件、CDSS 前端契约对齐。
- T2-2 前端逐页 triage：清除假业务数据，catch 改报错+空态，数据仅来自 query；保留合法静态 UI 常量但去 eslint-disable。

### 阶段 3 · 接入真实大模型（4–6 周）
- AIK-B1 provider SPI + 真实接入（本地 B1/获批外部 B2/Dify），缺位仍诚实降级。
- AIK-B2 提示词/工具/模型版本治理（可重放/回滚/审计）。
- AIK-B3 模型增强各骨架环节（只产候选）。
- AIK-B4 数据最小化与外调安全（字段白名单/脱敏/审批/证据）。
- AIK-B5 模型安全与医学回归评测（引用真实性/红线/基准集/专家复核）。

### 阶段 4 · 医疗严谨性横切（与 1/3 并行，3–4 周）
- MED-C1 字典映射改医学语义匹配 + 高危近似判别器。
- MED-C2 规则 DSL 补临床算子（单位/区间/时序/算术）+ 高危测试病例门禁。
- MED-C3 安全撤回与旧版下游隔离端到端。

### 阶段 5 · 真连接器与重新验收（3–4 周）
- T5-1 第三方对接真实连接器（ping 真探活/retry 真重投递）。
- T5-2 真跨引擎全链路 E2E（源→标准化→规则/路径→CDSS→评估→随访→包→嵌入→证据）。
- T5-3 据实重跑 QA-01/03/04/05/06/07/08，通过后方可重新宣告引擎全能力。

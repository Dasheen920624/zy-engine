## 5. 本次范围未覆盖（盲区）

- **前端 lint / typecheck / test / build 实测**：本机 Node 12 装不上依赖（vite-node@2.1.2 需 Node 18+），需远端 CI 或升级本机 Node 验证
- **DDL 一致性核查**：Oracle / DM / PG / Kingbase 4 套 SQL 增量同步状态未逐表对照（参考 `smoke-plan-ddl-consistency.md`）
- **Smoke 脚本是否真在 Oracle 主库跑通**：依赖 `MEDKERNEL_DB_URL` 实际连通的 Oracle 实例，非本机能力
- **NeoJ4 / Dify 实际 Provider 通断**：DB-only 模式假设外部依赖可降级，需在有 Neo4j 实例的环境跑 smoke 验证
- **多 AI 并行冲突复盘**：本次没看 `ai-dev-input/10_task_claims/active/` 是否有锁，建议下次审计专门看一次

## 1. License 授权模型

### 1.1 授权维度

| 维度 | 说明 | 计量方式 |
|------|------|---------|
| 部署模式 | 本地部署 / SaaS | 按实例 |
| 用户数 | 并发用户 / 命名用户 | 按人数 |
| 模块 | 基础版 / 专业版 / 旗舰版 | 按功能集 |
| 期限 | 年度订阅 / 永久授权 | 按时间 |
| 数据库方言 | Oracle / DM / PG / KingbaseES | 按方言 |

### 1.2 版本套餐

| 功能模块 | 基础版 | 专业版 | 旗舰版 |

### 1.3 License 文件格式

```json
{
  "license_id": "MK-LIC-2026-XXXXX",
  "customer": "XX医院",
  "product": "medkernel-mvp",
  "edition": "professional",

  "issued_at": "2026-05-23",

  "signature": "SM2-SIG-XXXX"

```

### 1.4 License 验证流程

```
1. 应用启动时读取 License 文件
2. 验证签名（SM2/RSA）
3. 检查有效期
4. 检查模块权限
5. 检查用户数限制
6. 验证通过 → 正常启动
7. 验证失败 → 降级模式或拒绝启动
```

## 2. 用量指标定义

### 2.1 核心用量指标

| 指标 ID | 指标名称 | 计算方式 | 采集频率 | 存储周期 |
|---------|---------|---------|---------|---------|
| USG-001 | 活跃用户数 | 当日登录用户去重 | 每日 | 12 个月 |
| USG-002 | API 调用量 | HTTP 请求计数 | 每小时 | 12 个月 |
| USG-003 | 路径实例数 | 患者路径实例累计 | 每日 | 12 个月 |
| USG-004 | 规则执行次数 | 规则引擎触发计数 | 每小时 | 12 个月 |
| USG-005 | 术语查询次数 | 术语服务调用计数 | 每小时 | 12 个月 |
| USG-006 | 图谱查询次数 | 知识图谱调用计数 | 每小时 | 12 个月 |
| USG-007 | AI 推荐次数 | Dify 调用计数 | 每小时 | 12 个月 |
| USG-008 | 存储使用量 | 数据库占用空间 | 每日 | 12 个月 |

### 2.2 指标采集方案

| 方案 | 说明 | 适用场景 |
|------|------|---------|
| Prometheus 指标 | 通过 Micrometer 自动采集 | 实时监控 |
| Actuator 端点 | `/actuator/metrics/*` | 按需查询 |
| 审计日志聚合 | 从 `sec_audit_chain` 聚合 | 历史分析 |
| 定时报告任务 | 每日汇总写入用量表 | 月度报告 |

### 2.3 用量数据存储

```sql
-- 用量指标日表
CREATE TABLE cfg_usage_daily (
    id              NUMBER PRIMARY KEY,
    metric_id       VARCHAR2(32) NOT NULL,
    metric_date     DATE NOT NULL,
    metric_value    NUMBER NOT NULL,
    org_id          VARCHAR2(64),
    created_at      TIMESTAMP DEFAULT SYSTIMESTAMP,
    CONSTRAINT uk_usage_daily UNIQUE (metric_id, metric_date, org_id)
);
```

## 4. 到期提醒机制

### 4.1 提醒规则

| 触发条件 | 提醒方式 | 提醒对象 | 提醒内容 |
|---------|---------|---------|---------|
| 到期前 90 天 | 系统通知 + 邮件 | 客户成功经理 | License 即将到期，建议续约 |
| 到期前 60 天 | 系统通知 + 邮件 | 客户成功经理 + 客户 IT | 续约提醒，附用量报告 |
| 到期前 30 天 | 系统通知 + 邮件 + 短信 | 客户 IT + 院领导 | 紧急续约提醒 |
| 到期前 7 天 | 系统通知 + 邮件 + 电话 | 客户 IT | 最后续约机会 |
| 到期日 | 系统降级 | 系统自动 | 功能降级，仅保留基础查看 |
| 到期后 30 天 | 系统锁定 | 系统自动 | 系统只读，无法新增数据 |

### 4.2 降级策略

| 阶段 | 功能限制 | 数据访问 |
|------|---------|---------|
| 正常 | 全功能 | 全访问 |
| 到期后（宽限期） | 禁止新增路径/规则 | 查看和导出 |
| 过期 30 天后 | 仅查看 | 查看和导出 |
| 过期 90 天后 | 系统锁定 | 需联系厂商解锁 |

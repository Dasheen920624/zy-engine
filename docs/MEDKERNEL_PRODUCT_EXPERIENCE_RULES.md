# MedKernel 全系统产品与交互体验固定规范

> 版本：1.0 · 2026-05-24
> 状态：v1.0 GA 产品体验硬约束
> 适用对象：产品、设计、前端、后端、AI 团队、QA、实施、临床专家
> 定位：本文把全系统产品与交互体验整改建议固化为明文规则。任何页面、接口、任务或 AI 细化方案与本文冲突，按本文整改。

---

## 0. 产品体验总目标

MedKernel 的体验目标不是“功能很多”，而是让医院在复杂医疗知识、复杂组织、复杂系统对接和复杂合规要求下，仍然觉得系统**清楚、可信、低打扰、可追溯、能落地**。

全系统体验必须同时满足 5 个判断：

| 判断 | 标准 |
|---|---|
| 清楚 | 用户 10 秒内知道当前页面在解决什么问题、下一步该做什么 |
| 可信 | 每条建议、规则、路径、知识和质控结论都能看到来源、版本、解释和责任人 |
| 低打扰 | 临床现场不被无价值弹窗、重复提醒、复杂表单和技术细节打断 |
| 可控 | 高风险动作必须审核、确认、灰度、回滚和留证 |
| 可扩展 | 知识、规则、路径、日志、证据达到百万级时仍能分页、筛选、批量、导出和审计 |

## 1. 世界级产品整改结论

| 问题 | 整改建议 | 固定规则 |
|---|---|---|
| 能力很多但用户心智容易分散 | 统一成“工作台、试点准备、临床运行、质控改进、合规运维 + 高级工具” | 禁止新增一级菜单；任何新页面必须归入 5+1 |
| 引擎对象偏技术 | 普通用户只看业务语言，复杂对象折叠到专家模式 | JSON、DSL、trace、图谱、原始规则默认隐藏 |
| 配置流程容易各做各的 | 所有配置能力复用 7 步流 | 配置包、规则、路径、字典、评估、适配器、AI 工作流不得自创流程 |
| 知识和日志可能无限增长 | 服务端分页、游标、默认筛选、保存视图、异步导出 | 禁止前端一次性加载大列表；导出必须任务化并留审计 |
| AI 容易让用户误以为系统自动决策 | AI 只做候选、解释和建议 | AI 标识、来源、置信度、风险、人工确认和回滚必须可见 |
| 医生时间被打断 | 嵌入、轻量、低打扰、可忽略但可追溯 | 默认不使用遮挡式弹窗；关键红线才允许强提醒 |
| 管理驾驶舱容易变成看板堆砌 | 只展示风险、价值、进度、待办和证据 | 禁止无动作图表；每个指标必须能下钻到责任对象 |
| 实施配置容易重学习 | 页面、动作、状态、错误和空态统一 | 同类页面必须复用同一结构、同一文案和同一状态机 |
| 审核体验容易低效 | 风险优先、影响优先、待我处理优先 | 审核队列默认不展示重复、旧版、低价值噪声 |
| 产品美观可能牺牲医疗安全 | 安全、解释和证据优先于视觉装饰 | 禁止为了“干净”隐藏高风险依据、冲突和责任链 |

## 2. 全系统体验原则

| 原则 | 明文规则 |
|---|---|
| 一页一目标 | 每个页面只能有 1 个主目标、1 个主按钮和不超过 3 个默认筛选 |
| 先摘要后细节 | 首屏只给状态、风险、下一步和关键证据；复杂信息进入抽屉、Tab 或专家模式 |
| 先业务后技术 | 客户可见文案使用医院能理解的中文业务词；技术对象只在专家模式出现 |
| 先行动后解释 | 页面必须告诉用户下一步动作；解释、来源和 traceId 作为可信支撑 |
| 先当前后历史 | 默认展示当前有效、待我处理、高风险和最近变化；历史进入时间线或分页 |
| 先角色后全量 | 默认视图按角色裁剪；全量数据通过高级筛选、专家模式或导出查看 |
| 先安全后效率 | 高风险医疗动作必须逐条确认；低风险重复事务才允许批量 |
| 先可恢复后发布 | 发布、同步、替换、回滚和撤回动作必须有影响分析与证据 |

## 3. 角色体验标准

| 角色 | 默认看到 | 默认不看到 | 世界级体验标准 |
|---|---|---|---|
| 院长 / 集团管理层 | 风险、价值、进度、ROI、整改闭环 | 字段映射、规则 DSL、trace | 10 秒看懂今天是否安全、有效、值得继续投入 |
| 医务处 | 高风险审核、路径发布、质控整改、影响范围 | 图谱原始结构、模型提示词 | 30 分钟完成一次知识审核和发布判断 |
| 临床医生 | 当前患者相关提醒、建议动作、来源摘要、采纳/不采纳 | 系统配置、包版本细节、长表格 | 不离开医生工作站，不被低价值提醒打断 |
| 护士 / 药师 / 医技 | 本专业相关风险、任务、解释和反馈 | 其它专业配置细节 | 只看到与当前工作场景相关的行动信息 |
| 信息科 | 接入进度、接口健康、字段质量、权限、安全、同步日志 | 医学审核细节 | 1 小时内完成新院区/科室接入准备路径 |
| 质控办 / 医保办 | 问题、证据、责任科室、整改状态、复核结果 | 原始技术日志 | 从发现到派单、整改、复核不重复录入 |
| 合规审计 | 来源、审核、发布、运行、反馈、回滚证据 | 非授权患者明细 | 任意关键动作可导出证据包 |
| 专科专家 | AI 候选、差异、冲突、影响病例、来源原文 | 平台运维配置 | 快速校对，而不是从零创建 |
| 实施工程师 | 向导、配置包、适配器、校验、发布进度 | 临床隐私明细 | 少培训、少问人、能自检、能回滚 |

## 4. 信息架构与导航

### 4.1 菜单规则

| 规则 | 标准 |
|---|---|
| 一级菜单 | 永远 5 组 + 1 个隐藏式高级工具 |
| 二级菜单 | 以任务命名，不以技术模块命名 |
| 高级工具 | 只给专家、实施、运维和架构角色；入口弱化 |
| 面包屑 | 必须来自统一路由元数据 |
| 页面标题 | 中文业务名 + 当前组织上下文 |
| 跨域跳转 | 不允许让用户记住系统结构；必须提供返回上下文 |

### 4.2 页面类型

| 类型 | 用途 | 固定结构 |
|---|---|---|
| 工作台 | 看状态、风险、价值和待办 | 摘要卡 + 待办 + 风险 + 最近变化 |
| 列表页 | 找对象、筛选、批量、进入详情 | 筛选 + 表格 + 分页 + 详情抽屉 |
| 配置页 | 导入、校验、影响、审核、发布 | 7 步流 |
| 审核台 | 判断候选是否可用 | 队列 + 左右对照 + 影响 + 审核结论 |
| 详情页 | 看一个对象全貌 | 摘要 + 状态 + 来源 + 版本 + 审计 |
| 嵌入面板 | 在院内系统里给建议 | 小卡片 + 动作 + 解释抽屉 + 反馈 |
| 驾驶舱 | 看趋势和责任 | 指标 + 风险 + 下钻 + 行动 |
| 专家工具 | 图谱、DSL、JSON、trace | 默认折叠，需权限进入 |

## 5. 页面交互固定规则

| 项 | 固定规则 |
|---|---|
| 主按钮 | 每页最多 1 个主按钮；其它动作用次按钮、菜单或抽屉承载 |
| 默认筛选 | 不超过 3 个；更多筛选进入“高级筛选” |
| 表格列 | 默认不超过 8 列；技术字段、长文本和审计字段进入详情 |
| 详情 | 长列表必须用右侧详情抽屉；打开详情不得刷新列表 |
| 状态 | 必须有加载、空、错误、无权限、部分成功、正常六态 |
| 空状态 | 必须告诉用户为什么空、下一步做什么、是否需要权限 |
| 错误状态 | 必须展示中文原因、重试入口和 traceId |
| 无权限 | 不暴露敏感数据；说明权限范围和申请路径 |
| 部分成功 | 展示成功数、失败数、失败原因和重试入口 |
| 危险动作 | 删除、撤回、回滚、全量发布必须二次确认并展示影响范围 |
| 草稿保护 | 长表单、路径编辑、规则编辑必须自动保存草稿 |
| 可撤销 | 批量、发布、替换、同步动作必须提供回滚点或撤销解释 |

## 6. 大规模知识与资产体验

知识、字典、规则、路径、评估指标、AI 候选、运行日志、审计证据和发布记录可能达到万级、百万级。体验设计必须从第一天按大规模处理，不允许先做小数据页面再返工。

| 场景 | 固定规则 |
|---|---|
| 普通列表 | 默认每页 20，允许 50/100；服务端分页；默认排序清晰 |
| 海量列表 | 必须支持游标分页；总数允许异步估算 |
| 审核队列 | 默认只看待我处理、高风险、新版差异；重复和旧版进入已拦截视图 |
| 搜索 | 支持关键词、来源、知识身份、版本、组织、风险、状态、时间范围 |
| 保存视图 | 复杂筛选可保存为个人视图或角色视图 |
| 批量处理 | 支持选择本页、选择筛选结果、异步任务；高风险不可批量自动通过 |
| 导出 | 大范围导出必须异步，记录筛选条件、操作者、导出时间和审批 |
| 长文档 | 按章节锚点分页，默认定位到变更片段 |
| 图谱结果 | 默认展示核心节点，超过阈值按关系类型、证据等级或路径折叠 |
| 历史版本 | 默认展示当前版、上一版、待替换版；完整历史时间线分页 |

统一性能门槛：

| 项 | 门槛 |
|---|---|
| 首屏加载 | P95 ≤ 1s，极端内网场景必须有骨架屏和进度反馈 |
| 筛选响应 | P95 ≤ 2s |
| 详情打开 | 不触发列表重载 |
| 批量任务提交 | 立即返回任务 ID 和预计完成方式 |
| 导出 | 不阻塞页面；完成后通知并可追溯 |

## 7. 医疗安全与可信体验

| 场景 | 固定规则 |
|---|---|
| AI 候选 | 必须标识“AI 候选 / AI 生成 / AI 修改建议” |
| 推荐卡 | 必须显示风险级别、建议动作、来源摘要和反馈入口 |
| 高风险医疗建议 | 必须展示禁忌、适用条件、替代方案和人工确认 |
| 来源解释 | 至少可追溯到来源类型、来源标题、版本、引用位置、hash |
| 知识替换 | 展示新版差异、旧版影响、替换原因和回滚/撤回策略 |
| 冲突仲裁 | 不允许静默选择；必须展示冲突双方和优先依据 |
| 静默试运行 | 规则或推荐正式打扰医生前，必须支持静默观察和报告 |
| 责任边界 | 页面文案不得暗示系统自动诊断、自动治疗或替代医师判断 |

## 8. 临床嵌入体验

| 项 | 固定规则 |
|---|---|
| 默认形态 | 在医生工作站、EMR、护理站等系统中嵌入轻量卡片或抽屉 |
| 打扰级别 | 信息提示不打断；重要风险提示弱打断；红线风险才强打断 |
| 动作 | 医生卡片默认只有采纳、不采纳并说明、稍后处理、查看依据 |
| 降级 | 引擎不可用时显示“智能建议暂不可用”，不得阻断院内系统主流程 |
| 反馈 | 关闭、忽略、采纳、不采纳、稍后处理都必须回传并审计 |
| 最小数据 | 嵌入启动只传当前场景必要字段 |
| 可读性 | 嵌入区域必须适配小屏、老旧浏览器和医生站低分辨率窗口 |

## 9. 驾驶舱与管理体验

| 规则 | 标准 |
|---|---|
| 指标有动作 | 每个指标必须能下钻到对象、责任人、科室或待办 |
| 不做装饰图 | 禁止只展示炫酷但不可行动的图表 |
| 先异常后全量 | 默认展示风险、异常、逾期、变化和趋势 |
| 价值指标 | 采纳率、误报率、漏报回溯、路径完成率、整改闭环率、医保违规减少必须可配置 |
| 证据导出 | 管理视图必须能导出阶段报告或证据快照 |

## 10. 表单、校验与发布体验

| 项 | 固定规则 |
|---|---|
| 表单分段 | 超过 7 个字段必须分组；超过 15 个字段必须步骤化或折叠 |
| 默认值 | 能从组织、角色、包版本、历史配置推断的字段必须给默认值 |
| 实时校验 | 字段级错误就地提示；发布级错误进入校验结果页 |
| 影响分析 | 发布、替换、撤回、回滚前必须显示影响组织、科室、患者、规则和路径 |
| 审核意见 | 高风险审核必须填写意见；低风险可使用结构化原因 |
| 发布完成 | 必须展示发布范围、同步状态、回滚点和证据入口 |

## 11. 文案与视觉

| 项 | 固定规则 |
|---|---|
| 语言 | 客户可见默认中文；英文缩写只保留 ICD、DRG、DIP、MPI、FHIR、OIDC 等行业标准 |
| 语气 | 清楚、克制、专业；不使用营销化、夸张化、拟人化文案 |
| 按钮 | 使用动词 + 对象，例如“提交审核”“开始灰度”“导出证据” |
| 风险文案 | 必须说明风险事实、影响范围和建议动作 |
| 颜色 | 颜色只表达状态，不表达装饰；危险色只用于真实危险 |
| 字号 | 正文 14px；老年医生模式 ≥ 16pt |
| 可访问性 | 对比度、键盘可达、焦点态、表格可读性必须验收 |
| 视觉密度 | 医疗运营界面以高信息效率为主，不做营销型大卡片堆叠 |

## 12. AI 开发与 PR 门禁

任何 AI 团队、开发者或后续任务拆分，都必须在交付中回答以下问题：

| 门禁 | 必须回答 |
|---|---|
| 用户目标 | 这个页面/能力让哪个角色完成什么任务 |
| 主路径 | 用户从进入到完成的 3-7 个关键步骤是什么 |
| 数据规模 | 是否可能达到万级/百万级；分页、筛选、导出如何处理 |
| 风险等级 | 是否涉及医疗建议、发布、替换、撤回、隐私或审计 |
| 六态 | 加载、空、错误、无权限、部分成功、正常是否完整 |
| 证据 | 来源、审核、发布、运行、反馈、回滚是否可追溯 |
| 降级 | 模型、Dify、图投影、外部系统、网络失败时用户看到什么 |
| 验收 | 产品、前端、后端、QA 分别如何判断完成 |

违反以下任一项，PR 必须整改：

1. 新增一级菜单。
2. 页面出现多个同权主按钮。
3. 默认筛选超过 3 个。
4. 大列表前端全量加载。
5. AI 建议没有标识、来源或人工确认。
6. 高风险发布没有影响分析、审核、回滚和证据。
7. 错误、无权限、空状态缺失。
8. 客户可见页面暴露 JSON、DSL、trace 或模型提示词。
9. 嵌入式临床提醒阻断医生主流程且无降级策略。
10. 驾驶舱指标不能下钻到行动对象。

## 13. 验收清单

| 验收项 | 通过标准 |
|---|---|
| 5 秒定位 | 用户能在 5 秒内说出页面目标 |
| 10 秒判断 | 用户能在 10 秒内知道是否需要行动 |
| 3 步到行动 | 关键任务从页面进入到首次有效动作不超过 3 步 |
| 7 步发布 | 配置类任务完整复用 7 步流 |
| 大数据可用 | 10 万级列表分页、筛选、打开详情、导出不崩溃 |
| 低打扰 | 临床提醒按打扰等级展示，非红线不强打断 |
| 可追溯 | 任意建议可看到来源、版本、解释和 traceId |
| 可恢复 | 发布、替换、撤回和同步都有回滚点或撤回说明 |
| 可降级 | 无模型、无 Dify、无图投影时主链路可完成 |
| 可审计 | 关键动作能导出证据快照 |

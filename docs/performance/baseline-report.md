## 3. 测试场景

### 场景 S1：P0 核心实时 API 压测

- **目标**：验证核心临床决策接口在 100 并发下的延迟和稳定性
- **并发数**：100 VU
- **持续时间**：30 分钟
- **加压策略**：阶梯式（0→10 VU 1min → 10→50 VU 9min → 50→100 VU 10min → 100 VU 10min）
- **测试接口**：
  - `POST /api/cdss/evaluate` — CDSS 评估
  - `POST /api/pathways/patient-pathways/candidates` — 路径入径
  - `POST /api/rule-engine/evaluate` — 规则评估
  - `POST /api/cdss/triggers/match` — 触发点匹配
  - `POST /api/adapters/query` — 适配器查询
  - `POST /api/auth/login` — 认证登录
- **流量配比**：CDSS 30% / 路径入径 25% / 规则评估 20% / 触发点 10% / 适配器 10% / 认证 5%

### 场景 S2：P1 查询类 API 压测

- **目标**：验证查询接口在 100 并发下的响应能力
- **并发数**：100 VU
- **持续时间**：30 分钟
- **测试接口**：
  - `GET /api/cdss/fatigue/alerts` — 告警查询
  - `GET /api/pathways` — 路径列表
  - `GET /api/knowledge/sources` — 知识源查询
  - `POST /api/interop/query` — 互操作查询
  - `GET /api/quality/dashboard` — 质量仪表盘

### 场景 S3：P2 管理类 API 压测

- **目标**：验证管理接口在 50 并发下的响应能力
- **并发数**：50 VU
- **持续时间**：15 分钟
- **测试接口**：
  - `POST /api/knowledge/sync` — 知识同步
  - `GET /api/security/baseline` — 安全基线
  - `GET /api/ops/tasks` — 部署管理

### 场景 S4：混合场景压测

- **目标**：模拟真实生产流量，P0:P1:P2 = 5:3:2
- **并发数**：100 VU
- **持续时间**：30 分钟

### 场景 S5：连接泄漏检测

- **目标**：100 并发持续 30 分钟后，验证数据库连接池无泄漏
- **并发数**：100 VU
- **持续时间**：30 分钟
- **检测方法**：压测前后对比 `hikaricp_connections_active` 指标

### 场景 S6：峰值突发测试

- **目标**：验证 2x 常规并发下系统弹性
- **并发数**：200 VU
- **持续时间**：5 分钟

## 附录 B：Prometheus 查询参考

```promql
# P0 API P95 延迟（按 URI 分组）
histogram_quantile(0.95,
  sum(rate(http_server_requests_seconds_bucket{application="medkernel-mvp",uri=~"/api/cdss/.*|/api/pathways/patient-pathways/.*|/api/rule-engine/.*"}[5m]))
  by (le, uri)
)

# 全局错误率
sum(rate(http_server_requests_seconds_count{application="medkernel-mvp",status=~"5.."}[5m]))
/
clamp_min(sum(rate(http_server_requests_seconds_count{application="medkernel-mvp"}[5m])), 0.001)

# 连接池饱和度
max(hikaricp_connections_active{application="medkernel-mvp"})
/
clamp_min(max(hikaricp_connections_max{application="medkernel-mvp"}), 1)
```

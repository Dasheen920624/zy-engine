# 05 AI 实施手册（Execution Playbook）

> 版本：V2.0 · 2026-05-18  
> 这是接下来 12 个 PR 的精确清单。AI 拿到 PR 编号 → 直接开工，无需再读其它文档（除了对应的剧本/规格章节）。  
> 颗粒度：单 PR 工时控制在 4-12 小时，独占文件清单明确，依赖关系明确。

---

## 0. 怎么用本手册

### 0.1 接手流程

```
1. 检查"PR 拓扑图"（§2），确认你的 PR 没有依赖未完成
2. 跳到对应 PR 章节（§3 起）
3. 创建 active claim（参考 00_总入口）
4. 按"实施步骤"做
5. 按"验收脚本"自检
6. 创建 PR + commit + push
7. 进入 review 流程
```

### 0.2 PR 卡片字段说明

每个 PR 固定 11 字段：

```
PR 编号     | 任务台账映射 | 归属泳道
依赖        | 阻断哪些 PR
独占文件    | （glob 列表）
共享文件    | （需协调的文件）
工时估算    | 4-12 小时
难度等级    | 初级/中级/高级
实施步骤    | 精确到代码改动
验收脚本    | 自检命令
DoD 检查表  | (复制到 PR 描述)
对应文档    | 必读的 V2 文档章节
```

### 0.3 不要做什么

- ❌ 不要"顺手"修改其它 PR 的独占文件
- ❌ 不要跳过验收脚本
- ❌ 不要在一个 PR 里做多个 task
- ❌ 不要在依赖未完成时硬上（先做 mock，标记 TODO）

## 1. 总览与战略

### 1.1 当前定位

```
后端：3 大引擎主路径已可用，22 个 API 在线，持久化部分接通
前端：4 个能用页面（脚手架），13 个公共组件待建，10+ 业务页面待建
身份：完全没做（SEC-001）
嵌入器：完全没做（B 类全部）
驾驶舱：完全没做（C 类全部）
```

### 1.2 12 PR 的设计原则

1. **底座优先** — 前 4 个 PR 解决"任何上层页面都依赖"的底座（设计 Token + 公共组件 + 路由 + 身份）
2. **剧本驱动** — 每个上层 PR 必须能让一个剧本跑通（S1-S6 中至少一个）
3. **可独立验证** — 每个 PR 自带 demo 数据 + Playwright 脚本
4. **可并行** — 同泳道的 PR 标明独占文件，避免冲突

### 1.3 12 PR 战略目标

| PR | 战略目标 | 解锁 |
|---|---|---|
| PR-01 ~ PR-04 | 底座完成 | 所有上层 PR 可开工 |
| PR-05 ~ PR-08 | A 配置工厂可演示 | 剧本 S5 + 路径模板创作可演示 |
| PR-09 ~ PR-10 | B 临床嵌入器可演示 | 剧本 S1+S4 可演示 |
| PR-11 ~ PR-12 | C 质控驾驶舱可演示 | 剧本 S2+S6 可演示 |

到 PR-12 完成时：**6 大剧本全部可演示**，客户验收基线达成 80%。

---

> ⚠️ **命名空间说明：** 12 PR 全部使用 `PR-V2-XX` 为权威主键，避免与历史 FE-XXX 编号冲突（如历史 FE-004 是"配置包中心" DONE，与 V2 路径模板列表无关）。任务台账以 PR-V2 编号登记。

- 独占文件冲突 → 后申请者必须改 PR 范围或等待

### 16.2 PR review 流程

1. AI 自检：所有 DoD 项目逐项打钩
2. 创建 `ai-dev-input/11_ai_reviews/pending/<review_id>.md`
3. 另一个 AI 或人类 review
4. 通过后才能 merge

### 16.3 功能验收记录

所有客户可见 PR（PR-05~PR-12）必须创建：

```
ai-dev-input/13_feature_acceptance/<feature_id>.md

含：
- 功能描述
- 对应剧本
- 验收等级（GOLD/SILVER/BRONZE/REJECTED）
- 验收截图
- Playwright 脚本路径
- 性能数据
- 已知问题
```

## 17. 常见陷阱（AI 必看）

| 陷阱 | 后果 | 防范 |
|---|---|---|
| 跳过 PR-01 直接做上层 | 视觉不一致、Token 散落 | PR-02 起强制依赖 PR-01 |
| 用 Ant Design 默认色 | 与品牌色冲突 | tokens.css 全局覆盖 |
| 实现"自由发挥"的页面 | 与规格书不一致 | 实现前必须读对应规格章节 |
| 跨多个 task 一个 PR | review 困难、回滚困难 | 单 PR 单 task |
| 不写 Playwright | 演示时翻车 | DoD 强制 |
| 缺来源直接发布 | 违反不变量 #8 | 后端强制 MISSING_SOURCE 阻断 |
| 拦截类规则随便用 | 违反不变量 #7 | 必须 action_mode=BLOCK 显式标记 |
| 写 ENGINE_AUDIT_LOG 漏字段 | 审计断链 | 用 AuditService 统一封装 |
| 前端硬编码角色判断 | 加新角色就崩 | 走 currentUser.permissions |
| WebSocket 不实现降级 | 网络抖动崩 | 必须有 HTTP polling 回退 |

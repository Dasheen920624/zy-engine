# 规则治理模块用户手册

## 1. 模块概述

规则治理模块帮助医院管理临床决策规则。通过本模块，您可以：

- 集中管理所有临床决策规则，包括用药提醒、检验预警、诊疗规范等
- 对规则进行新建、编辑、发布、停用和删除操作
- 在规则正式上线前进行试运行，验证规则效果
- 导入导出规则配置，方便在不同环境之间迁移

> **适用人员**：医院信息科管理人员、临床规则管理员

---

## 2. 快速入门

### 2.1 查看规则列表

1. 在左侧菜单中点击「规则治理」
2. 进入规则列表页面，默认显示所有已发布的规则

[截图：规则列表页面]

列表中每条规则显示以下信息：

| 列名 | 说明 |
|------|------|
| 规则名称 | 规则的显示名称 |
| 规则编码 | 系统自动生成的唯一编号 |
| 状态 | 草稿 / 已发布 / 已停用 |
| 类型 | 用药提醒 / 检验预警 / 诊疗规范等 |
| 更新时间 | 最近一次修改的时间 |
| 操作人 | 最近一次修改的操作人 |

### 2.2 搜索规则

1. 在规则列表页面上方的搜索框中输入关键词
2. 支持按规则名称、规则编码搜索
3. 可使用筛选条件按状态、类型、更新时间范围进行过滤

[截图：搜索与筛选区域]

### 2.3 查看规则详情

1. 在规则列表中点击规则名称
2. 进入规则详情页面，可查看规则的完整配置信息

[截图：规则详情页面]

详情页面包含以下标签页：

- **基本信息**：规则名称、编码、类型、描述、创建时间等
- **条件配置**：规则触发条件的详细设置
- **动作配置**：规则触发后执行的操作
- **版本历史**：规则的所有历史版本记录

## 3. 核心操作

### 3.1 新建规则

#### 步骤一：选择模板

1. 点击规则列表页面的「新建规则」按钮
2. 在弹出的模板选择窗口中，选择合适的规则模板
3. 模板会预置常用的条件和动作配置，减少手动设置的工作量

[截图：规则模板选择窗口]

> **提示**：如果没有合适的模板，可以选择「空白规则」从零开始配置。

#### 步骤二：填写条件

1. 进入规则编辑页面后，切换到「条件配置」标签
2. 点击「添加条件」按钮
3. 选择条件类型（如：患者年龄、诊断编码、检验结果等）
4. 设置条件参数（如：年龄 ≥ 65、诊断包含 I10 等）
5. 多个条件之间可设置「且」或「或」的逻辑关系

[截图：条件配置页面]

#### 步骤三：设置动作

1. 切换到「动作配置」标签
2. 点击「添加动作」按钮
3. 选择动作类型（如：弹窗提醒、发送消息、标记患者等）
4. 填写动作参数（如：提醒内容、消息接收人等）

[截图：动作配置页面]

#### 步骤四：保存草稿

1. 完成条件和动作配置后，点击「保存草稿」按钮
2. 系统会自动校验配置的完整性和正确性
3. 校验通过后，规则状态变为「草稿」

> **注意**：草稿状态的规则不会在实际临床环境中生效。

#### 步骤五：发布规则

1. 确认规则配置无误后，点击「发布」按钮
2. 系统会再次进行校验，确认规则可以正常执行
3. 校验通过后，规则状态变为「已发布」，开始在实际环境中生效

[截图：发布确认对话框]

### 3.2 编辑规则

1. 在规则列表中找到需要编辑的规则，点击「编辑」按钮
2. 进入规则编辑页面，修改条件或动作配置
3. 修改完成后，点击「保存草稿」保存修改
4. 确认无误后，点击「发布」使修改生效

> **说明**：编辑已发布的规则时，原规则在修改发布前仍然生效，不会中断临床使用。

### 3.3 删除规则

1. 在规则列表中找到需要删除的规则，点击「删除」按钮
2. 系统会弹出确认对话框，显示该规则的关联影响信息：
   - 当前有多少患者正在受该规则影响
   - 是否有其他规则依赖此规则
3. 确认影响范围后，点击「确认删除」

[截图：删除确认对话框]

> **警告**：删除操作不可恢复，请务必确认后再操作。建议先停用规则观察一段时间，确认无影响后再删除。

### 3.4 规则试运行

试运行功能允许您在规则正式发布前，使用真实患者数据验证规则效果，不影响实际临床流程。

#### 步骤一：选择患者数据

1. 在规则编辑页面，点击「试运行」按钮
2. 选择试运行的数据范围：
   - 按时间范围选择（如：最近 30 天的住院患者）
   - 按科室选择（如：心内科的所有患者）
   - 按患者列表选择（手动输入患者ID）

[截图：试运行数据选择页面]

#### 步骤二：执行试运行

1. 确认数据范围后，点击「开始执行」
2. 系统会在后台运行规则评估，页面显示执行进度
3. 执行时间取决于数据量，通常几分钟内完成

#### 步骤三：查看结果

1. 试运行完成后，系统自动跳转到结果页面
2. 结果页面显示：
   - 触发次数：规则在试运行数据中被触发的总次数
   - 触发患者数：被规则命中的患者人数
   - 触发率：触发次数占评估总数的比例
   - 详细记录：每条触发记录的详细信息

[截图：试运行结果页面]

### 3.5 规则导入导出

#### 导出规则

1. 在规则列表中，勾选需要导出的规则
2. 点击「导出」按钮
3. 系统会生成一个配置包（JSON 格式），自动下载到本地

#### 导入规则

1. 在规则列表页面，点击「导入」按钮
2. 选择本地的配置包文件
3. 系统自动进行校验，检查内容包括：
   - 文件格式是否正确
   - 规则编码是否与现有规则冲突
   - 条件和动作配置是否完整
4. 校验通过后，显示导入预览列表
5. 确认后点击「确认导入」

[截图：规则导入校验页面]

> **注意**：如果导入的规则编码与已有规则冲突，系统会提示您选择覆盖或跳过。

## 4. 高级功能

### 4.1 规则评估日志

规则评估日志记录了每条规则的历史评估情况，方便您追溯和审计。

1. 在左侧菜单中点击「评估日志」
2. 进入评估日志列表页面

[截图：评估日志列表页面]

日志列表显示以下信息：

| 列名 | 说明 |
|------|------|
| 评估时间 | 规则被评估的时间 |
| 规则名称 | 被评估的规则 |
| 患者ID | 被评估的患者 |
| 评估结果 | 命中 / 未命中 |
| 触发动作 | 命中后执行的动作 |

您可以通过筛选条件按时间范围、规则、患者等维度查看日志。

### 4.2 规则引擎状态

规则引擎状态页面展示规则引擎的运行情况。

1. 在左侧菜单中点击「引擎状态」
2. 进入引擎状态页面

[截图：引擎状态页面]

页面显示以下信息：

- **运行状态**：正常运行 / 异常
- **已加载规则数**：当前引擎中已加载的规则数量
- **今日评估次数**：当天规则评估的总次数
- **平均响应时间**：规则评估的平均耗时
- **异常告警**：引擎运行中的异常信息

## 5. 常见问题

### Q1：规则不生效怎么办？

请按以下步骤排查：

1. **检查规则状态**：确认规则状态为「已发布」，草稿状态的规则不会生效
2. **检查条件配置**：确认触发条件是否设置正确，逻辑关系（且/或）是否符合预期
3. **检查适用范围**：确认规则的适用科室、患者类型等范围设置是否正确
4. **检查引擎状态**：在「引擎状态」页面确认规则引擎是否正常运行
5. **查看评估日志**：在「评估日志」中搜索相关患者，查看规则是否被评估以及评估结果

如果以上步骤均未发现问题，请联系系统管理员。

### Q2：试运行结果怎么看？

试运行结果主要关注以下指标：

- **触发率过高**：可能条件设置过于宽泛，建议收窄条件范围
- **触发率过低**：可能条件设置过于严格，建议放宽部分条件
- **触发次数与触发患者数差异大**：说明部分患者被多次触发，建议检查是否有重复评估的情况

### Q3：导入失败怎么办？

导入失败通常有以下原因：

1. **文件格式错误**：请确认导入文件为系统导出的配置包，未被手动修改
2. **编码冲突**：导入的规则编码与现有规则重复，请选择覆盖或修改编码后重新导入
3. **配置不完整**：导入的规则缺少必要的条件或动作配置，请检查原文件
4. **版本不兼容**：导入文件的版本与当前系统版本不匹配，请联系技术支持

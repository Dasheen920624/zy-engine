# 安全治理模块用户手册

## 1. 模块概述

安全治理模块帮助医院管理信息系统安全。通过本模块，您可以：

- 查看和管理安全基线合规状态，确保系统满足等保要求
- 查看和导出审计日志，追踪系统操作记录
- 校验审计链完整性，确保日志未被篡改
- 管理系统用户和角色权限
- 管理数据加密密钥的版本和轮换
- 执行漏洞扫描并处理安全漏洞
- 配置单点登录（SSO）集成

> **适用人员**：医院信息科管理人员、系统安全管理员

---

## 2. 快速入门

### 2.1 查看安全基线

1. 在左侧菜单中点击「安全治理」
2. 默认进入安全基线概览页面

[截图：安全基线概览页面]

概览页面展示以下核心信息：

- **合规总览**：已满足 / 缺证据 / 需整改的项目数量
- **合规率**：当前安全基线的整体合规百分比
- **最近检查时间**：上次基线检查的时间
- **待整改项**：需要立即处理的安全问题列表

### 2.2 查看审计日志

1. 在左侧菜单中点击「审计日志」
2. 进入审计日志列表页面

[截图：审计日志列表页面]

日志列表显示以下信息：

| 列名 | 说明 |
|------|------|
| 操作时间 | 操作发生的时间 |
| 操作人 | 执行操作的用户 |
| 操作类型 | 登录 / 数据访问 / 配置变更 / 系统管理等 |
| 操作内容 | 操作的具体描述 |
| 操作结果 | 成功 / 失败 |
| IP 地址 | 操作来源的 IP 地址 |

## 3. 核心操作

### 3.1 安全基线概览

安全基线概览页面展示系统当前的安全合规状态。

#### 查看合规状态

合规状态分为三类：

| 状态 | 含义 | 颜色标识 |
|------|------|---------|
| 已满足 | 该项安全要求已合规 | 绿色 |
| 缺证据 | 该项可能已合规，但缺少证明材料 | 黄色 |
| 需整改 | 该项不合规，需要整改 | 红色 |

[截图：合规状态分布图]

#### 查看基线检查项

1. 在安全基线概览页面，点击各分类标签查看详细检查项
2. 检查项分类包括：
   - **身份认证**：密码策略、登录限制、多因素认证等
   - **访问控制**：权限管理、角色分配、最小权限原则等
   - **数据保护**：数据加密、脱敏、备份等
   - **网络安全**：防火墙、TLS 配置、网络隔离等
   - **审计追踪**：日志记录、日志保护、审计链等
   - **系统安全**：漏洞管理、补丁更新、安全配置等

3. 点击单个检查项可查看详细信息和整改建议

[截图：基线检查项详情页面]

### 3.2 审计日志

#### 查看操作记录

1. 进入审计日志列表页面
2. 默认显示最近的操作记录，按时间倒序排列
3. 支持分页浏览历史记录

#### 筛选条件

使用页面上方的筛选区域精确定位日志：

- **时间范围**：选择起止时间
- **操作人**：输入用户名或从下拉列表选择
- **操作类型**：选择操作类型分类
- **操作结果**：成功 / 失败 / 全部
- **关键词**：在操作内容中搜索关键词

[截图：审计日志筛选区域]

> **提示**：多个筛选条件之间为"且"的关系，即同时满足所有条件。

#### 导出日志

1. 设置好筛选条件后，点击「导出」按钮
2. 选择导出格式（CSV / Excel）
3. 选择导出范围：
   - 当前筛选结果：导出符合筛选条件的所有记录
   - 当前页：仅导出当前页面显示的记录
4. 点击「确认导出」

[截图：日志导出选项页面]

> **注意**：导出大量日志可能需要较长时间，建议缩小时间范围后导出。

### 3.3 审计链校验

审计链校验用于验证审计日志的完整性，确保日志未被篡改。

1. 在左侧菜单中点击「审计链校验」
2. 进入审计链校验页面
3. 点击「一键校验」按钮
4. 系统自动对所有审计日志进行完整性校验
5. 校验完成后显示结果

[截图：审计链校验页面]

校验结果包括：

- **校验状态**：通过 / 未通过
- **校验范围**：校验的时间范围和日志条数
- **异常记录**：如存在被篡改的日志，列出具体记录
- **校验耗时**：校验过程花费的时间

> **说明**：建议定期执行审计链校验（如每周一次），确保日志完整性。

### 3.4 用户管理

#### 创建用户

1. 在左侧菜单中点击「用户管理」
2. 点击「新建用户」按钮
3. 填写用户信息：
   - **用户名**：登录账号，创建后不可修改
   - **姓名**：用户真实姓名
   - **手机号**：用于接收通知和验证
   - **邮箱**：用于接收系统邮件
   - **初始密码**：设置登录密码（需符合密码策略）
4. 点击「保存」

[截图：新建用户页面]

> **提示**：用户首次登录时系统会要求修改初始密码。

#### 分配角色

1. 在用户列表中，点击用户行的「角色」按钮
2. 在角色分配页面，勾选需要分配的角色
3. 角色说明：

| 角色 | 权限范围 |
|------|---------|
| 系统管理员 | 所有模块的完整权限 |
| 安全管理员 | 安全治理模块的完整权限 |
| 规则管理员 | 规则治理模块的完整权限 |
| 路径管理员 | 路径治理模块的完整权限 |
| 质控管理员 | 质控治理模块的完整权限 |
| 只读用户 | 所有模块的查看权限 |

4. 点击「确认分配」

[截图：角色分配页面]

#### 重置密码

1. 在用户列表中，点击用户行的「重置密码」按钮
2. 系统生成临时密码
3. 可选择通过手机短信或邮箱将临时密码发送给用户
4. 用户使用临时密码登录后需立即修改密码

#### 强制下线

当发现用户账号异常时，可强制该用户下线。

1. 在用户列表中，点击用户行的「强制下线」按钮
2. 系统立即终止该用户的所有会话
3. 该用户需要重新登录才能继续使用系统

> **警告**：强制下线会中断用户当前的所有操作，请谨慎使用。

### 3.5 密钥管理

#### 查看密钥版本

1. 在左侧菜单中点击「密钥管理」
2. 进入密钥列表页面，显示所有密钥及其版本信息

[截图：密钥列表页面]

每个密钥显示以下信息：

| 列名 | 说明 |
|------|------|
| 密钥名称 | 密钥的标识名称 |
| 用途 | 数据加密 / 签名 / 令牌等 |
| 当前版本 | 正在使用的密钥版本号 |
| 创建时间 | 密钥的创建时间 |
| 最近轮换时间 | 上次轮换的时间 |
| 状态 | 正常 / 轮换中 / 已停用 |

#### 密钥轮换

密钥轮换是更换加密密钥的安全操作，定期轮换可降低密钥泄露风险。

1. 在密钥列表中，点击需要轮换的密钥行的「轮换」按钮
2. 系统显示轮换确认信息：
   - 当前密钥版本
   - 轮换后的新版本
   - 影响范围说明
3. 确认后点击「执行轮换」
4. 系统自动完成以下操作：
   - 生成新版本密钥
   - 将新密钥设为当前使用版本
   - 旧密钥保留用于解密历史数据
5. 轮换完成后，密钥版本号自动递增

[截图：密钥轮换确认页面]

> **说明**：密钥轮换不会影响已有加密数据的解密，系统会自动使用对应版本的密钥进行解密。

#### 查看轮换历史

1. 在密钥列表中，点击密钥行的「轮换历史」按钮
2. 进入轮换历史页面，显示该密钥的所有轮换记录

[截图：密钥轮换历史页面]

每条轮换记录包含：

- 轮换时间
- 操作人
- 旧版本号
- 新版本号
- 轮换原因

### 3.6 漏洞扫描

#### 执行扫描

1. 在左侧菜单中点击「漏洞扫描」
2. 进入漏洞扫描页面
3. 点击「执行扫描」按钮
4. 选择扫描范围：
   - 全量扫描：扫描所有系统组件
   - 增量扫描：仅扫描最近变更的组件
5. 确认后点击「开始扫描」
6. 扫描过程中页面显示进度

[截图：漏洞扫描执行页面]

> **说明**：全量扫描可能需要较长时间（通常 30 分钟至数小时），建议在业务低峰期执行。

#### 查看结果

1. 扫描完成后，系统自动展示扫描结果
2. 结果按漏洞等级分类：

| 等级 | 说明 | 颜色标识 |
|------|------|---------|
| 严重 | 可被远程利用，影响系统安全 | 红色 |
| 高危 | 存在被利用风险，需尽快修复 | 橙色 |
| 中危 | 有一定风险，建议修复 | 黄色 |
| 低危 | 风险较低，可择机修复 | 蓝色 |

3. 点击漏洞可查看详细信息，包括漏洞描述、影响范围、修复建议等

[截图：漏洞扫描结果页面]

#### 处理漏洞

1. 在漏洞详情页面，点击「处理」按钮
2. 选择处理方式：
   - **标记修复**：已修复该漏洞，需填写修复说明
   - **标记忽略**：评估后认为无需修复，需填写忽略原因
   - **分配处理人**：将漏洞分配给其他人员处理
3. 填写处理说明
4. 点击「确认」

[截图：漏洞处理页面]

## 4. 高级功能

### 4.1 SSO 单点登录配置

单点登录（SSO）允许用户使用医院统一身份认证系统登录本系统，无需单独记忆账号密码。

#### 配置 SSO 提供商

1. 在左侧菜单中点击「SSO 配置」
2. 点击「添加提供商」按钮
3. 填写提供商信息：
   - **提供商名称**：如"医院统一认证平台"
   - **协议类型**：SAML 2.0 / OIDC
   - **认证地址**：SSO 服务器的认证 URL
   - **令牌地址**：获取令牌的 URL（OIDC 适用）
   - **客户端 ID**：系统在 SSO 提供商中注册的标识
   - **客户端密钥**：系统在 SSO 提供商中注册的密钥
   - **证书/公钥**：用于验证 SSO 令牌的公钥

[截图：SSO 提供商配置页面]

#### 测试连接

1. 配置完成后，点击「测试连接」按钮
2. 系统尝试与 SSO 提供商建立连接
3. 测试结果：
   - **连接成功**：显示用户信息映射测试结果
   - **连接失败**：显示错误原因和排查建议

> **重要**：启用 SSO 前务必完成连接测试，确保配置正确。

#### 启用 SSO

1. 连接测试通过后，点击「启用」按钮
2. 系统弹出确认对话框，提示启用后的影响：
   - 用户将跳转至 SSO 页面登录
   - 本地密码登录将被禁用
   - 建议保留至少一个管理员账号使用本地登录
3. 确认后点击「启用 SSO」

[截图：SSO 启用确认对话框]

### 4.2 安全基线详情

安全基线详情页面展示各项安全配置的具体信息。

1. 在安全基线概览页面，点击检查项名称
2. 进入检查项详情页面

[截图：安全基线详情页面]

#### 算法配置

展示系统使用的加密算法配置：

- **对称加密算法**：如 AES-256-GCM
- **非对称加密算法**：如 RSA-2048、SM2
- **哈希算法**：如 SHA-256、SM3
- **密钥交换算法**：如 ECDH

> **说明**：系统支持国密算法（SM2/SM3/SM4），满足国产化要求。

#### TLS 版本

展示系统支持的 TLS 协议版本：

- **TLS 1.2**：推荐使用
- **TLS 1.3**：最新版本，安全性更高
- **TLS 1.0 / 1.1**：已不安全，应禁用

系统会标记当前启用的 TLS 版本，并对不安全的版本给出整改建议。

#### HSTS 配

展示 HTTP 严格传输安全（HSTS）的配置状态：

- **是否启用**：HSTS 是否已开启
- **Max-Age**：HSTS 策略的有效期
- **Include SubDomains**：是否包含子域名
- **Preload**：是否加入浏览器预加载列表

## 5. 常见问题

### Q1：用户锁定怎么办？

用户连续输错密码超过设定次数后，账号会被自动锁定。

**解锁方法**：

1. 使用管理员账号登录系统
2. 进入「用户管理」页面
3. 找到被锁定的用户，点击「解锁」按钮
4. 用户解锁后可重新登录

**预防措施**：

- 提醒用户使用密码找回功能，而非反复尝试
- 检查是否有异常登录尝试，可能存在安全风险
- 如频繁出现锁定，建议为该用户配置 SSO 登录

### Q2：密钥轮换影响什么？

密钥轮换会影响以下方面：

- **数据加密**：新产生的数据使用新密钥加密，历史数据仍使用旧密钥解密，不影响业务
- **令牌签发**：新签发的令牌使用新密钥签名，已签发的令牌在有效期内仍可验证
- **系统性能**：轮换过程中可能有短暂的性能波动，通常在秒级内恢复
- **备份恢复**：如需恢复历史数据，需确保旧版本密钥未被删除

> **建议**：密钥轮换建议在业务低峰期执行，并提前通知相关人员。

### Q3：审计链校验失败怎么办？

审计链校验失败说明可能存在日志被篡改的情况。

**排查步骤**：

1. **查看异常记录**：在校验结果中查看具体哪些日志记录异常
2. **确认操作人**：检查异常记录对应的操作人和操作内容
3. **排查原因**：
   - 系统故障导致日志损坏（较少见）
   - 人为篡改日志（需高度重视）
4. **处理措施**：
   - 如为系统故障，联系技术支持修复
   - 如怀疑人为篡改，立即启动安全应急流程
   - 保留校验结果作为证据

> **重要**：审计链校验失败属于安全事件，应按照医院信息安全应急预案处理。

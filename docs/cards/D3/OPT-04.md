# OPT-04 · 临床安全案例与红线规则库

> 读卡前置：[核心 CONSTITUTION](../../CONSTITUTION.md) + [D3 域简报](_brief.md)。
> 迁移来源（覆盖矩阵锚点）：核心价值排序"临床安全（OPT-04 + D3）"第一 · 详规 S8 临床安全 · 详规 §临床红线。

## 身份
- 卡 ID：OPT-04（引擎卡；临床安全红线库单一归属）
- 域：D3 临床运行
- 关联场景：S8 临床嵌入运行 · S16 辅助诊疗
- 依赖卡：[RULE-01](../D2/RULE-01.md) 规则引擎（建于其上）· [CDSS-01](CDSS-01.md) 命中 · [OPT-03](OPT-03.md) 风险分级 · [MED-C3](MED-C3.md) 安全撤回 · [SYS-08](../D2/SYS-08.md) 权威版本
- 工作量：7d
- owner / reviewer：待派单（owner ≠ reviewer）

## 目标
建立**临床安全红线规则库**：DDI（药物相互作用）/ 危急值 / 剂量上限 / 抗菌药物限制 / 特殊人群禁忌等红线，配套**危害分析 + 静默试运行**，红线命中**强优先、不可被疲劳抑制、安全级别下级不可关**。这是全系统**价值排序第一**的临床安全防线。

## 现状（搬迁时核查 2026-05-30，以 `medkernel-backend` 为准）
建于 D2 规则引擎之上：`engine/rule/` 已有 `RuleDslEvaluator`/`RuleDslEvaluation`（[RULE-01](../D2/RULE-01.md) 归属）。本卡＝在其上新建**红线规则类目 + 危害分析 + 静默试运行**框架，红线作为受保护的特殊规则集（不可被下级静默关闭），非重造规则引擎。

## 功能要求（原子可测条目）
- [ ] FR-1 红线类目：DDI/危急值/剂量上限/抗菌限制/特殊人群禁忌为受控红线类目，内容版本化（[SYS-08](../D2/SYS-08.md)）。
- [ ] FR-2 强优先：红线命中卡强优先、**不可被疲劳抑制**（[CDSS-01](CDSS-01.md) FR-5）。
- [ ] FR-3 危害分析：每条红线带危害严重度/证据/来源，绑定 [OPT-03](OPT-03.md) 分级。
- [ ] FR-4 静默试运行：红线上线前静默跑（只记录不提醒）达标后才正式生效。
- [ ] FR-5 下级不可关：安全红线下级租户不可静默关闭（核心 §9 安全红线不可下沉关闭）。
- [ ] FR-6 不写死：红线内容来自配置/知识库、**不写死医学常量在代码**。

## 接口契约 / 页面契约
### 接口契约（引擎/API 卡）
- 端点：`GET /api/v1/engine/safety/redlines` · `POST .../safety/redlines:dry-run`（静默试运行）· `POST .../safety/redlines:promote`（达标上线）

## 视角清单（11 视角逐条）
1. 产品架构：★全系统临床安全的最高防线。
2. 产品体验：红线提醒强可见、不可忽略（[REMIND-01](REMIND-01.md)）。
3. 系统与数据架构：红线命中 P95 ≤500ms；静默试运行可大规模回放。
4. 临床医疗安全：★DDI/危急值/剂量/禁忌红线；命中强优先、不可抑制、不可下级关。
5. 知识与数据治理：★红线内容版本化、可追溯证据来源（[SYS-08](../D2/SYS-08.md)）；不写死常量。
6. 安全合规与监管：危害分析 + 静默试运行 + 召回链（[MED-C3](MED-C3.md)）全审计（[BASE-04](../D0/BASE-04.md)）。
7. 集团化与多租户治理：★安全红线平台统一、下级不可静默关闭（核心 §9）。
8. 集成与互操作：红线随 CDSS 经 [API-07](API-07.md)/[EMBED-01](EMBED-01.md) 输出。
9. 运维 / SRE / 国产化：静默试运行可观测；红线热更可灰度。
10. 质量与真实性审计：★无写死医学常量、红线命中可复现、不可绕。
11. AI / 模型治理与可降级：★红线纯确定性（B0），绝不依赖模型；关模型红线照常命中。

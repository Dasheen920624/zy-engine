## 目标
提供**院内字典 ↔ 标准字典映射**引擎：ICD-10 / ICD-9-CM-3 / 药品本位码 / LOINC 等标准与院内本地术语的**候选 → 冲突 → 审核 → 发布**闭环 + **医学语义匹配**辅助 + **高危近似负样本判别器**（MED-C1）。核心安全约束：**形近/义近但临床后果迥异**的映射（钾/钠、肌钙蛋白 T/I、左/右、剂量量级）**强制标 HIGH、禁批量确认、禁自动确认**，必须人工逐条复核。

## 现状（搬迁时核查 2026-05-30，以 `medkernel-backend/src` 为准）
`engine/terminology` **映射基础设施已建**，本卡＝**补语义匹配 + 高危判别 + 发布版本化**：
- 已有：`LocalTerm`/`StandardTerm`、`TermMapping`、`MappingCandidate`、`MappingConflict`、`TermMappingPackage`(+`Item`/`Release`/各 Repository)、`TerminologyService`/`TerminologyController`、`TerminologyEnums`/`Filters`/`Requests`。
- 缺口（本卡补）：① **医学语义匹配**辅助候选（同义词/缩写/编码族，确定性词典 + 规则，非模型）；② **高危近似负样本判别器**（MED-C1：强制 HIGH + 禁批量/禁自动确认）；③ 映射包**版本与发布**复用 [SYS-04](SYS-04.md)（灰度/全量/回滚）；④ 标准字典来源接 [OPT-07](OPT-07.md) 分级。

## 功能要求（原子可测条目）
- [ ] **FR-1 标准/院内字典**：维护 ICD-10 / ICD-9-CM-3 / 药品本位码 / LOINC 标准字典 + 院内 `LocalTerm`；标准字典来源可溯、版本化。
- [ ] **FR-2 候选 + 语义匹配**：给定院内术语 → 生成标准候选（精确码 + 同义词/缩写/编码族**确定性**匹配打分）；候选可解释（命中依据）。
- [ ] **FR-3 高危近似判别（MED-C1）**：候选命中**高危近似规则集**（钾 K⁺/钠 Na⁺、肌钙蛋白 T/I、左/右、剂量量级 10×、胰岛素单位 U/mL）→ **强制 `risk=HIGH`**、**禁批量确认**、**禁自动确认**、必须人工逐条复核 + 二次确认。

- [ ] **FR-5 映射包发布**：映射成 `TermMappingPackage` 经 [SYS-04](SYS-04.md) **草稿→审核→灰度→全量→回滚**发布；已发布映射不可原地改，只能新版。
- [ ] **FR-6 诚实降级**：无语义词典/无模型时仍可**精确码匹配 + 人工映射**（B0）；语义辅助关闭返回确定性候选，不伪造智能。

## 视角清单（11 视角逐条）
1. **产品架构**：院内 ↔ 标准术语的统一映射底座，供规则/路径/质控/上报消费标准码。
2. **产品体验**：N·A —— 字典映射页（D2）呈现；高危行强制逐条、禁批量。
3. **系统与数据架构**：候选打分确定性可解释；大字典列表走 [API-13](../D0/API-13.md) 分页；映射包版本化。
4. **临床医疗安全**：★主战场 —— 高危近似映射强制 HIGH + 禁批量/禁自动（错映=临床误判），人工二次确认（铁律 #2）。
5. **知识与数据治理**：标准字典来源可溯分级（[OPT-07](OPT-07.md)）；映射替代非覆盖、版本可回滚。
6. **安全合规与监管**：映射/审核/发布留审计；高危确认留操作人 + 时点 + 依据。
7. **集团化与多租户治理**：标准字典集团统一、院内映射本地化，经 [SYS-04](SYS-04.md) 继承覆盖。
8. **集成与互操作**：外部系统编码经映射归一进标准上下文（[API-01](API-01.md)/[INTEG-01](INTEG-01.md)）。
9. **运维 / SRE / 国产化**：5 方言；标准字典离线包导入；国产编码（药品本位码）一等支持。
10. **质量与真实性审计**：语义打分真实、无伪造"已映射"；高危批量/自动确认 CI + 运行期双拒（铁律 #1/#2）。
11. **AI / 模型治理与可降级**：★语义匹配 **B0＝确定性词典/规则**；模型增强语义（第二波）只提候选、**绝不自动确认高危**，关模型退回精确码 + 人工。

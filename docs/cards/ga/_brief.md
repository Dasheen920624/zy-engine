# GA 总验收 · 域简报

> 读卡前置：先读 [核心 CONSTITUTION](../../CONSTITUTION.md)，再读本简报，再读你领的那张卡。
> 本域所有卡共享的上下文放这里，**不在卡间复制**。冲突裁决：核心 > 本简报 > 卡。
> 准入：**第一波（D0–D6）+ 第二波（wave2 AI 加深）全部完成后**才跑 GA 总验收（核心 §0）。GA 卡（验收规格）现在可建，但**实际 GA pass 待 wave2 完成**——部分门禁依赖 wave2（AIK-STD / 15 领域门面 / KNOWGEN-15）。

## 域目标

把**跨域总验收**做成可执行规格：12 张验收卡（QA-01…08 + DEGRADE-01 + SYS-07 + INFRA-07 + INFRA-10）+ **14 条 GA 门禁**，把全系统（D0–D6 核心 + wave2 AI）的**真实性、跨引擎 E2E、医疗安全、B0 降级、非功能基线、产品体验、第三方对接、代码净化**编排为"全部满足才能宣告 **v1.0 GA**"的最终闸门。GA 不新增业务能力，只**验证**已建能力**真实、可降级、可举证**。

## 现状（搬迁时核查 2026-05-30；以仓库真实门禁/测试为准）

验收基建**已有**，GA 卡是"把现有门禁 + 剧本编排为总验收闭环"而非从零：

- **前端真实性门禁**＝`frontend/eslint-rules/`：`no-page-mock.js` + `no-inline-style.js` + `forbid-deprecated-naming.js` + `eslint.config.js`（`medkernel/*` 规则）—— [INFRA-01](../D0/INFRA-01.md) 归属；[QA-07](QA-07.md) 验收其全绿。
- **前端 E2E**＝`frontend/e2e/`——[INFRA-07](INFRA-07.md)（全页面可打开性）/ [INFRA-10](INFRA-10.md)（A1-A9 剧本）承接。
- **后端真实性门禁/契约门禁**＝迁移契约门禁（`docs/superpowers/specs/2026-05-26-migration-contract-gate-*`）+ 安全/契约测试——[INFRA-02](../D0/INFRA-02.md) 归属。
- **模型网关 / 降级**＝`engine/llm/` `ModelGatewayService/Controller`——[DEGRADE-01](DEGRADE-01.md)/[QA-04](QA-04.md) 验收"关模型/Dify/图/外部"四类降级；模型真实接入是 wave2（X-LLM）。
- **临床安全**＝[OPT-04](../D3/OPT-04.md) 红线 + [MED-C3](../D3/MED-C3.md) 旧版隔离 + [OPT-03](../D3/OPT-03.md) 风险分级——[QA-03](QA-03.md) 验收。
- **缺口/依赖 wave2**：AI 工厂无模型可运行组（AIK-STD-01~12）、15 领域门面、KNOWGEN-15 首发知识包——GA 门禁第 3/8/10 条依赖，**待 wave2 建成后才能跑通**；GA 卡现在定义验收规格、占位等待。

## 登入 / 使用角色（13 角色矩阵本域子集，全量见 [质量基线 §9](../../audit/质量基线.md)）

| 角色 | 在 GA 主要干什么 |
|---|---|
| QA / 测试 | 跑 E2E、降级链、非功能基线、可打开性、A1-A9 剧本，出证据 |
| AI 团队 | 无模型可运行验收、AI 标识、审核台真实可审可发 |
| 实施 implementation | 第三方对接断连/重试/降级证据、首发知识包同步试点 |
| 架构师 architect | 跨引擎 E2E、架构边界、投影非权威核验 |
| 合规 compliance | 证据/审计/隐私/等保门禁、四方签字 |
| 业务 / 院方 | GA 上线评审四方签字（业务+架构+实施+合规） |

> GA 是**只读验收**：不改业务数据，只跑用例 + 收证据 + 评审签字。**证据优先——空口不算通过**（核心 §13 / 铁律 #1）。

## 共享数据模型 / 实体（GA 卡共用，单一归属在此声明、卡内只引用）

- **验收证据**：GA 全部用例产出的证据（截图/traceId/导出包）复用 [EVID-01](../D5/EVID-01.md) 证据链 + [SYS-06](../D5/SYS-06.md) 证据框架，**不另造证据机制**。
- **真实性门禁（T-GATE）**：前端 [INFRA-01](../D0/INFRA-01.md) + 后端 [INFRA-02](../D0/INFRA-02.md) + no-page-mock（[INFRA-09](../D1/INFRA-09.md)）单一归属在各自卡；GA 卡只**调用/验收**其全绿。
- **A1-A9 验收剧本**：全功能验收剧本（详规 §16.2）是 GA 的统一脚本基准；[INFRA-10](INFRA-10.md) 承接自动化。
- **14 条 GA 门禁**：见 backlog「GA 门禁」节，是宣告 v1.0 GA 的硬条件清单；各卡映射到对应门禁条目。
- **降级矩阵**：模型/Dify/图投影/外部 四类降级单一归属 [DEGRADE-01](DEGRADE-01.md)；[QA-04](QA-04.md) 聚焦 B0（无模型/无 Dify/无图）。

## 依赖

- **上游（全系统）**：D0–D6 全部卡 + wave2 全部卡（GA 验收它们）。关键门禁卡：[INFRA-01](../D0/INFRA-01.md)/[INFRA-02](../D0/INFRA-02.md)/[INFRA-09](../D1/INFRA-09.md) 真实性 · [OPT-04](../D3/OPT-04.md)/[MED-C3](../D3/MED-C3.md) 临床安全 · [EVID-01](../D5/EVID-01.md)/[SYS-06](../D5/SYS-06.md) 证据 · [SYS-04](../D2/SYS-04.md)/[SYS-08](../D2/SYS-08.md) 版本权威 · [SVC-INTEGRATION-01](../D2/SVC-INTEGRATION-01.md) 第三方。
- **下游**：无（GA 是终点，通过即宣告 v1.0 GA）。

## 本域最烫的不变量（点核心编号 + 为何在本域最关键）

- **核心 §13 真实性 + 铁律 #1/#2**：★GA 的全部存在理由——**证据优先**，每条验收必须有可举证证据（traceId/截图/导出），无假闭环、无伪造、无 `eslint-disable medkernel/*`。
- **核心 §11 B0 先于模型 + 降级**：★关模型/Dify/图/外部四类逐项关闭，主链路仍真实可运行（[QA-04](QA-04.md)/[DEGRADE-01](DEGRADE-01.md)）。
- **临床安全（价值第一）**：AI 标识 + 医师确认 + 红线 + 旧版隔离 + 高风险审核（[QA-03](QA-03.md)）全部通过才算安全达标。
- **非功能基线**：性能 P95 + 可用 ≥99.9% + 并发幂等 + 5 方言一致（[SYS-07](SYS-07.md)）。
- **可打开性**：★每个页面登录对应角色后真实可打开、无 Hook 错误/null/404（[INFRA-07](INFRA-07.md)）——防"卡建了页打不开"。
- **四方签字**：业务 + 架构 + 实施 + 合规签字才宣告 GA（门禁 14）。

## 域级验收（GA = 终点本身）

GA 全部卡（12 项）`done` 且 **backlog「GA 门禁」14 条全部满足**，方可宣告 **v1.0 GA**：

1. 真实性：T-GATE 前后端全绿、无 `eslint-disable medkernel/*`、§0.2 铁律 0 违反（[QA-07](QA-07.md)）；
2. 跨引擎 E2E：源→标准化→规则/路径→CDSS→评估→随访→包→嵌入→证据 真跨引擎全链路 + A1-A9 剧本（[QA-01](QA-01.md)/[INFRA-10](INFRA-10.md)）；
3. 医疗安全：AI 标识 + 医师确认 + 禁忌红线 + 旧版隔离 + 高风险审核 + 高危近似禁批量（[QA-03](QA-03.md)）；
4. B0 / 降级：无模型 / 无 Dify / 无图 / 断网 五组双向通过（[QA-04](QA-04.md)/[DEGRADE-01](DEGRADE-01.md)）；
5. 非功能：性能 P95 + 可用 ≥99.9% + 并发幂等 + 5 方言一致 + 备份恢复 + 国产化自检（[SYS-07](SYS-07.md)/[QA-02](QA-02.md)）；
6. 可打开性 + 体验：全页面可打开无错 + 10 万级列表 + 低打扰 + 六态 + 可信解释 + 证据导出 + 驾驶舱下钻（[INFRA-07](INFRA-07.md)/[QA-06](QA-06.md)）；
7. 第三方：HIS/EMR/LIS/PACS/手麻/输血/医保/公卫/区域平台/Provider 断连重试降级证据（[QA-08](QA-08.md)）；
8. 上线评审：全能力上线评审 + 四方签字（[QA-05](QA-05.md) + 门禁 14）。

**14 条门禁有依赖 wave2 的项（AIK-STD / 15 领域门面 / KNOWGEN-15）；GA 实际宣告须待 wave2 建成。** GA 卡本期只搬规格、定义"怎么验"。

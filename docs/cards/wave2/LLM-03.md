## 目标
模型**外调前的数据最小化与安全**：字段白名单 + 脱敏 + 审批 + 阻断 + 证据——出域数据只含必需、已脱敏、经批准、可追证。

## 现状（搬迁时核查 2026-05-31）
**部分**：[LLM-01](LLM-01.md) `ModelGatewayService.desensitize()` 已有手机/身份证/银行卡/邮箱 + `MASK_ALL` 姓名/病历号正则脱敏（DEFAULT/MASK_ALL/NONE）。本卡＝补**字段白名单（出域允许字段清单）+ 外调审批流 + 超限阻断 + 出域证据**；策略中枢与 [OPT-09](OPT-09.md) 同源、本卡为网关侧执行点。

## 功能要求（原子可测条目）
- [ ] FR-1 字段白名单：外调 payload 仅含能力码声明的允许字段，其余剥离。
- [ ] FR-2 脱敏强制：白名单字段按策略脱敏后才出域（复用 LLM-01 脱敏 + 扩展）。
- [ ] FR-3 审批：B2 外调按敏感级触发审批（高敏需人工批准才发）。
- [ ] FR-4 阻断：超白名单/未脱敏/未审批 → 阻断 + 诚实报错，不静默放行。
- [ ] FR-5 证据：每次外调留出域字段清单 + 脱敏后 hash + 审批人（[SYS-06](../D5/SYS-06.md)/[EVID-01](../D5/EVID-01.md)）。

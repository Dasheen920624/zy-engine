## 目标
定义并落地**插件和生态安全边界**：第三方插件**只读优先、受控写入需门禁**（权限 + 审计 + 数据主权 + 临床安全），插件**不得绕引擎直写权威数据、不得作权威源**，越界即拒。

## 现状（搬迁时核查 2026-05-30，以 `medkernel-backend` 为准）
基本待建：现有无插件安全边界框架。本卡＝新建插件能力声明 + 权限/门禁 + 受控写入审批 + 审计，建在引擎域边界（[SYS-02](../D0/SYS-02.md)）与权限（[BASE-02](../D0/BASE-02.md)）之上。

## 功能要求（原子可测条目）
- [ ] FR-1 能力声明：插件声明所需能力（读/写/调用），按最小权限授予。
- [ ] FR-2 只读优先：默认只读；写入须显式门禁（权限 + 审批）。
- [ ] FR-3 受控写入：受控写入只经引擎契约（[SYS-02](../D0/SYS-02.md)），不绕引擎直写权威库。
- [ ] FR-4 临床安全门禁：触临床的插件过 [OPT-04](../D3/OPT-04.md) 红线 + [OPT-03](../D3/OPT-03.md) 风险分级。
- [ ] FR-5 数据主权 + 审计：插件数据访问按数据主权约束、全审计（[BASE-04](../D0/BASE-04.md)）；越界拒绝。

## 接口契约 / 页面契约
### 接口契约（引擎/API 卡）
- 端点：`POST /api/v1/plugins/register`（能力声明）· `POST .../plugins/{id}/grants`（授权门禁）· 插件调用经网关校验
- DTO：插件能力/授权 Record；信封 `ApiResult`/`ProblemDetail`
- 状态机：配置类（注册→待审→已授权/已禁用）；trace（[OBS-01](../D0/OBS-01.md)）
- 幂等 / 错误码：越界拒绝明确错误码

## 数据与迁移
- 表族：`plugin` / `plugin_grant`（能力 + 授权 + 状态 + 组织字段 + 审计）；五方言（[BASE-05](../D0/BASE-05.md)）

## 视角清单（11 视角逐条）
1. 产品架构：生态扩展的"安全闸门"。
2. 产品体验：N·A（技术契约；管理在 [DEVCON-01](DEVCON-01.md)）。
3. 系统与数据架构：插件调用校验 O(1)；不影响主链路性能。
4. 临床医疗安全：★触临床插件过红线/分级门禁、不绕引擎。
5. 知识与数据治理：插件不可作权威源、不污染权威数据。
6. 安全合规与监管：★权限 + 审计 + 数据主权；越界拒绝留痕。
7. 集团化与多租户治理：插件授权按租户隔离。
8. 集成与互操作：★只经引擎契约（[SYS-02](../D0/SYS-02.md)），受控集成。
9. 运维 / SRE / 国产化：插件可禁用/熔断。
10. 质量与真实性审计：★只读优先、受控写入有据、不绕引擎、不作权威。
11. AI / 模型治理与可降级：AI 类插件受同等边界约束。

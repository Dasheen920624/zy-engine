## 域目标

登入后**第一屏真实可用**：让不同角色 10 秒内知道"今天要处理什么、系统是否安全、试点是否可验收"（详规 S0）。D1 只做**聚合呈现**：把系统健康、知识同步、待审核、临床运行、质控整改、价值指标等**各域只读摘要**聚合到工作台一页，按 13 角色五维 RBAC 呈现默认视图，六态完整、指标可下钻；并把残留的演示页清出生产路由（真实性）。

工作台**不拥有任何业务数据**，只做只读聚合呈现；权威源仍在各域关系库（核心 §7 / 铁律#5）。**接口口径（详规 S0 锁定）：不新增独立工作台接口**——复用现有"系统健康 / 审计 / 任务待办 / 大规模列表 / 各引擎状态"API；专门的"工作台聚合 API"后移（后续 E6 / 第二波）。D1 时多数业务摘要源（待办/质控/随访）尚在后续域（D2–D5）未建，故 D1 对**未建域 / 未连接依赖 / 关模型**一律**诚实降级**（"暂无数据 / 该域未启用 / 未连接 / 智能建议暂不可用"），绝不伪造健康灯、待办数、风险条，更不得让前端去调一个**不存在的后端端点**白屏（见现状）。

## 登入 / 使用角色（13 角色矩阵的本域子集，全量矩阵见 [质量基线 §9](../../audit/质量基线.md)）

工作台是**全 13 角色登入后的统一第一落点**，但每个角色默认视图不同（五维 RBAC 数据维 + 默认角色视图）：

| 角色 | 在工作台主要看什么 |
|---|---|
| 平台管理员 platform-admin | 租户开通/生命周期阶段、许可证、全局标准包、平台级健康 |
| 医院管理员 hospital-admin | 本院实施/演示就绪度、用户与发布审批待办、院内健康 |
| 信息科 it-ops | 系统健康灯、Provider/适配器连通、国产化自检、会话与安全告警 |
| 合规审计 audit-compliance | 审计异常、安全基线偏差、证据导出待办（只读） |
| 院长 / 医务处 | 价值指标、质控整改概览（不出现技术词，详规 S0 验收） |
| 临床等业务角色 | 我的待办、与我相关的提醒/风险、最近变化（D1 多为空态/降级，随 D2–D5 上线回灌） |

> 角色 → 默认视图映射由 [WORKBENCH-01](WORKBENCH-01.md) 前端落点，数据维过滤由各**现有来源 API** 按 `OrgContext.roleSet` 执行（[INFRA-05](../D0/INFRA-05.md)）；**前端不写死角色逻辑、不前端兜底假数**。

## 共享上下文（D1 卡共用，单一归属在此声明、卡内只引用）

- **聚合 = 只读组合，非新权威实体**：工作台不建任何权威表、不引入"工作台快照"权威源；各主卡片**各自**消费其所属域的现有只读 API（系统健康←[OBS-01](../D0/OBS-01.md)/运行底座 [SYS-05](../D0/SYS-05.md)、待办/最近变化←审计 [BASE-04](../D0/BASE-04.md) 与各域、大列表←[API-13](../D0/API-13.md)、各引擎状态←各引擎现有端点）。每区块**独立取数、独立加载、独立降级**，单源慢/挂只降级该卡片，不拖垮首屏、不整页 404。
- **作用域载体**：聚合一律经 D0 的 `OrgContext`（租户 + 组织 + 角色，[BASE-01](../D0/BASE-01.md)）下推到各来源 API；跨租户/越权默认 0 + 审计。
- **降级语义统一**：未建域 = `NOT_AVAILABLE`、外部断连 = `NOT_CONNECTED`（核心 §10）、关模型 = `MODEL_DISABLED`（核心 §11）；前端据此渲染降级态文案，不渲染为 0/空白冒充正常。

## 本域涉及

- **租户层级**：聚合按 `OrgContext` 全七层作用域；平台/集团角色看跨院聚合，院内角色看本院；跨租户不串（[BASE-01](../D0/BASE-01.md) 行级隔离）。
- **外部系统**：仅**只读读取**健康/连通状态（Provider/适配器/Dify/图投影）用于健康灯与降级标记；工作台**不**发起任何外部写或业务调用。

## 本域最烫的不变量（点核心编号 + 为何在本域最关键）

- **铁律#1 真实性 / #2 诚实降级**：工作台是"系统是否真实可用"的门面；伪造健康灯/待办数、或前端调不存在端点白屏、或演示页留在生产，都是最高危假闭环。
- **核心 §13 #8 六态 + §16 体验门禁**：登入第一屏必须六态齐全（"空/降级/无权限"在 D1 是常态），一页一目标、低打扰。
- **核心 §1 #7 五维权限（数据维）+ 默认角色视图**：同一工作台，13 角色各见各的；越权摘要不可见，应见摘要不缺失；院长视角无技术词（详规 S0 验收）。
- **核心 §7 关系库权威 / 铁律#5**：工作台是只读组合，每个数字可回溯权威源、可重建；不产生新权威事实、不新增独立接口（详规 S0）。
- **核心 §9 多租户隔离 + 六阶段生命周期**：聚合按 `OrgContext` 作用域；平台/院管视图含租户生命周期阶段面板。

## 域级验收（D1-验收）

D1 全部卡 `done` 后过域级验收（[质量基线 §2.3](../../audit/质量基线.md)）：

1. 13 角色逐个登入 → 工作台为默认落点，各见**正确默认角色视图**（无越权摘要可见、无应见摘要缺失；院长视角无技术词）；
2. 六态逐态可见：正常 / 加载（骨架）/ 空（带下一步文案）/ 错误（中文 + traceId）/ 无权限 / 部分成功（部分卡片降级其余正常）；
3. 每个摘要指标**可下钻**到所属域对象/责任人/待办（未建域下钻指向"该域未启用"诚实页，不 404、不假链接）；
4. 关闭外部依赖（Provider/Dify/图/模型）→ 健康灯与摘要**诚实降级**（NOT_CONNECTED/MODEL_DISABLED），首屏不崩、不伪造、无 dangling 404；
5. 演示与校验页给出**真实**演示就绪度自检（可演示项/阻塞项 + 原因），不假绿；
6. 生产路由**无** StepFlowDemo 等无数据演示页（[INFRA-09](INFRA-09.md)）；首屏 P95 ≤ 1s；聚合按租户隔离；
7. T-GATE 前后端真实性门禁全绿；owner ≠ reviewer 签字。

**域级验收过，才算 D1 落实，才走 D2。**

## 目标

把工作台页（`/dashboard`）做成**登入后第一屏真实可用**：不同角色 10 秒内知道"今天要处理什么、系统是否安全、试点是否可验收"。按角色呈现摘要主卡片 + 待办 + 本周建议动作 + 租户生命周期 + 最近变化；**复用现有 API、不新增独立工作台接口**（详规 S0）；六态完整、一页一目标、指标可下钻、降级诚实、去除硬编码 mock（`todoMock`/静态指标）接真实来源 API。

## 视角清单（11 视角逐条）
1. **产品架构**：登入唯一第一屏；卡片由"所属域现有 API"驱动，新域上线挂新卡片，前端不硬编码聚合后端、不造独立接口。
2. **产品体验**：★本卡主战场 —— 六态 + 默认角色视图 + 一页一目标 + ≤1 主按钮 + ≤3 筛选 + 院长无技术词 + 降级文案克制（详规 S0 / 体验契约）。
3. **系统与数据架构**：各卡片并行取数 + 独立 react-query 缓存 + 骨架；单源慢/挂只降级该卡片不拖垮首屏；首屏 P95 ≤ 1s；待办大列表走 [API-13](../D0/API-13.md) 服务端分页不前端全量。
4. **临床医疗安全**：临床相关卡片仅**提醒与跳转**，不在工作台直接下医嘱/确认（医师确认在所属域，核心 §6）。
5. **知识与数据治理**：知识同步卡片不展示未核验"最新"；AI 相关卡片按 §AI 明显标识渲染。
6. **安全合规与监管**：越权卡片不渲染（依赖来源 API 过滤，前端无假兜底）；错误/空态不泄露越权对象；无敏感字段明文（脱敏在源）。
7. **集团化与多租户治理**：★平台/集团角色看跨院聚合、院内角色看本院；含租户六阶段生命周期面板 + 多维治理切片 ≤3 维；租户切换（[BASE-06](../D0/BASE-06.md)）后重取（核心 §9）。
8. **集成与互操作**：系统健康/Provider 连通卡片如实展示 `NOT_CONNECTED`，不伪造"已连接"（核心 §10）。
9. **运维 / SRE / 国产化**：内网慢场景骨架 + 进度；国产浏览器/老旧分辨率可读（体验契约可读性、[BASE-10](../D0/BASE-10.md) 老年模式 ≥16pt）。
10. **质量与真实性审计**：★去除现状硬编码 `todoMock`/静态指标值、各卡片接真实来源 API（无 page-mock、无伪造摘要数）；degraded 如实渲染（核心 §13 / 铁律#1）。
11. **AI / 模型治理与可降级**：AI 相关卡片（如 AI 知识审核待办）关模型显示"智能建议暂不可用"，工作台主体仍可用（天然 B0）。

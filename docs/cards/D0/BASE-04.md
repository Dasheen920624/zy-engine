## 目标

交付**统一审计骨干**：写操作 / 审核 / 发布 / 运行 / 反馈 / 导出 / 回滚七类动作统一留痕，审计 append-only、fail-soft（写失败不回滚业务但可观测），高危项不可从 UI 关。

## 功能要求（原子可测条目）

- [ ] **FR-1 统一审计事件**：`{who, org, role, action, target, before, after, traceId, env, ts}` 统一模型；任何留痕走同一 `AuditRecorder`。
- [ ] **FR-2 七类动作全留痕**：写操作 / 审核 / 发布 / 运行 / 反馈 / 导出 / 回滚 七类均产审计（核心 §8）。
- [ ] **FR-3 fail-soft**：审计写入失败**不回滚业务**（异步队列或降级落本地），但失败本身告警可观测（[OBS-01](OBS-01.md)）。
- [ ] **FR-4 append-only + 高危护栏**：审计记录只追加不可改删；"关闭审计"等高危项本身被审计且**不可从 UI 关**（核心 #19）。
- [ ] **FR-5 超管动作高亮**：内置超管的所有动作独立高亮审计（核心 #20）。
- [ ] **FR-6 可查询导出**：审计按 {who/org/action/时间} 可查询、可导出证据包（呼应 D5 审计日志页 + EVID 卡 + A6 剧本）。

## 接口契约 / 页面契约
### 接口契约
- 端点：交付内部 `AuditRecorder` 切面/Port + 审计查询端点（供 D5 审计日志页消费）。
- DTO：审计查询 Record DTO（时间范围 / 动作 / 主体 / 组织）+ Bean Validation；分页用 `PageResult`（[API-13](API-13.md)）。
- 响应信封：`ApiResult` / `ProblemDetail`。
- 状态机：N·A —— 审计是事实记录非资产。
- 幂等 / 错误码 / traceId：审计写按 `(traceId, action, target)` 去重；审计自身 traceId 串联业务 traceId。

### 页面契约
N·A —— 审计日志页在 D5「审计日志」槽（页面卡），本卡供其查询 API 与数据模型。

## 数据与迁移
- 表族：`sys_audit_log`（append-only）；可选 `sys_audit_archive`（冷归档）。
- 主键：ULID；唯一约束：`(trace_id, action, target_id)` 去重；索引：`who`、`org_path`、`action`、`ts`、`env`。
- 组织字段：`tenant_id` + `org_path`（审计可按组织维过滤）；不可变（无 update/delete 权限，DB 层尽量约束）。
- 5 方言迁移：h2 / postgres / oracle / dm / kingbase + 中文注释 + 时间分区建议。

## 视角清单（11 视角逐条）
1. **产品架构**：审计是单一留痕源；任何模块不得自建私有日志当审计。
2. **产品体验**：N·A（无页面）—— 审计日志页在 D5 消费本数据。
3. **系统与数据架构**：fail-soft 异步留痕、append-only、高写入量分区；10 万级/日审计写入不阻塞业务主链路。
4. **临床医疗安全**：临床高危动作（确认医嘱 / 双签 / 危急值处置 / 旧版隔离切换）必留审计可回溯（核心 §6）。
5. **知识与数据治理**：知识来源/生成/审核/发布/替换全留痕（核心 §7/§8），支撑唯一权威知识的可审计替换。
6. **安全合规与监管**：★本卡主战场 —— 审计链是等保/个保法核心证据；脱敏后留痕（before/after 不存敏感明文）。
7. **集团化与多租户治理**：审计带组织维，集团可按院/科下钻审计。
8. **集成与互操作**：外部对接的连接/同步/回调动作留审计（核心 §10 同步证据）。
9. **运维 / SRE / 国产化**：审计写失败告警；归档 RPO；5 方言；离线形态审计仍可导出证据。
10. **质量与真实性审计**：★禁假审计（假证据/假同步/UUID 充哈希，核心 #18）；审计 hash 用 SHA-256/SM3 非 UUID。
11. **AI / 模型治理与可降级**：模型调用的数据策略/模型版本/输出 hash/审核动作/降级情况留审计（核心 §11）。

## 适用不变量
- 命中核心约束：**§8 审计链** · **#19 配置外置（审计不可从 UI 关）** · **#20 超管动作高亮** · **#18 禁假证据**。

# D0 登录域 / 平台脊柱 · 域简报

> 读卡前置：先读 [核心 CONSTITUTION](../../CONSTITUTION.md)，再读本简报，再读你领的那张卡。页面卡另读 [体验契约](../../EXPERIENCE_CONTRACT.md)。
> 本域所有卡共享的上下文放这里，**不在卡间复制**。冲突裁决：核心 > 本简报 > 卡。

## 域目标

登录与登入后可导航的空壳**真实可用**：把"组织/租户/身份/权限/审计/迁移/前端骨架/真实性门禁/运行底座"这条**所有页面都绕不开的地基**做成 B0 真实、可验证。D0 不交付业务功能，只交付脊柱——脊柱不真，上面所有域都是沙上筑塔。

## 登入 / 使用角色（13 角色矩阵的本域子集，全量矩阵见 [质量基线 §9](../../audit/质量基线.md)）

D0 是**登录域**——13 角色全部在此登入，菜单按五维 RBAC 各自呈现。其中**配置/治理 D0 脊柱**的角色：

| 角色 | 在 D0 主要干什么 |
|---|---|
| 内置超级管理员（核心 #20）| 系统启动自动授满五维 + 系统配置中心；不可降权/删除/旁路；强制 MFA；独立高亮审计 |
| 平台管理员 platform-admin | 租户开通、许可证、全局标准包、系统配置（脊柱视角：建组织根、发种子身份）|
| 医院管理员 hospital-admin | 院内组织树、用户、角色绑定、发布审批 |
| 信息科 it-ops | 运行监控、健康检查、国产化自检、会话与安全策略 |
| 合规审计 audit-compliance | 审计日志、安全基线、证据导出、跨租户隔离核查 |
| 其余 8 角色 | 仅作为"登入 + 菜单按 RBAC 呈现 + 退出/会话过期"的受测对象（业务能力在各自域）|

## 共享数据模型 / 实体（D0 卡共用，单一归属在此声明、卡内只引用）

- **组织层级（核心 §9 七层）**：平台 → 集团 → 医院 → 院区 → 社区服务点 → 科室 → 专病；父子链单根无环、可继承可局部覆盖可冲突仲裁。
- **上下文载体**：`OrgContext`（请求级不可变）= {tenantId, orgPath, userId, roleSet, packageVersionId, env}。
- **身份权限**：user / role / permission（五维：菜单/动作/数据/资产/环境）/ 13 角色×27 二级+5 高级 矩阵 / 身份绑定（OIDC 等）。
- **包版本快照**：package_version——同一请求看到一致的配置/知识版本（呼应核心 §7 唯一权威知识）。
- **审计骨干**：统一留痕（写操作/审核/发布/运行/反馈/导出/回滚）；audit 异步或 fail-soft，不回滚业务。
- **标准临床模型骨架**：SYS-01 的 12 类标准对象（Patient/Encounter/… ）作为上层域的共享类型定义在此立骨（D0 只立类型与事件上下文，不做业务）。

> **权威源铁律**：以上实体的唯一权威源是**院内关系数据库**（核心 §7 / 铁律#5）；任何缓存/图投影/Dify 只能是投影或执行器，必须可重建。

## 依赖

- **上游引擎 / API**：无（D0 是最底脊柱，被所有域依赖，自身不依赖业务引擎）。
- **上游卡**：域内强耦合顺序——BASE-01 组织上下文 → BASE-02 五维权限 / BASE-03 API 契约 / BASE-04 审计 是同层地基；INFRA-01/02 真实性门禁应**最先落**（后续所有卡的验收前提）；BASE-06 前端骨架依赖 BASE-02 路由元数据 + BASE-10 token。卡内"依赖卡"逐张声明。

## 本域涉及

- **租户层级**：全部七层（本域正是建立层级本身）。
- **外部系统**：仅身份提供方（OIDC / 院内 LDAP / 统一身份）用于登录与身份绑定；HIS/EMR/LIS/PACS/医保/区域等业务外部系统**不在 D0**（在 D2 适配器中心起）。

## 本域最烫的不变量（点核心编号 + 为何在本域最关键）

- **核心 §1 #7 五维权限 + #19 配置外置 + #20 内置超管**：脊柱若漏权限维或留旁路黑洞或写死配置，全系统合规与安全归零。
- **核心 §9 集团化与多租户**：七层继承覆盖 + 租户隔离是脊柱的存在理由；跨租户串数据是最高危 bug。
- **核心 §7 关系库权威 / 铁律#5**：上下文与组织树唯一权威是关系库，缓存只能投影。
- **核心 §13 真实性门禁 / 铁律#1**：INFRA-01/02 门禁是后续**所有**域 done 的前置闸；D0 必须先把闸立起来。
- **核心 §2 菜单 IA 锁**：5+1 一级 / 27+5 二级 锁死，BASE-06 前端骨架不得净增菜单。

## 域级验收（D0-验收）

D0 全部卡 `done` 后过域级验收（[质量基线 §2.3](../../audit/质量基线.md)）：

1. 按 13 角色矩阵逐个登入；
2. 菜单按五维 RBAC 正确呈现（无越权可见、无应见缺失），到 27 二级 + 5 高级粒度；
3. 各页路由可打开到**六态空态**（无 Hook 错误 / null reference / API 404）；
4. 退出登录、会话过期、多 tab 同步生效；
5. 跑通 D0 B0 主链路 E2E：建组织 → 建用户 → 绑角色 → 登入 → 菜单呈现 → 跨租户被拒 → 审计留痕 → 退出；
6. T-GATE 前后端真实性门禁全绿；
7. owner ≠ reviewer 签字。

**域级验收过，才算 D0 落实，才走 D1。** 模型增强不在 D0（第二波）。

## 功能要求（原子可测条目）

- [ ] **FR-1 菜单 IA 锁**：左侧 SideMenu 渲染 5+1 一级 + 27+5 二级（核心 §2，顺序固定）；新增一级菜单 / 净增二级菜单被测试与门禁拒绝（核心 #2/#3）。
- [ ] **FR-2 路由元数据**：每路由带 `{sectionKey, menuKey, menuLabel, requiredPermissions, requiredRoles}`；菜单可见性由 [BASE-02](BASE-02.md) 五维判定源驱动（前后端同源）。
- [ ] **FR-3 PageShell + 六态**：统一 `PageShell` 外壳 + 加载/空/错误/无权限/部分成功/正常 六态组件（体验契约）；任何页面缺一态被拒。
- [ ] **FR-4 4 状态机 Badge**：配置/变更/待办/告警 四套状态机的统一 Badge 组件（核心 §3），禁自创状态文案。
- [ ] **FR-5 7 步流组件**：统一 `StepFlow` 组件（选模板→校验→看影响→提交审核→灰度→全量→留证，核心 §4），配置类页面复用、禁自创两步流。
- [ ] **FR-6 命令面板**：全局命令面板（快速导航/搜索菜单），SideMenu 始终为主菜单（核心 #2，不改顶部横向导航）。

## 视角清单（11 视角逐条）
1. **产品架构**：★菜单 IA 锁是产品骨架单一源；5+1/27+5 锁死，所有页面归入、不旁生。
2. **产品体验**：★PageShell 六态 + 一页一目标 + 状态机 Badge + 7 步流 = 全平台一致体验骨架（核心 §16、体验契约）。
3. **系统与数据架构**：路由元数据 schema 是前后端权限同源契约；菜单大列表按需加载。
4. **临床医疗安全**：嵌入提醒载体（卡片/抽屉）的打扰等级骨架在此预留（红线强打断 vs 弱打扰，核心 §6），具体在 D3。
5. **知识与数据治理**：N·A —— 骨架不承载知识。
6. **安全合规与监管**：无权限态不暴露敏感数据（六态规约，核心 §8）。
7. **集团化与多租户治理**：租户切换器位 + 组织维筛选骨架（消费 BASE-01 上下文）。
8. **集成与互操作**：N·A —— 骨架层。
9. **运维 / SRE / 国产化**：5 主题模式（含老年模式）适配国产化终端；前端构建产物可离线部署。
10. **质量与真实性审计**：★无 mock 假闭环、无写死医学常量、无 JSON 裸渲染（核心 #18，门禁 INFRA-01）；骨架不得含 eslint-disable。
11. **AI / 模型治理与可降级**：AI 内容标识组件（"AI 候选/生成"，核心 #9）骨架预留，D3 消费。

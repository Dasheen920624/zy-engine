## 目标

交付**统一大规模列表 API**：分页/游标/排序/过滤/total_estimate/异步导出标准口径，支撑 10 万级知识与资产列表服务端分页，杜绝前端全量加载（核心 #16）。

## 功能要求（原子可测条目）

- [ ] **FR-1 统一分页**：offset 分页 + cursor 游标分页（深翻页用游标，稳定不漂移），统一 `PageResult<T>`（[BASE-03](BASE-03.md)）。
- [ ] **FR-2 排序/过滤白名单**：排序字段 + 过滤条件走**白名单**（防 SQL 注入与任意字段排序拖垮库）。
- [ ] **FR-3 total_estimate**：大表返回估算总数（避免 `count(*)` 全表扫），精确总数可选异步。
- [ ] **FR-4 异步导出**：超阈值导出走异步任务（与 [BASE-08](BASE-08.md) 导出任务同源），不同步拼大响应。
- [ ] **FR-5 服务端分页强制**：默认每页上限 + 禁一次性返回全集（核心 #16，门禁联动）。
- [ ] **FR-6 10 万级性能**：游标分页 P95 稳定（覆盖索引），深翻页不退化。

## 接口契约 / 页面契约
### 接口契约
- 端点：统一列表查询规约（`GET .../list?cursor=&size=&sort=&filter=`）+ 异步导出触发端点。
- DTO：`PageQuery`（cursor/size/sort/filter 白名单）+ `PageResult<T>`（items/cursor/total_estimate）Record + Bean Validation。
- 响应信封：`ApiResult<PageResult<T>>`。
- 状态机：导出任务用**待办类**状态机（核心 §3）。
- 幂等 / 错误码 / traceId：导出任务幂等键；`SORT_FIELD_NOT_ALLOWED`/`PAGE_SIZE_EXCEEDED`；带 traceId。

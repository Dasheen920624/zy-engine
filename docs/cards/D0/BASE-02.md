## 目标

交付**五维权限引擎**（菜单 / 动作 / 数据 / 资产 / 环境）与 13 角色模型：任何鉴权决策走统一 RBAC 引擎（含超管不旁路），后端方法级、数据行级、前端菜单级口径一致。

## 功能要求（原子可测条目）

- [ ] **FR-1 五维权限点**：定义 菜单 / 动作 / 数据 / 资产 / 环境 五维权限点；`can(subject, dimension, target, ctx)` 返回**确定性**结果（无随机、无写死放行）。
- [ ] **FR-2 13 角色**：内置 13 角色（质量基线 §9），每角色绑定五维权限集；27 二级菜单粒度矩阵的**目录与默认策略**在 [INFRA-05](INFRA-05.md)，本卡只定义引擎与角色实体。
- [ ] **FR-3 求值经引擎**：所有鉴权（含内置超管，核心 #20 不旁路）走同一 `PermissionEvaluator`；后端 `@RequirePermission` + 数据行级 + 前端菜单可见性三处取同一判定源。
- [ ] **FR-4 数据维结合组织**：数据权限（本科室 / 全院 / 集团跨院 / 脱敏）基于 [BASE-01](BASE-01.md) 的 `orgPath` 求值；跨域默认拒绝。
- [ ] **FR-5 环境维 + 应急**：测试 / 试运行 / 正式 / 应急 环境权限；应急权限（break-glass）强制审计（[BASE-04](BASE-04.md)）+ 时限自动回收。
- [ ] **FR-6 越权诚实拒绝**：缺权限返回 403 `ProblemDetail` + 权限范围说明 + 申请入口（呼应六态"无权限态"），不暴露目标存在性。

## 视角清单（11 视角逐条）
1. **产品架构**：五维权限是全平台唯一鉴权口径；前后端同源，杜绝"前端藏菜单、后端不校验"的假权限。
2. **产品体验**：N·A（无页面）—— 但无权限态的"范围说明 + 申请入口"由本卡定义数据，BASE-06 呈现。
3. **系统与数据架构**：`PermissionEvaluator` 确定性、可缓存（关系库权威、缓存投影）；13 角色×五维 求值 P95 < 10ms。
4. **临床医疗安全**：高危动作（双签 / 高风险发布 / 应急）的权限位在此定义；无权限不得执行高危动作（核心 §6）。
5. **知识与数据治理**：资产维权限（病种 / 字典 / 路径 / 知识包级）保证知识资产授权可控（核心 §7）。
6. **安全合规与监管**：★本卡主战场 —— 五维 RBAC 是等保 / 个保法落点（核心 §8）；最小授权 + 应急可审计 + 越权留痕。
7. **集团化与多租户治理**：角色随组织层级继承覆盖（科室角色覆盖院级，核心 §9）；跨租户角色不串。
8. **集成与互操作**：N·A —— 外部系统调用的服务账号权限在 D2 适配器卡按服务身份纳入本引擎。
9. **运维 / SRE / 国产化**：权限缓存可重建；应急权限（break-glass）运维场景受控 + 时限回收。
10. **质量与真实性审计**：无写死角色绕权、无 `if(superadmin) return true` 旁路（核心 #20）；门禁校验鉴权不被注释绕过。
11. **AI / 模型治理与可降级**：N·A —— 脊柱层无模型；天然 B0。

## 适用不变量
- 命中核心约束：**#7 五维权限** · **#20 内置超管不旁路** · **§9 继承覆盖** · **§6 高危动作权限位**。

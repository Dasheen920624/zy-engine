# 数据治理用户手册

> 适用版本：MedKernel v1.0 GA
> 适用对象：医院信息科管理员、数据管理员、质控科人员
> 最后更新：2026-05-23

---

## 1. 功能概述

数据治理模块是 MedKernel 平台的数据底座，负责患者、医生、科室等主数据的统一管理和数据质量监控。通过标准化的主数据管理和自动化的质量检查，确保平台运行所依赖的基础数据准确、完整、一致。

数据治理模块的核心能力：

- **主数据管理**：统一管理患者、医生、科室、诊断、医嘱、药品等核心主数据
- **数据质量监控**：自动执行数据质量检查，发现缺失、重复、冲突等质量问题
- **质量报告**：生成数据质量报告，支持趋势分析和整改追踪
- **隐私保护**：患者敏感信息加密存储，展示时自动脱敏，满足个保法要求
- **多租户隔离**：主数据按租户隔离，确保跨院区数据安全

所有主数据操作均记录审计日志，满足等保 2.0 三级审计要求。

## 2. 核心概念

### 2.1 患者主数据（Patient Master Data）

患者主数据是平台中最核心的主数据实体，用于唯一标识和管理患者信息。

| 字段 | 说明 | 隐私保护 |
|------|------|----------|
| 患者ID | 患者的业务唯一标识 | — |
| 姓名 | 患者姓名 | 展示脱敏：张\*\* |
| 性别 | M/F | — |
| 出生日期 | 出生日期 | — |
| 身份证号 | 18 位身份证号 | 加密存储，展示脱敏：110\*\*\*\*\*\*\*\*1234 |
| 联系电话 | 手机号码 | 展示脱敏：138\*\*\*5678 |
| 地址 | 住址信息 | 加密存储 |
| 状态 | ACTIVE/INACTIVE | — |

患者主数据管理要点：
- **身份合并**：同一患者在不同系统中可能有不同 ID，需进行身份合并
- **冲突处理**：不同来源的患者信息不一致时，需人工确认正确数据
- **脱敏展示**：所有患者数据展示页强制执行脱敏规则，需"查看完整"权限才能查看原文

### 2.2 医生主数据（Doctor Master Data）

医生主数据管理院内医生的执业信息和专业归属。

| 字段 | 说明 |
|------|------|
| 医生ID | 医生的业务唯一标识 |
| 姓名 | 医生姓名 |
| 性别 | 性别 |
| 职称 | 主任医师/副主任医师/主治医师/住院医师等 |
| 专业编码 | 专科方向编码 |
| 所属科室 | 当前所属科室编码 |
| 执业证书号 | 医师执业证书编号 |
| 状态 | ACTIVE/INACTIVE |

医生主数据管理要点：
- **科室归属**：医生可能同时归属多个科室（会诊、兼职），需明确主科室
- **职称变更**：职称变更需及时更新，影响路径权限和审核流程
- **状态管理**：离职医生标记为 INACTIVE，关联的待办任务自动转派

### 2.3 科室主数据（Department Master Data）

科室主数据管理院内组织架构和科室层级关系。

| 字段 | 说明 |
|------|------|
| 科室编码 | 科室的业务唯一标识 |
| 科室名称 | 科室全称 |
| 科室类型 | 临床/医技/行政 |
| 上级科室 | 上级科室编码，支持多级层级 |
| 状态 | ACTIVE/INACTIVE |

科室主数据管理要点：
- **层级结构**：支持多级科室树，如"内科 → 心内科 → 冠心病病房"
- **类型分类**：临床科室参与路径执行，医技科室参与检查流程，行政科室参与管理
- **科室合并/拆分**：组织架构调整时需同步更新科室主数据和相关规则

### 2.4 数据质量（Data Quality）

数据质量模块对主数据进行自动化质量检查，发现并追踪数据问题。

质量检查维度：

| 维度 | 说明 | 检查规则示例 |
|------|------|--------------|
| 完整性 | 必填字段是否为空 | 患者身份证号不能为空 |
| 唯一性 | 是否存在重复记录 | 同一身份证号不能有多条患者记录 |
| 一致性 | 关联数据是否一致 | 医生所属科室必须存在于科室主数据中 |
| 准确性 | 数据值是否符合规范 | 性别字段只能是 M 或 F |
| 时效性 | 数据是否及时更新 | INACTIVE 状态的患者不应有新的就诊记录 |

质量问题处理流程：
1. 系统自动执行质量检查，生成质量问题清单
2. 质量问题按严重程度分级：严重/一般/提示
3. 严重问题自动生成质控预警，进入整改闭环
4. 整改完成后重新执行检查，确认问题已解决

## 3. 操作步骤

### 3.1 管理患者

**查看患者列表**：
1. 在左侧菜单选择 **临床运行 → 患者主索引**
2. 默认展示当前租户下的患者列表
3. 使用筛选条件缩小范围：
   - 按姓名、患者 ID 搜索
   - 按状态筛选（ACTIVE/INACTIVE）
   - 按科室筛选

**新建患者**：
1. 在患者主索引页面，点击 **新建患者** 按钮
2. 填写患者基本信息：
   - 必填项：患者 ID、姓名、性别
   - 选填项：身份证号、联系电话、地址
3. 系统自动校验：
   - 患者 ID 唯一性检查
   - 身份证号格式校验
   - 手机号格式校验
4. 点击 **保存**

**处理身份合并**：
1. 在患者主索引页面，选择多个疑似重复的患者记录
2. 点击 **身份合并** 按钮
3. 系统展示合并预览：
   - 各来源的患者信息对比
   - 冲突字段高亮标注
4. 逐字段确认合并结果：
   - 选择保留哪个来源的数据
   - 或手动输入正确的数据
5. 确认合并后，系统将多条记录合并为一条，保留合并历史

> **注意**：身份证号和手机号采用加密存储。数据库中存储密文，仅授权用户通过"查看完整"功能可查看原文。展示页面默认脱敏显示。

### 3.2 管理医生

**查看医生列表**：
1. 在左侧菜单选择 **合规运维 → 用户管理**
2. 切换到 **医生主数据** 标签页
3. 查看医生列表，支持按科室、职称、状态筛选

**新建医生**：
1. 点击 **新建医生** 按钮
2. 填写医生信息：
   - 必填项：医生 ID、姓名、所属科室、职称
   - 选填项：专业编码、执业证书号
3. 系统校验：
   - 医生 ID 唯一性
   - 所属科室必须存在于科室主数据中
   - 执业证书号格式校验
4. 点击 **保存**

**更新医生信息**：
1. 在医生列表中找到目标医生，点击 **编辑**
2. 修改需要更新的字段
3. 关键变更（如科室调动、职称变更）需填写变更原因
4. 保存后系统自动更新关联的路径权限和审核流程

### 3.3 管理科室

**查看科室树**：
1. 在用户管理页面，切换到 **科室主数据** 标签页
2. 系统以树形结构展示科室层级关系
3. 支持展开/折叠查看子科室

**新建科室**：
1. 点击 **新建科室** 按钮
2. 填写科室信息：
   - 科室编码：使用标准编码，如 `DEPT_CARDIOLOGY`
   - 科室名称：填写中文全称
   - 科室类型：选择临床/医技/行政
   - 上级科室：选择父级科室（顶级科室留空）
3. 系统校验：
   - 科室编码唯一性
   - 上级科室必须存在
   - 不允许循环引用
4. 点击 **保存**

**科室调整**：
1. 科室合并：将一个科室的患者和医生迁移到另一个科室
2. 科室拆分：从一个科室中拆分出新的子科室
3. 科室停用：将科室标记为 INACTIVE，关联数据保留但不再参与新业务
4. 所有调整操作需填写变更原因，记录在审计日志中

### 3.4 执行质量检查

**手动执行检查**：
1. 在左侧菜单选择 **质控改进 → 评估指标库**
2. 切换到 **数据质量** 标签页
3. 点击 **执行质量检查** 按钮
4. 选择检查范围：
   - 主数据类型：患者/医生/科室/诊断/医嘱/药品
   - 检查维度：完整性/唯一性/一致性/准确性/时效性
   - 时间范围：全量/增量（最近 N 天变更的数据）
5. 点击 **开始检查**
6. 系统执行检查并生成质量问题清单

**查看检查结果**：
1. 检查完成后，自动跳转到质量问题列表
2. 列表展示：
   - 问题类型和严重程度
   - 涉及的数据记录
   - 问题描述和建议修复方式
3. 按严重程度排序，严重问题优先处理
4. 点击某条问题查看详情：
   - 问题数据的原始值和期望值
   - 关联影响范围（影响哪些规则、路径）
   - 修复建议

**处理质量问题**：
1. 对于可自动修复的问题，点击 **自动修复**
2. 对于需人工确认的问题，点击 **手动处理**：
   - 修改数据值
   - 标记为误报
   - 创建整改任务
3. 处理完成后，问题状态更新为"已解决"
4. 严重问题自动关联质控预警，需在质控驾驶舱确认闭环

### 3.5 查看质量报告

1. 在数据质量页面，点击 **质量报告** 按钮
2. 选择报告类型：
   - 综合报告：所有主数据的整体质量评分
   - 专项报告：某类主数据的详细质量分析
   - 趋势报告：质量指标随时间的变化趋势
3. 设置报告参数：
   - 统计周期：本周/本月/本季度
   - 数据范围：全院/指定科室
4. 生成报告，包含以下内容：
   - 质量评分：按维度（完整性/唯一性/一致性/准确性/时效性）打分
   - 问题分布：按主数据类型和问题类型统计
   - 整改进度：已解决/整改中/待处理的问题数量
   - 趋势对比：与上一周期的质量对比
5. 支持导出 PDF/CSV 格式报告

## 4. 常见问题

### Q1：患者身份合并后，原 ID 还能使用吗？

- 合并后系统保留所有原始 ID 作为"别名"，通过任一 ID 均可查询到合并后的患者
- 合并后的主 ID 为合并时选择的主记录 ID
- 合并历史永久保留在审计日志中，支持追溯

### Q2：数据质量检查发现大量问题，如何优先处理？

- 按严重程度排序，优先处理"严重"级别的问题
- 按影响范围排序，优先处理影响临床路径执行的问题
- 按问题类型排序：
  1. 唯一性问题（重复记录）→ 可能导致路径执行错误
  2. 一致性问题（关联数据不匹配）→ 可能导致规则误判
  3. 完整性问题（必填字段缺失）→ 影响数据可用性
  4. 准确性和时效性问题 → 可后续批量修复

### Q3：科室调整后，关联的规则和路径如何处理？

- 科室停用后，该科室关联的规则和路径不会自动下线
- 系统会生成预警，提示管理员检查关联规则和路径的适用范围
- 建议在科室调整前，先评估影响范围：
  - 在科室详情页查看关联的规则、路径和医生数量
  - 制定迁移计划后再执行调整

### Q4：患者数据脱敏规则是否可以自定义？

- 当前版本采用固定脱敏规则：
  - 身份证号：保留前 3 位和后 4 位，中间脱敏
  - 手机号：保留前 3 位和后 4 位，中间脱敏
  - 姓名：保留第一个字，其余用 \* 替代
- 如需查看完整数据，需具备"查看完整"权限
- 所有查看完整数据的操作记录在审计日志中

### Q5：主数据从 HIS 同步时出现冲突，如何处理？

- 系统支持配置冲突处理策略：
  - 以 HIS 为准：HIS 数据覆盖平台数据
  - 以平台为准：保留平台数据，忽略 HIS 变更
  - 人工确认：冲突时生成待办任务，由管理员确认
- 建议首次同步时选择"以 HIS 为准"，后续增量同步选择"人工确认"
- 冲突记录保留在同步日志中，支持回溯

### Q6：数据质量报告中的评分是如何计算的？

- 每个维度按问题数量占比计算得分：得分 = (1 - 问题数/总记录数) × 100
- 综合评分为各维度的加权平均：
  - 完整性权重 30%
  - 唯一性权重 25%
  - 一致性权重 20%
  - 准确性权重 15%
  - 时效性权重 10%
- 评分低于 80 分时系统自动生成质控预警

### Q7：如何批量导入主数据？

1. 在对应主数据页面，点击 **导入** 按钮
2. 下载导入模板（Excel 格式）
3. 按模板格式填写数据
4. 上传导入文件
5. 系统自动校验：
   - 格式校验：字段类型和长度
   - 唯一性校验：ID 和编码不重复
   - 关联性校验：外键引用存在
6. 校验通过后确认导入
7. 校验不通过时，下载错误报告，修正后重新导入

### Q8：主数据变更是否会影响正在执行的路径？

- 患者状态变更为 INACTIVE 不会影响已入径的路径
- 医生科室调动后，该医生负责的待办任务自动转派到新科室
- 科室停用后，该科室下进行中的患者路径继续执行，但不接受新入径
- 所有关键变更均生成通知，提醒相关人员进行确认

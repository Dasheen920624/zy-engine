# 知识管理用户手册

> 适用版本：MedKernel v1.0 GA
> 适用对象：医院信息科管理员、医学知识管理员、路径管理员
> 最后更新：2026-05-23

---

## 1. 功能概述

知识管理模块是 MedKernel 平台的知识底座，负责医学知识的来源注册、订阅管理、同步调度和知识包管理。通过标准化的知识生命周期管理，确保平台中的规则、路径、图谱等资产始终基于最新、可信、可追溯的医学知识。

知识管理模块的核心能力：

- **来源注册**：注册指南、文献、标准目录等 7 类知识来源，管理授权和权威等级
- **知识订阅**：按主题订阅知识更新，自动获取最新内容
- **同步调度**：支持定时同步和手动同步，差异预览和失败重试
- **知识包管理**：打包导出/导入知识资产，支持跨环境迁移和冲突检测
- **AI 候选审核**：对 AI 生成的知识候选进行人工审核，确保知识质量

所有知识操作均保留来源证据链，满足等保 2.0 三级审计和循证医学要求。

## 2. 核心概念

### 2.1 知识来源（Knowledge Source）

知识来源是医学知识的生产方，定义了知识的出处和授权信息。

平台支持以下 7 类知识来源：

| 来源类型 | 说明 | 示例 |
|----------|------|------|
| 临床指南 | 国内外权威学会发布的诊疗指南 | ACC/AHA STEMI 指南、中国心衰指南 |
| 专家共识 | 专家组达成的共识意见 | 中国房颤抗凝共识 |
| 标准目录 | 国家/行业标准编码目录 | ICD-10、药品编码、手术编码 |
| 文献数据库 | PubMed、CNKI 等学术文献库 | 随机对照试验、Meta 分析 |
| 院内制度 | 医院内部制定的诊疗规范 | 院内抗菌药物使用规范 |
| 医保政策 | 医保目录和报销规则 | DRG/DIP 分组规则 |
| AI 生成 | AI 模型辅助生成的知识候选 | 规则推荐、路径建议 |

每个来源需配置以下属性：

| 属性 | 说明 |
|------|------|
| 来源编码 | 唯一标识 |
| 来源名称 | 业务可读名称 |
| 权威等级 | 高/中/低，影响知识审核优先级 |
| 授权范围 | 使用授权、再分发授权、商用授权、出口授权 |
| 语言 | 知识内容的语言 |
| 地区 | 知识适用的地区范围 |
| 有效期 | 知识的有效时间范围 |

### 2.2 订阅（Subscription）

订阅定义了平台对某类知识主题的持续关注和自动获取机制。

- 支持按主题订阅：如"心衰诊疗""抗凝管理""医保 DRG"等 6 类主题
- 订阅后系统自动监控来源更新，发现新内容时触发同步
- 支持订阅优先级和过滤条件，避免获取无关内容
- 订阅状态：活跃（ACTIVE）、暂停（PAUSED）、已取消（CANCELLED）

### 2.3 同步（Sync）

同步是将外部来源的知识内容拉取到平台的过程。

| 同步方式 | 说明 |
|----------|------|
| 定时同步 | 按配置的周期自动执行（每日/每周/每月） |
| 手动同步 | 管理员手动触发，适用于紧急更新场景 |
| 增量同步 | 仅同步上次同步后的新增和变更内容 |
| 全量同步 | 重新拉取来源的全部内容 |

同步流程：
1. 系统连接来源，获取变更清单
2. 生成差异预览，展示新增/修改/删除的知识条目
3. 管理员确认后执行同步
4. 同步结果持久化到 `OPS_SYNC_TASK` 表
5. 同步失败支持重试

### 2.4 知识包（Knowledge Package）

知识包是将相关知识资产打包为可迁移单元的机制，支持跨环境部署。

知识包可包含以下资产类型：
- 规则定义及版本
- 术语映射及标准码
- 路径模板及版本
- 知识图谱版本
- 来源文档引用

知识包支持以下操作：
- **导出**：将选定资产打包为知识包文件
- **导入**：从知识包文件导入资产到目标环境
- **冲突检测**：导入时自动检测与现有资产的冲突
- **合并策略**：支持覆盖、跳过、手动选择三种冲突处理方式
- **版本管理**：知识包支持版本号，便于追溯

## 3. 操作步骤

### 3.1 注册来源

1. 登录 MedKernel 管理工作台
2. 在左侧菜单选择 **高级工具 → 来源追溯**
3. 点击 **注册来源** 按钮
4. 选择来源类型：
   - 从 7 种来源类型中选择（临床指南/专家共识/标准目录/文献数据库/院内制度/医保政策/AI 生成）
5. 填写来源基本信息：
   - 来源编码：使用标准缩写，如 `GUIDE_ACC_AHA_STEMI_2023`
   - 来源名称：填写中文全称
   - 发布年份：来源的发布/更新年份
   - 权威等级：选择高/中/低
6. 配置授权信息：
   - 使用授权：是否允许在平台内使用
   - 再分发授权：是否允许跨院区分发
   - 商用授权：是否允许商业用途
   - 出口授权：是否允许数据出境
7. 设置语言和地区范围
8. 上传来源文档（如指南 PDF、标准文件等）
9. 点击 **保存**，来源进入"待审核"状态
10. 审核通过后来源状态变为"已生效"

> **注意**：授权不明的来源会被系统标记为风险项，在知识审核和发布流程中可能被阻断。建议在注册来源时明确标注授权范围。

### 3.2 创建订阅

1. 在来源追溯页面，切换到 **知识订阅** 标签页
2. 点击 **新建订阅** 按钮
3. 选择订阅主题：
   - 心衰诊疗、抗凝管理、AMI 急救、医保 DRG、抗菌药物、肿瘤诊疗等
4. 配置订阅参数：
   - 关联来源：选择该主题关注的来源列表
   - 过滤条件：设置关键词、权威等级、语言等过滤条件
   - 同步频率：选择每日/每周/每月
   - 同步方式：增量同步（推荐）/全量同步
5. 设置通知方式：
   - 有新知识时发送站内通知
   - 同步失败时发送告警通知
6. 点击 **保存**，订阅状态变为"活跃"

### 3.3 触发同步

**手动同步**：
1. 在知识订阅列表中，找到目标订阅
2. 点击 **立即同步** 按钮
3. 系统开始连接来源并获取变更
4. 同步完成后，点击 **查看差异预览**：
   - 展示新增、修改、删除的知识条目
   - 每条变更显示来源、变更类型和影响范围
5. 确认差异后，点击 **应用变更**
6. 系统将变更写入知识库，同步任务状态更新为"已完成"

**处理同步失败**：
1. 在同步任务列表中查看失败记录
2. 查看失败原因（网络超时、来源不可达、格式错误等）
3. 修复问题后，点击 **重试** 重新执行同步
4. 多次重试仍失败时，检查来源连接配置和网络状态

> **提示**：建议在非业务高峰期执行全量同步，避免影响系统性能。增量同步可在任意时间执行。

### 3.4 导入导出知识包

**导出知识包**：
1. 在左侧菜单选择 **试点准备 → 配置包中心**
2. 点击 **导出知识包** 按钮
3. 选择导出范围：
   - 勾选需要导出的资产类型（规则/术语/路径/图谱/来源）
   - 可按科室、场景类型进一步筛选
4. 系统自动生成知识包清单，展示包含的资产数量和版本
5. 点击 **确认导出**，生成知识包文件
6. 下载知识包文件到本地

**导入知识包**：
1. 在配置包中心，点击 **导入配置包** 按钮
2. 上传知识包文件
3. 系统自动解析并展示导入预览：
   - 包含的资产列表和版本信息
   - 与现有资产的冲突检测结果
4. 处理冲突：
   - 覆盖：用导入版本替换现有版本
   - 跳过：保留现有版本，不导入冲突项
   - 手动选择：逐条确认处理方式
5. 确认导入，系统写入资产并更新版本记录
6. 导入完成后，可在配置包中心查看导入的资产

### 3.5 审核候选

AI 生成的知识候选需要经过人工审核才能正式入库使用。

1. 在左侧菜单选择 **质控改进 → AI 知识审核**
2. 查看待审核候选列表：
   - 候选类型：规则建议、路径建议、术语映射建议
   - 来源模型和置信度
   - 自动质检结果（6 类质检项）
3. 点击某条候选进入审核详情：
   - 查看 AI 生成的内容和依据
   - 查看自动质检报告：
     - 来源完整性检查
     - 有效期检查
     - 授权明确性检查
     - 规则冲突检查
     - 置信度检查
     - 多候选冲突检查
4. 审核操作：
   - **通过**：候选转为正式知识资产
   - **修改后通过**：编辑候选内容后通过
   - **退回**：退回给 AI 重新生成
   - **拒绝**：标记为不可用，不再展示
5. 审核结果记录在审计日志中

## 4. 常见问题

### Q1：来源注册时授权信息不明确，如何处理？

- 优先联系来源提供方确认授权范围
- 如无法确认，将授权标记为"待确认"，系统会在审核流程中提示风险
- 授权不明的来源生成的知识候选在发布时会被阻断
- 建议建立院内知识授权台账，统一管理来源授权

### Q2：同步任务频繁失败，如何排查？

- 检查网络连通性：确认平台服务器可访问来源地址
- 检查来源认证信息：API Key 或账号密码是否过期
- 查看同步日志：在同步任务详情中查看具体错误信息
- 常见错误类型：
  - 网络超时：增加超时时间或更换同步时段
  - 格式不兼容：检查来源数据格式是否与平台要求一致
  - 认证失败：更新来源的认证凭据

### Q3：知识包导入后部分资产显示冲突，如何处理？

- 优先选择"手动选择"模式，逐条确认冲突处理方式
- 对于规则冲突：比较两个版本的差异和来源，选择更权威的版本
- 对于术语映射冲突：检查映射的正确性，选择与院内码一致的版本
- 对于路径冲突：建议保留当前生效版本，新版本先在测试环境验证

### Q4：AI 候选审核中置信度较低的知识如何处理？

- 置信度低于阈值的候选会被自动质检标记为"低置信度"
- 建议由领域专家（主任医师或科室负责人）进行审核
- 可要求 AI 补充更多依据或参考来源
- 不建议直接通过低置信度候选，应修改后重新评估

### Q5：如何查看某条知识的完整来源证据链？

1. 在来源追溯页面，搜索目标知识资产
2. 点击进入详情，查看 **来源证据链** 标签页
3. 证据链展示：来源文档 → 引用片段 → 资产绑定 → 审核记录 → 运行证据
4. 每个环节包含时间戳和操作人信息，支持审计追溯

### Q6：知识包跨院区同步时数据安全如何保障？

- 知识包导出时自动脱敏，不包含患者隐私数据
- 导入时系统验证知识包完整性（SHA256 校验）
- 跨院区分发需来源授权支持"再分发"，否则系统阻断
- 所有导入导出操作记录在审计日志中

### Q7：订阅的主题没有收到更新通知，可能的原因？

- 检查订阅状态是否为"活跃"（非暂停或已取消）
- 检查来源是否有实际更新（部分来源更新频率较低）
- 检查通知设置是否正确配置
- 检查站内通知是否被免打扰策略过滤

### Q8：如何管理过期来源？

- 系统自动监控来源有效期，过期来源标记为"已过期"
- 过期来源关联的知识资产会生成质控预警
- 处理方式：
  - 更新来源：上传新版本指南/文献，重新审核
  - 延长有效期：如来源仍有效，可手动延长有效期
  - 标记废弃：来源确已过时，标记为废弃并关联替代来源

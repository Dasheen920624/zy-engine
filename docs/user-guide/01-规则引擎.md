# 规则引擎用户手册

> 适用版本：MedKernel v1.0 GA
> 适用对象：医院信息科管理员、质控科人员、路径管理员
> 最后更新：2026-05-23

---

## 1. 功能概述

规则引擎是 MedKernel 平台的核心决策组件，负责将医学指南、院内制度、医保政策和质控要求转化为可执行的规则，在临床现场实时提醒医生，在质控侧形成整改闭环。

规则引擎覆盖以下业务场景：

- **医嘱安全审核**：对高风险医嘱（如超剂量、禁忌症用药）进行实时拦截或提醒
- **医保合规审核**：基于 DRG/DIP 规则检查医嘱合规性，减少医保拒付风险
- **病历质控**：检查病历书写完整性、诊断与医嘱一致性等
- **路径质控**：校验患者诊疗行为是否符合临床路径规范

所有规则执行结果均保留审计痕迹，支持来源追溯，满足等保 2.0 三级审计要求。

---

## 2. 核心概念

### 2.1 规则（Rule）

规则是规则引擎的最小执行单元，定义了一条具体的判断逻辑和对应的处理动作。

| 属性 | 说明 |
|------|------|
| 规则编码 | 规则的唯一标识，如 `R_HF_DIURESIS`（心衰利尿规则） |
| 规则名称 | 业务可读的规则名称，如"心衰患者利尿剂使用审核" |
| 场景类型 | 规则适用的业务场景：医嘱安全（ORDER_SAFETY）、医保审核（INSURANCE_QC）、病历质控（EMR_QC）、路径质控（PATHWAY_QC） |
| 严重度 | 规则命中后的严重等级：提示（INFO）、警告（WARNING）、阻断（BLOCK） |
| 动作模式 | 命中后的处理方式：仅提醒（ALERT）、需确认（CONFIRM）、强制拦截（BLOCK） |
| 状态 | 草稿（DRAFT）、待审核（PENDING_REVIEW）、已发布（PUBLISHED）、已下线（OFFLINE） |

### 2.2 规则组（Rule Group）

规则组将多条相关规则组织在一起，便于统一管理和发布。

- 按业务场景分组：如"医保审核规则组""医嘱安全规则组"
- 按科室分组：如"心内科规则组""肿瘤科规则组"
- 规则组支持版本化管理，可整体发布和回滚

### 2.3 触发点（Trigger Point）

触发点定义了规则在什么时机被调用。平台支持以下触发方式：

| 触发方式 | 说明 | 典型场景 |
|----------|------|----------|
| 医嘱提交 | 医生在 HIS 中提交医嘱时触发 | 医嘱安全审核、医保合规检查 |
| 路径入径 | 患者进入临床路径时触发 | 入径条件校验 |
| 节点流转 | 路径节点状态变更时触发 | 节点完成条件检查 |
| 定时巡检 | 按计划周期性执行 | 病历质控、批量质控检查 |
| 手动触发 | 在规则校验工作台手动执行 | 规则调试、试运行验证 |

### 2.4 评估（Evaluation）

评估是规则引擎的一次完整执行过程，包含输入、执行和输出三个阶段。

- **输入**：患者上下文（患者信息、就诊信息、临床事实）
- **执行**：规则引擎匹配规则并计算命中结果
- **输出**：评估结果（命中/未命中）、命中原因、来源依据、建议动作

每次评估均生成唯一的评估 ID（evalId），用于结果查询和审计追溯。

## 3. 操作步骤

### 3.1 创建规则

1. 登录 MedKernel 管理工作台
2. 在左侧菜单选择 **试点准备 → 规则库**
3. 点击页面上方 **新建规则** 按钮
4. 选择规则模板：
   - 系统提供"医保审核""医嘱安全""病历质控""路径质控"四类模板
   - 选择模板后自动填充规则基本信息和默认参数
5. 填写规则基本信息：
   - 规则编码：使用业务含义明确的编码，如 `R_AMI_ANTICOAG`（AMI 抗凝规则）
   - 规则名称：填写中文业务名称
   - 适用科室：选择规则适用的科室范围
   - 严重度：选择提示/警告/阻断
   - 动作模式：选择提醒/确认/拦截
6. 配置规则条件：
   - 普通模式：使用表单配置患者条件、医嘱条件、排除条件
   - 专家模式：切换到 DSL 编辑器编写规则表达式（需具备规则 DSL 基础）
7. 绑定来源文档：关联指南、文献等来源，确保规则可追溯
8. 点击 **保存**，规则状态变为"草稿"

> **提示**：建议优先使用系统提供的规则模板创建规则，避免从零开始编写 DSL。模板已内置常见业务场景的最佳实践配置。

### 3.2 配置触发点

1. 在规则库列表中，找到目标规则，点击 **编辑**
2. 切换到 **触发配置** 标签页
3. 选择触发方式：
   - 医嘱提交：选择关联的医嘱类型（药品/检查/检验/手术）
   - 路径入径：选择关联的路径模板
   - 定时巡检：配置执行周期（每日/每周/每月）和执行时间
4. 配置触发条件：设置规则执行的前置条件，如特定科室、特定诊断
5. 保存触发配置

> **注意**：阻断（BLOCK）类型的规则建议仅在医嘱提交触发点使用，避免在非关键路径上过度拦截影响临床效率。

### 3.3 发布规则

1. 在规则库列表中，选择状态为"草稿"或"待审核"的规则
2. 点击 **提交审核**
3. 审核人在待办中心收到审核任务：
   - 查看规则内容和影响范围
   - 确认来源文档完整性和有效性
   - 选择"通过"或"退回"
4. 审核通过后，点击 **发布**：
   - 系统自动校验规则完整性（条件、动作、来源是否齐全）
   - 校验不通过会提示具体缺失项
   - 校验通过后填写发布原因并确认
5. 发布成功后规则状态变为"已发布"，在配置的触发点生效

> **重要**：发布前建议使用规则校验工作台（临床运行 → 规则校验）进行试运行验证，确保规则逻辑正确且不会产生误报。

### 3.4 查看评估结果

1. 在左侧菜单选择 **质控改进 → 评估结果**
2. 默认展示最近评估结果列表，包含：
   - 评估时间、规则名称、患者信息
   - 命中状态（命中/未命中）
   - 严重度和动作模式
3. 使用筛选条件缩小范围：
   - 按规则编码、场景类型、严重度筛选
   - 按时间范围筛选
   - 按科室、医生筛选
4. 点击某条评估记录查看详情：
   - 命中原因和规则依据
   - 来源文档引用
   - 医生处理结果（采纳/忽略/覆盖）
   - 追溯 ID（traceId），用于全链路审计
5. 导出评估报告：支持 CSV 格式导出，用于医务处月度报告

## 4. 常见问题

### Q1：规则发布后没有生效，可能的原因有哪些？

检查以下几项：
- 规则状态是否为"已发布"（非草稿或待审核）
- 触发点是否已正确配置并启用
- 规则适用的科室范围是否包含当前科室
- 患者上下文是否满足规则的前置条件
- 如使用适配器获取外部数据，检查适配器连接状态是否正常

### Q2：规则命中误报率较高，如何优化？

- 检查规则条件是否过于宽泛，增加限定条件（如诊断、年龄范围）
- 在规则校验工作台使用真实患者数据试运行，验证命中逻辑
- 调整严重度：非关键提醒可从"警告"降级为"提示"
- 配置提醒疲劳治理参数，避免同一患者短时间内重复提醒

### Q3：如何回滚已发布的规则？

1. 在规则库中找到目标规则
2. 点击 **版本历史**，查看历史版本列表
3. 选择需要回滚的版本，点击 **回滚**
4. 填写回滚原因并确认
5. 系统自动将规则恢复到目标版本，原版本保留在历史记录中

### Q4：规则 DSL 编写报语法错误，如何排查？

- 切换到专家模式查看完整的 DSL 编辑器，错误位置会高亮标注
- 检查字段名是否与患者上下文中的事实字段一致
- 检查运算符和逻辑连接词（AND/OR/NOT）是否正确
- 使用规则校验工作台的单条验证功能逐步调试

### Q5：评估结果中来源文档显示为空，如何处理？

- 检查规则是否绑定了来源文档（规则编辑 → 来源绑定）
- 确认来源文档状态为"已审核"，未审核的来源不会在评估结果中展示
- 确认来源文档未过期，过期来源需更新后重新审核

### Q6：阻断规则拦截了正常医嘱，医生如何处理？

- 医生可在弹窗中选择"覆盖并说明理由"，填写覆盖原因后继续提交
- 覆盖操作会记录在审计日志中，包含覆盖人、覆盖原因和追溯 ID
- 信息科可在审计日志中查看覆盖统计，评估规则是否需要调整

### Q7：如何批量导入规则？

1. 在规则库页面点击 **导入** 按钮
2. 下载规则导入模板
3. 按模板格式填写规则数据
4. 上传导入文件，系统自动校验格式和内容
5. 校验通过后确认导入，导入的规则默认为"草稿"状态

### Q8：规则引擎性能如何保障？

- 规则引擎采用内存执行模式，单条规则评估耗时通常在毫秒级
- 核心 API P95 响应时间 < 300ms（参考 SLO 定义）
- 如遇性能问题，检查规则数量是否过多、规则条件是否涉及复杂计算
- 可在 Provider 状态页面查看规则引擎健康状态

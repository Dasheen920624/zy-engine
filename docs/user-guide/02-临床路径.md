# 临床路径用户手册

> 适用版本：MedKernel v1.0 GA
> 适用对象：医院信息科管理员、临床路径管理员、质控科人员
> 最后更新：2026-05-23

---

## 1. 功能概述

临床路径模块是 MedKernel 平台的核心业务组件，用于将原本"画在 Word 里"的路径定义转化为可执行、可版本化、可度量的数字化路径。平台支持从路径模板创建、患者入径、节点任务执行到变异登记的完整闭环管理。

临床路径模块的核心价值：

- **路径数字化**：将纸质路径转化为结构化模板，支持版本管理和灰度发布
- **执行闭环**：患者入径 → 节点流转 → 任务执行 → 变异记录 → 出径评估，全流程可追溯
- **智能推荐**：基于规则引擎和知识图谱，自动识别候选路径并推荐入径
- **质控联动**：路径执行数据自动汇入质控驾驶舱，形成改进闭环
- **来源追溯**：每个关键节点绑定指南来源，满足循证医学和审计要求

## 2. 核心概念

### 2.1 路径（Pathway）

路径是一个完整的诊疗流程模板，定义了某类疾病从入院到出院的标准诊疗步骤。

| 属性 | 说明 |
|------|------|
| 路径编码 | 路径的唯一标识，如 `AMI_STEMI`（急性心肌梗死 STEMI 路径） |
| 路径名称 | 业务可读的路径名称，如"AMI/STEMI 标准诊疗路径" |
| 适用科室 | 路径适用的临床科室 |
| 适用诊断 | 路径对应的 ICD 诊断编码 |
| 版本号 | 路径的版本标识，支持多版本并存和灰度发布 |
| 状态 | 草稿（DRAFT）、待审核（PENDING_REVIEW）、已发布（PUBLISHED）、已下线（OFFLINE） |

### 2.2 阶段（Stage）

阶段是路径中的时间分段，代表诊疗过程中的一个宏观阶段。

- 典型阶段划分：急诊期、急性期、恢复期、出院准备期
- 每个阶段包含若干节点，阶段之间有明确的流转条件
- 阶段可配置预计时长，用于监控路径执行进度

### 2.3 节点（Node）

节点是路径中的最小执行单元，代表一个具体的诊疗任务或检查环节。

| 属性 | 说明 |
|------|------|
| 节点编码 | 节点的唯一标识 |
| 节点名称 | 如"胸痛识别""再灌注策略评估""PCI 术后管理" |
| 节点类型 | 评估节点、治疗节点、检查节点、观察节点 |
| 完成条件 | 节点流转到下一节点的前置条件 |
| 关联任务 | 节点包含的具体医嘱或操作任务 |
| 来源文档 | 节点依据的指南文献引用 |

节点支持以下操作：
- **任务生成**：节点激活时自动生成待办任务，如"完成十二导联心电图"
- **条件流转**：满足完成条件后自动流转到下一节点
- **变异登记**：偏离标准流程时记录变异原因
- **医生确认**：关键节点需医生确认后才能流转

### 2.4 出入径标准

出入径标准定义了患者进入和退出路径的条件，是路径执行的关键控制点。

**入径标准**：
- 诊断匹配：患者诊断符合路径适用的 ICD 编码
- 临床条件：满足特定的临床指标（如心电图 ST 段抬高）
- 排除条件：排除不符合路径执行条件的患者

**出径标准**：
- 正常出径：完成路径所有阶段和节点
- 变异出径：出现重大变异，无法继续按路径执行
- 转科出径：患者转入其他科室，路径自然终止

## 3. 操作步骤

### 3.1 创建路径

1. 登录 MedKernel 管理工作台
2. 在左侧菜单选择 **试点准备 → 路径配置**
3. 点击 **新建路径** 按钮
4. 选择路径模板：
   - 系统提供常见专病路径模板（如 AMI/STEMI、心衰、脑卒中等）
   - 选择模板后自动生成路径结构，包含阶段和节点
5. 填写路径基本信息：
   - 路径编码：使用标准缩写，如 `AMI_STEMI`
   - 路径名称：填写中文全称
   - 适用科室：选择路径归属科室
   - 适用诊断：关联 ICD-10 诊断编码
6. 配置入径标准：
   - 诊断条件：选择必须满足的诊断
   - 临床条件：添加必要的临床指标条件
   - 排除条件：添加排除入径的条件
7. 配置出径标准：
   - 正常出径条件
   - 变异出径阈值
8. 绑定来源文档：关联临床指南、专家共识等来源
9. 点击 **保存**，路径状态变为"草稿"

> **提示**：建议优先从系统模板创建路径，模板已内置标准阶段划分和节点配置。如需自定义，可在模板基础上修改。

### 3.2 配置阶段和节点

1. 在路径配置列表中，找到目标路径，点击 **编辑**
2. 切换到 **路径结构** 标签页
3. 配置阶段：
   - 点击 **添加阶段**，填写阶段名称和预计时长
   - 拖拽调整阶段顺序
   - 配置阶段间的流转条件
4. 配置节点：
   - 在阶段内点击 **添加节点**
   - 填写节点基本信息（名称、类型、预计完成时间）
   - 配置完成条件：
     - 普通模式：使用表单选择条件类型和参数
     - 专家模式：使用画布编辑器可视化配置节点关系
   - 添加关联任务：为节点绑定医嘱、检查、评估等任务
   - 绑定来源文档：为关键节点关联指南依据
5. 配置节点间流转规则：
   - 顺序流转：前一节点完成后自动进入下一节点
   - 条件流转：根据评估结果选择不同分支
   - 并行节点：多个节点可同时执行
6. 保存路径结构

> **注意**：每个关键节点都应绑定来源文档。发布时系统会检查来源完整性，缺少来源的节点将阻断发布。

### 3.3 发布路径

1. 在路径配置列表中，选择状态为"草稿"的路径
2. 点击 **提交审核**
3. 系统自动校验路径完整性：
   - 阶段和节点是否完整
   - 出入径标准是否已配置
   - 关键节点是否绑定来源文档
   - 关联规则是否存在冲突
4. 校验通过后进入审核流程：
   - 审核人在待办中心查看路径详情
   - 确认路径结构和来源完整性
   - 选择"通过"或"退回"
5. 审核通过后，点击 **发布**：
   - 查看影响评估：显示本次发布影响的科室和患者范围
   - 填写发布原因并确认
6. 发布成功后路径状态变为"已发布"，可在患者入径时使用

> **重要**：支持版本对比功能。发布新版本前，可在路径详情页点击"版本对比"查看与当前生效版本的差异，确认变更内容。

### 3.4 患者入径管理

1. 在左侧菜单选择 **临床运行 → 患者路径**
2. 查看患者路径列表：
   - 默认展示当前科室进行中的患者路径
   - 可按状态（进行中/已完成/变异出径）和路径类型筛选
3. 患者入径操作：
   - 系统基于规则引擎自动识别候选患者，在列表中标记"推荐入径"
   - 点击患者记录，查看入径推荐详情和证据依据
   - 医生确认后点击 **入径**，系统生成患者路径实例
4. 节点任务管理：
   - 入径后自动激活首节点，生成待办任务
   - 在患者路径详情页查看当前节点和待办任务
   - 完成任务后，节点自动流转到下一节点
5. 变异登记：
   - 当诊疗偏离路径标准时，点击 **登记变异**
   - 选择变异类型（患者因素/医源性因素/系统因素）
   - 填写变异原因和处理措施
   - 重大变异可能导致路径终止（变异出径）
6. 出径评估：
   - 患者完成所有阶段后，系统自动评估出径条件
   - 确认出径后，路径状态变为"已完成"
   - 生成路径执行报告，包含节点完成率、变异率等指标

## 4. 常见问题

### Q1：患者没有被自动推荐入径，可能的原因？

- 检查患者诊断是否匹配路径的适用诊断（ICD 编码）
- 检查入径规则是否已发布且触发点已启用
- 检查患者是否已在其他路径中（同一患者同一时间只能在一个路径中）
- 检查适配器是否正常获取了患者的临床数据

### Q2：节点无法自动流转到下一节点，如何处理？

- 检查当前节点的完成条件是否已全部满足
- 部分节点需要医生手动确认，确认后才会流转
- 检查关联的医嘱/检查是否已完成
- 如为条件流转节点，检查评估结果是否满足分支条件

### Q3：如何查看路径的版本差异？

1. 在路径配置列表中，点击目标路径进入详情
2. 点击 **版本对比** 按钮
3. 选择对比的两个版本（如当前生效版本 vs 新草稿版本）
4. 系统展示差异视图，标注新增、修改和删除的阶段/节点

### Q4：路径执行中变异率过高，如何优化？

- 在质控驾驶舱查看变异分析报告，识别高频变异节点
- 分析变异原因分布：患者因素、医源性因素、系统因素
- 针对高频变异节点调整路径结构或完成条件
- 考虑增加可选节点，提高路径的灵活性
- 与临床科室沟通，确认路径是否符合实际诊疗流程

### Q5：如何回滚已发布的路径版本？

1. 在路径配置列表中，找到目标路径
2. 点击 **版本历史**，查看所有历史版本
3. 选择需要回滚的目标版本
4. 点击 **回滚**，填写回滚原因
5. 确认回滚后，系统将当前生效版本替换为目标版本
6. 已在执行中的患者路径不受影响，新入径患者使用回滚后的版本

### Q6：患者路径数据如何与质控驾驶舱联动？

- 路径执行数据（入径率、完成率、变异率、节点滞留时长）自动汇入质控驾驶舱
- 在质控驾驶舱可按科室、路径类型、时间范围查看统计
- 变异登记自动生成质控预警，进入整改闭环流程
- 路径执行报告支持导出，用于医务处月度分析

### Q7：同一患者能否同时进入多个路径？

- 当前版本不支持同一患者同时处于多个路径中
- 如患者需要转入其他路径，需先从当前路径出径（变异出径），再入新路径
- 出径时系统会记录出径原因，确保审计完整

### Q8：路径配置中来源文档缺失会有什么影响？

- 发布时系统会自动校验关键节点的来源绑定
- 缺少来源的路径无法通过发布审核
- 已发布路径的来源文档过期后，系统会生成质控预警
- 建议在创建路径时同步绑定来源，避免发布阻断

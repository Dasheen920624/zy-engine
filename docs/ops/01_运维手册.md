## 1. 系统架构概览

```
┌─────────────┐    HTTPS     ┌──────────────┐    HTTP     ┌──────────────────┐
│   浏览器     │ ──────────→ │   Nginx      │ ──────────→ │  MedKernel MVP   │
│  (前端SPA)   │             │  (TLS 终结)   │             │  (Spring Boot)   │
└─────────────┘             └──────────────┘             └──────────────────┘
                                                                  │
                                                         ┌────────┼────────┐
                                                         │        │        │
                                                    ┌────┴──┐ ┌──┴───┐ ┌┴──────┐
                                                    │ H2/   │ │Neo4j │ │Dify/  │
                                                    │MySQL  │ │(可选) │ │LLM(可选)│
                                                    └───────┘ └──────┘ └───────┘
```

### 1.1 端口规划

| 服务 | 端口 | 用途 |
|------|------|------|
| Nginx | 443 (HTTPS) / 80 (HTTP→HTTPS) | 前端静态资源 + API 反向代理 |
| MedKernel 业务 | 18080 | REST API |
| MedKernel 管理 | 18081 | Actuator/健康检查/Prometheus |
| Prometheus | 9090 | 指标采集 |
| Grafana | 3000 | 监控看板 |

### 1.2 目录结构

```
/opt/medkernel/
├── app/                    # 应用程序
│   ├── medkernel-mvp.jar   # Spring Boot 可执行 JAR
│   └── application.yml     # 配置文件
├── frontend/               # 前端静态资源
│   └── dist/               # 构建产物
├── deploy/                 # 部署配置
│   ├── nginx/              # Nginx 配置
│   ├── monitoring/         # Prometheus + Grafana
│   └── scripts/            # 运维脚本
├── logs/                   # 应用日志
├── backup/                 # 数据备份
└── data/                   # H2 数据文件（开发模式）
```

## 2. 部署指南

### 2.1 环境要求

| 组件 | 最低版本 | 推荐版本 |
|------|----------|----------|
| JDK | 8u151+ | 17 |
| OS | CentOS 7 / Ubuntu 18.04 | Ubuntu 22.04 LTS |
| 内存 | 4 GB | 8 GB |
| 磁盘 | 20 GB | 50 GB SSD |
| Nginx | 1.18+ | 1.24+ |

### 2.2 安装步骤

```bash
# 1. 安装 JDK
sudo yum install java-17-openjdk -y   # CentOS
sudo apt install openjdk-17-jdk -y    # Ubuntu

# 2. 创建应用目录
sudo mkdir -p /opt/medkernel/{app,frontend,logs,backup,data}
sudo useradd -r -s /bin/false medkernel
sudo chown -R medkernel:medkernel /opt/medkernel

# 3. 部署应用
cp medkernel-mvp.jar /opt/medkernel/app/
cp application.yml /opt/medkernel/app/
cp -r frontend/dist/* /opt/medkernel/frontend/

# 4. 配置 systemd 服务
cat > /etc/systemd/system/medkernel.service << 'SERVICE'
[Unit]
Description=MedKernel MVP
After=network.target

[Service]
Type=simple
User=medkernel
WorkingDirectory=/opt/medkernel/app
ExecStart=/usr/bin/java -jar medkernel-mvp.jar
Restart=on-failure
RestartSec=10

[Install]
WantedBy=multi-user.target
SERVICE

systemctl daemon-reload
systemctl enable medkernel
systemctl start medkernel

# 5. 配置 Nginx
cp deploy/nginx/medkernel-tls.conf /etc/nginx/conf.d/
nginx -t && systemctl reload nginx
```

### 2.3 配置说明

关键配置项（application.yml）：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `server.port` | 18080 | 业务端口 |
| `management.server.port` | 18081 | 管理端口 |
| `medkernel.database.mode` | local | 数据库模式：local/h2/mysql |
| `medkernel.graph.enabled` | false | 图数据库开关 |
| `medkernel.dify.enabled` | false | Dify/LLM 开关 |

## 3. 日常运维

### 3.1 服务管理

```bash
# 启动
systemctl start medkernel

# 停止
systemctl stop medkernel

# 重启
systemctl restart medkernel

# 查看状态
systemctl status medkernel

# 查看日志
tail -f /opt/medkernel/logs/medkernel.log
journalctl -u medkernel -f
```

### 3.2 健康检查

```bash
# 业务健康
curl -s http://localhost:18080/api/health | jq .

# Actuator 健康（含 Provider 状态）
curl -s http://localhost:18081/medkernel/actuator/health | jq .

# 就绪探针
curl -s http://localhost:18081/medkernel/actuator/health/readiness

# 存活探针
curl -s http://localhost:18081/medkernel/actuator/health/liveness
```

### 3.3 监控

```bash
# Prometheus 指标
curl -s http://localhost:18081/medkernel/actuator/prometheus | head -20

# 启动 Prometheus
docker run -d -p 9090:9090 \
  -v /opt/medkernel/deploy/monitoring/prometheus/prometheus.yml:/etc/prometheus/prometheus.yml \
  -v /opt/medkernel/deploy/monitoring/prometheus/alert_rules.yml:/etc/prometheus/alert_rules.yml \
  prom/prometheus

# 启动 Grafana
docker run -d -p 3000:3000 \
  -v /opt/medkernel/deploy/monitoring/grafana/dashboards:/var/lib/grafana/dashboards \
  grafana/grafana
```

### 3.4 数据备份

```bash
# H2 数据库备份（开发模式）
cp /opt/medkernel/data/medkernel.mv.db /opt/medkernel/backup/medkernel_$(date +%Y%m%d).mv.db

# MySQL 备份（生产模式）
mysqldump -u root -p medkernel > /opt/medkernel/backup/medkernel_$(date +%Y%m%d).sql

# 审计日志导出
curl -s "http://localhost:18080/api/admin/audit-logs?limit=10000" > /opt/medkernel/backup/audit_$(date +%Y%m%d).json
```

### 3.5 日志管理

| 日志文件 | 位置 | 保留期 | 说明 |
|----------|------|--------|------|
| 应用日志 | /opt/medkernel/logs/ | 30 天 | Spring Boot 日志 |
| Nginx 访问日志 | /var/log/nginx/access.log | 90 天 | HTTP 请求日志 |
| Nginx 错误日志 | /var/log/nginx/error.log | 90 天 | Nginx 错误 |
| 审计日志 | 数据库 sec_auth_audit_log | 180 天 | 等保合规 |

## 4. 安全运维

### 4.1 密钥轮换

```bash
# 通过 API 触发密钥轮换
curl -X POST http://localhost:18080/api/security/admin/keys/rotate \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"key_alias": "field-encryption", "rotated_by": "admin"}'
```

### 4.2 用户管理

```bash
# 创建用户
curl -X POST http://localhost:18080/api/security/admin/users \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"username": "doctor1", "password": "Str0ng!Pass", "display_name": "张医生"}'

# 强制用户下线
curl -X POST http://localhost:18080/api/security/admin/users/{userId}/force-logout \
  -H "Authorization: Bearer $TOKEN"

# 重置密码
curl -X POST http://localhost:18080/api/security/admin/users/{userId}/reset-password \
  -H "Authorization: Bearer $TOKEN"
```

### 4.3 审计链验证

```bash
# 验证审计链完整性
curl -s http://localhost:18080/api/security/admin/audit-chain/verify \
  -H "Authorization: Bearer $TOKEN" | jq .
```

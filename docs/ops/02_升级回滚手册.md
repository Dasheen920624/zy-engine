# MedKernel 升级与回滚手册

> 版本：1.0 · 2026-05-24
> 适用：MedKernel v1.0 GA
> 受众：医院信息科运维人员

## 1. 升级流程

### 1.1 升级前检查

```bash
# 1. 确认当前版本
curl -s http://localhost:18080/api/health | jq '.version'

# 2. 确认服务健康
curl -s http://localhost:18081/medkernel/actuator/health | jq '.status'

# 3. 备份数据（必须！）
mysqldump -u root -p medkernel > /opt/medkernel/backup/pre_upgrade_$(date +%Y%m%d%H%M).sql

# 4. 备份当前 JAR
cp /opt/medkernel/app/medkernel-mvp.jar /opt/medkernel/backup/medkernel-mvp_$(date +%Y%m%d%H%M).jar

# 5. 备份配置
cp /opt/medkernel/app/application.yml /opt/medkernel/backup/application_$(date +%Y%m%d%H%M).yml
```

### 1.2 滚动升级步骤

```bash
# 1. 下载新版本
cp medkernel-mvp-new.jar /opt/medkernel/app/medkernel-mvp-new.jar

# 2. 停止服务
systemctl stop medkernel

# 3. 替换 JAR
mv /opt/medkernel/app/medkernel-mvp.jar /opt/medkernel/app/medkernel-mvp-old.jar
mv /opt/medkernel/app/medkernel-mvp-new.jar /opt/medkernel/app/medkernel-mvp.jar

# 4. 启动服务
systemctl start medkernel

# 5. 验证升级
sleep 30  # 等待启动
curl -s http://localhost:18080/api/health | jq '.version'
curl -s http://localhost:18081/medkernel/actuator/health | jq '.status'
```

### 1.3 前端升级

```bash
# 1. 备份当前前端
cp -r /opt/medkernel/frontend/dist /opt/medkernel/backup/dist_$(date +%Y%m%d%H%M)

# 2. 部署新前端
rm -rf /opt/medkernel/frontend/dist
cp -r frontend/dist /opt/medkernel/frontend/

# 3. 清理 Nginx 缓存
nginx -s reload
```

### 1.4 数据库迁移

MedKernel 使用 Flyway 管理数据库版本。升级时自动执行迁移：

```bash
# 查看当前迁移版本
curl -s http://localhost:18080/api/system/providers | jq '.flyway'

# 迁移在启动时自动执行，无需手动操作
# 如需手动迁移：
# java -jar medkernel-mvp.jar --spring.flyway.baseline-on-migrate=true
```

## 2. 回滚流程

### 2.1 应用回滚

```bash
# 1. 停止服务
systemctl stop medkernel

# 2. 恢复旧版本 JAR
mv /opt/medkernel/app/medkernel-mvp.jar /opt/medkernel/app/medkernel-mvp-failed.jar
mv /opt/medkernel/app/medkernel-mvp-old.jar /opt/medkernel/app/medkernel-mvp.jar

# 3. 恢复配置（如有变更）
cp /opt/medkernel/backup/application_XXXXXXXXXX.yml /opt/medkernel/app/application.yml

# 4. 启动服务
systemctl start medkernel

# 5. 验证回滚
sleep 30
curl -s http://localhost:18080/api/health | jq '.version'
```

### 2.2 数据库回滚

```bash
# 警告：数据库回滚会丢失升级后的数据！

# 1. 停止服务
systemctl stop medkernel

# 2. 恢复数据库
mysql -u root -p medkernel < /opt/medkernel/backup/pre_upgrade_XXXXXXXXXX.sql

# 3. 启动服务
systemctl start medkernel
```

### 2.3 前端回滚

```bash
# 恢复旧前端
rm -rf /opt/medkernel/frontend/dist
cp -r /opt/medkernel/backup/dist_XXXXXXXXXX /opt/medkernel/frontend/dist
nginx -s reload
```

## 3. 多数据库升级

MedKernel 支持 H2/MySQL/PostgreSQL/Oracle/达梦/KingbaseES 六种数据库。

### 3.1 MySQL 升级

```bash
# 升级前备份
mysqldump -u root -p --single-transaction --routines medkernel > /opt/medkernel/backup/pre_upgrade_$(date +%Y%m%d%H%M).sql

# 升级后验证
mysql -u root -p -e "SELECT * FROM medkernel.flyway_schema_history ORDER BY installed_rank DESC LIMIT 5;"
```

### 3.2 PostgreSQL 升级

```bash
# 升级前备份
pg_dump -U postgres -d medkernel -F c -f /opt/medkernel/backup/pre_upgrade_$(date +%Y%m%d%H%M).dump

# 升级后验证
psql -U postgres -d medkernel -c "SELECT version, description, installed_on FROM flyway_schema_history ORDER BY installed_rank DESC LIMIT 5;"
```

### 3.3 Oracle 升级

```bash
# 升级前备份（RMAN）
rman target / << 'EOF'
BACKUP DATABASE PLUS ARCHIVELOG;
EOF

# 或使用 expdp
expdp system/password@medkernel schemas=MEDKERNEL directory=BACKUP_DIR dumpfile=pre_upgrade_$(date +%Y%m%d%H%M).dmp

# 升级后验证
sqlplus medkernel/password@medkernel << 'EOF'
SELECT version, description, installed_on FROM flyway_schema_history ORDER BY installed_rank DESC FETCH FIRST 5 ROWS ONLY;
EOF
```

### 3.4 达梦/KingbaseES 升级

```bash
# 达梦备份
dmbackup /opt/dameng/data/MEDKERNEL /opt/medkernel/backup/dm_$(date +%Y%m%d%H%M)

# KingbaseES 备份
sys_dump -U system -d medkernel -f /opt/medkernel/backup/kb_$(date +%Y%m%d%H%M).sql

# 升级后验证 Flyway 迁移历史
# 两种数据库均使用 flyway_schema_history 表
```

## 4. Flyway 迁移管理

### 4.1 查看迁移状态

```bash
# 通过 Actuator 查看
curl -s http://localhost:18081/medkernel/actuator/flyway | jq '.contexts.application.flywayBeans'

# 或直接查询数据库
mysql -u root -p -e "SELECT installed_rank, version, description, type, script, installed_on, success FROM medkernel.flyway_schema_history ORDER BY installed_rank;"
```

### 4.2 Flyway 回滚（需 Flyway Teams 版本）

```bash
# 注意：开源版 Flyway 不支持 undo 迁移
# 如需回滚数据库，必须使用备份恢复

# Teams 版本 undo 迁移
flyway -url=jdbc:mysql://localhost:3306/medkernel -user=root -password=xxx undo
```

### 4.3 Flyway 修复

```bash
# 修复失败的迁移记录
flyway -url=jdbc:mysql://localhost:3306/medkernel -user=root -password=xxx repair

# 或通过 Spring Boot
java -jar medkernel-mvp.jar --spring.flyway.repair-on-migrate=true
```

## 5. 配置包回滚

### 5.1 规则配置包回滚

```bash
# 1. 查看历史版本
curl -s "http://localhost:18080/api/config-packages?name=规则包&limit=10" \
  -H "Authorization: Bearer $TOKEN" | jq '.items[] | {id, version, status}'

# 2. 回滚到指定版本
curl -X POST "http://localhost:18080/api/config-packages/{id}/rollback" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"target_version": "1.0.2", "rolled_back_by": "admin"}'

# 3. 验证回滚
curl -s "http://localhost:18080/api/config-packages/{id}" \
  -H "Authorization: Bearer $TOKEN" | jq '.version, .status'
```

### 5.2 路径配置包回滚

```bash
# 1. 查看路径模板历史
curl -s "http://localhost:18080/api/pathway/templates/{id}/versions" \
  -H "Authorization: Bearer $TOKEN" | jq '.[] | {version, status, created_at}'

# 2. 回滚路径模板
curl -X POST "http://localhost:18080/api/pathway/templates/{id}/rollback" \
  -H "Authorization: Bearer $TOKEN" \
  -d '{"target_version": 3, "rolled_back_by": "admin"}'
```

## 6. 升级检查清单

升级前请逐项确认：

- [ ] 当前版本已确认
- [ ] 目标版本已确认
- [ ] 数据库完整备份已完成
- [ ] 应用 JAR 备份已完成
- [ ] 配置文件备份已完成
- [ ] 前端静态资源备份已完成
- [ ] 升级窗口已与业务方确认
- [ ] 回滚方案已准备
- [ ] 监控告警已确认正常
- [ ] 升级通知已发送

升级后请逐项验证：

- [ ] 服务启动成功
- [ ] 健康检查通过
- [ ] 版本号正确
- [ ] Flyway 迁移成功
- [ ] 核心功能可用（登录、规则评估、路径查询）
- [ ] 监控指标正常
- [ ] 日志无异常错误
